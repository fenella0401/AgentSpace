# 定时任务

## 1. 基本信息

| 字段 | 内容 |
| --- | --- |
| 模块名称 | 定时任务 |
| 最近更新 | 2026-06-13 |
| 模块目标 | 配置周期或固定时间调度规则，并查看和处理每次调度产生的任务实例 |
| 适用角色 | 团队创建者、团队管理员、团队成员、访客 |
| 导航入口 | 团队空间左侧导航“定时任务” |
| 上位定义 | [AgentSpace 产品定义](../product-definition.md) |

## 2. 模块边界

本模块包含：

- 长期存在的定时任务配置。
- 每次到达调度时间后产生的定时任务实例。
- 实例执行记录和关闭、重试操作。

单次实例完成不会结束定时任务配置；配置保持启用并等待下一次调度。实例关联的 Agent Run 或 AgentFlow Run 详情由[任务记录](./task-records.md)承载。

---

## 3. 信息架构

定时任务页面包含两个页签：

1. **任务配置**：配置清单、查看、新建、编辑和启停。
2. **任务执行记录**：按状态查看历次调度实例。

默认进入任务配置。页签行为与事件任务保持一致。

---

## 4. 任务配置清单

### 4.1 清单字段

每条配置展示：

- 任务名称。
- 调度时间或周期摘要。
- 执行方式：Agent 或 AgentFlow。
- 启用状态。

### 4.2 查询与筛选

- 按任务名称搜索。
- 按执行方式、调度类型和启用状态筛选。
- 默认按最近更新时间倒序。
- 清单为空和筛选无结果使用不同提示。

### 4.3 清单操作

- 查看详情。
- 编辑配置。
- 启用或停用。
- 查看实例。

停用后不再生成新实例，不取消正在运行或已经生成的实例。重新启用后从下一次符合规则的调度点开始执行，不补跑停用期间错过的任务；补跑策略如需支持，必须另行定义。

---

## 5. 新建与编辑

### 5.1 表单字段

| 字段 | 规则 |
| --- | --- |
| 任务名称 | 必填；团队空间内不可与其他定时任务重名 |
| 任务说明 | 必填 |
| 调度时间/周期 | 必填；使用系统支持的周期选择器 |
| 执行方式 | 必填：Agent 或 AgentFlow |
| 绑定对象 | 必填；只能选择当前可用的 Agent 或 AgentFlow |
| 使用模型 | 必填；只能选择当前可用的模型 |
| 使用项目文件夹 | 选填；从知识库中选择某个文件夹 |
| 预置 Prompt | 必填；作为每次实例的初始输入 |
| 是否立即启用 | 默认启用 |

以下规则首期由系统内置：

- 同时运行的定时任务实例上限。
- 任务参与人首版本默认为任务创建人。

### 5.2 调度规则

- 页面以当前团队空间统一时区展示和保存调度时间。
- 编辑时同时展示规则摘要和下一次执行时间预览。
- 无法计算下一次执行时间时阻止保存。
- 修改调度规则只影响保存后尚未生成的实例。
- 已生成实例保留创建时的配置快照。
- 首期支持的具体周期粒度由调度服务能力决定；前端不得展示后端不支持的表达式。

### 5.3 执行方式联动

- 选择 Agent 或 AgentFlow 后加载对应可用对象。
- 切换执行方式时清空原绑定对象。
- 绑定对象不可用、已停用或未发布时，阻止启用配置。
- AgentFlow 首 Step 的 实际 Prompt 输入拼接任务提示词和 Step 设置输入提示词。

### 5.4 保存反馈

1. 提交前校验必填项、调度规则、参与人和绑定对象。
2. 保存成功后刷新清单并展示下次执行时间。
3. 保存失败时保留输入和调度预览。
4. 配置冲突时阻止覆盖并提示刷新。
5. 审计写入失败时，配置变更不得静默成功。

---

## 6. 调度与实例生成

1. 启用配置到达调度时间。
2. 系统创建定时任务实例并固化任务配置和 Harness 引用快照。
3. 实例进入系统调度并启动 Agent 或 AgentFlow。
4. 达到系统并发上限时，由系统排队等待。
5. 单次实例结束后，配置继续等待下一次调度。

配置本身不进入 Succeeded、Failed 或 Cancelled；这些状态只属于每次实例。

---

## 7. 定时任务执行记录

### 7.1 状态类型

- Running。
- Waiting Approval。
- Blocked。
- Succeeded。
- Failed。
- Cancelled。

定时任务没有 TODO 状态。

### 7.2 卡片信息

每张卡片展示：

- 任务名称。
- 任务启动执行时间和耗时。
- 执行状态。

点击卡片进入实例运行的任务记录。

### 7.3 查询与筛选

- 支持按实例名称和配置名称搜索。
- 支持按 Agent/AgentFlow、参与人和执行时间筛选。

---

## 8. 实例操作

| 当前显示状态 | 操作 | 规则 |
| --- | --- | --- |
| Running | 关闭 | 二次确认并填写原因，实例转为 Cancelled |
| Waiting Approval | 进入处理、关闭 | 人工处理在任务记录中完成 |
| Blocked | 查看原因、关闭 | 依赖恢复后系统可继续；关闭后转为 Cancelled |
| Failed | 重试 | 实例重新进入 Running，保留失败尝试并记录新的 Agent Run 及重试次数 |
| Cancelled | 重试 | 实例重新进入 Running，保留取消记录并记录新的 Agent Run 及重试次数 |
| Succeeded | 查看结果 | 不允许修改完成状态 |

重试只影响该次实例，不修改定时任务配置，也不改变下一次正常调度时间。

---

## 9. 权限与只读表现

| 能力 | 创建者/管理员 | 团队成员 | 访客 |
| --- | --- | --- | --- |
| 查看配置清单 | 是 | 按授权只读 | 否 |
| 新建、编辑、启停配置 | 是 | 否 | 否 |
| 查看实例清单和详情 | 是 | 是 | 是 |
| 处理 Waiting Approval | 是 | 仅任务参与人 | 否 |
| 关闭或重试实例 | 是 | 仅被授权任务 | 否 |

被授权只读查看配置时，配置详情不展示保存或启停操作；访客只进入实例清单和被授权的实例详情，不展示任何控制操作。

---

## 10. 页面状态与反馈

- **配置为空**：展示“暂无定时任务”，管理员可新建。
- **调度规则无效**：在对应字段显示原因，不提交表单。
- **无可用 Agent/AgentFlow**：提示前往 Harness 配置。
- **错过调度时间**：首期不自动补跑，并在审计中记录原因。
- **调度服务不可用**：已有配置保持可见并标记风险；系统恢复后按调度服务规则继续。
- **记录为空**：展示当前筛选下暂无实例。
- **无权限**：配置和记录保持只读。

## 12. 首期范围与扩展

首期支持基础周期配置、Agent/AgentFlow 执行、任务执行记录、关闭、重试和持续调度。

后续可扩展复杂日历规则、节假日策略、错过调度补跑、消息提醒、参与人自动匹配、任务准出校验和高级并发策略。
