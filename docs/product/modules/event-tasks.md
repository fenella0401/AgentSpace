# 事件任务

## 1. 基本信息

| 字段 | 内容 |
| --- | --- |
| 模块名称 | 事件任务 |
| 最近更新 | 2026-06-13 |
| 模块目标 | 配置事件触发规则，并通过实例看板查看和处理事件任务执行 |
| 适用角色 | 团队创建者、团队管理员、团队成员、访客 |
| 导航入口 | 团队空间左侧导航“事件任务” |
| 上位定义 | [AgentSpace 产品定义](../product-definition.md) |

---

## 2. 模块边界

本模块包含两个层次：

- **事件任务配置**：长期存在的事件匹配和执行规则。
- **事件任务实例**：每次事件命中配置后生成的 Task Run。

事件任务实例关联的 Agent Run 或 AgentFlow Run 详情由[任务记录](./task-records.md)承载。

630版本仅支持 eDevOps FE 外部事件源。团队空间内部资源事件和更多外部事件源作为后续扩展。

---

## 3. 信息架构

事件任务页面包含两个页签：

1. **任务配置**：配置清单、查看、新建、编辑、启停。
2. **任务看板**：按状态查看事件任务实例并执行启动、关闭和重试。

默认进入任务配置。系统记住用户在当前团队空间最后使用的页签，但不跨团队空间继承。

## 4. 任务配置清单

![事件任务配置清单和新建任务低保真图](../pic/event%20task-setup.png)

### 4.1 清单字段

每条配置展示：

- 任务名称。
- 事件对象。
- 触发动作或条件摘要。
- 执行方式：Agent 或 AgentFlow。
- 绑定对象。
- 启动方式：自动执行或人工启动。
- 任务参与人。
- 启用状态。
- 操作入口。

### 4.2 查询与筛选

- 按任务名称搜索。
- 按事件对象、执行方式、启用状态和触发条件筛选。
- 支持清空全部筛选。
- 清单默认按最近更新时间倒序。
- 无结果时区分“暂无配置”和“没有符合筛选条件的配置”。

### 4.3 清单操作

- **查看详情**：只读查看完整规则和最近实例。
- **编辑**：打开与新建相同的表单，并加载当前配置。
- **启用/停用**：切换配置是否继续接收新事件。
- **更多**：预留审计记录等次要操作；未定义能力不得提前展示。

停用只影响新的事件匹配，不取消已生成的任务实例。

---

## 5. 新建与编辑

### 5.1 表单字段

| 字段 | 规则 |
| --- | --- |
| 任务名称 | 必填；团队空间内不可与其他事件任务重名 |
| 任务说明/提示词 | 必填；用于描述目标并作为执行输入的一部分 |
| 事件对象 | 必填；630版本选择 eDevOps FE 支持的对象 |
| 动作类型 | 必填；从事件对象支持的动作中选择 |
| 过滤条件 | 可选；同组条件关系和支持字段由事件源能力决定 |
| 执行方式 | 必填：Agent 或 AgentFlow |
| 绑定对象 | 必填；只能选择已发布且当前可用的 Agent 或 AgentFlow |
| 完成 checklist | 必填；至少一条，可排序和删除 |
| 任务参与人 | 必填；至少一名可处理实例的团队成员，访客不可选择 |
| 自动重试 | 按系统支持范围配置；上限由系统约束 |
| 触发后执行方式 | 必填：自动启动或人工启动 |
| 是否启用 | 默认启用；启用后新规则立即生效 |

以下能力在630版本由系统内置，不在前端展示：

- 事件源类型。
- 输入参数映射。
- 事件同时启动条数上限。
- 实例自动命名规则。

### 5.2 过滤条件

- 点击“添加过滤条件”后选择字段、运算符和值。
- 字段、运算符和值必须来自事件源返回的可选范围。
- 条件不完整时不能提交。
- 删除条件不影响其他已填写字段。
- 630版本不自行定义复杂条件组或嵌套逻辑；如后端仅支持全部满足，前端明确展示“同时满足以下条件”。

### 5.3 执行方式联动

- 选择 Agent 后只展示可用 Agent 列表。
- 选择 AgentFlow 后只展示已发布的 AgentFlow。
- 切换执行方式会清空原绑定对象，并二次确认已经填写的绑定选择将被移除。
- 绑定对象不可用时保留原值用于识别，但阻止保存为启用状态。

### 5.4 保存反馈

1. 用户提交后先执行字段和依赖校验。
2. 保存成功后关闭弹窗、刷新清单并高亮新增或修改项。
3. 保存失败时保留表单内容，在字段旁或表单顶部展示原因。
4. 编辑期间配置已被他人修改时，阻止覆盖并提示刷新最新配置。
5. 审计记录保存失败时，配置变更不得静默成功。

## 6. 触发与实例生成

1. 启用的配置接收 eDevOps FE 事件。
2. 系统匹配事件对象、动作类型和过滤条件。
3. 命中后生成事件任务实例并固化配置快照。
4. 自动启动的实例在并发额度允许时进入 Running。
5. 人工启动的实例进入 Pending，并在看板显示为 TODO。
6. 自动启动实例因并发上限等待时进入 Ready，同样显示为 TODO，但必须标记“等待系统调度”，不能展示人工“启动”按钮。

### 防循环

同一事件任务执行过程产生的资源变更，默认不再次触发同一个事件任务。系统应保存触发链路标识，并在被拦截的循环事件上保留审计记录。

## 7. 事件任务实例看板

![事件任务实例看板低保真图](../pic/event%20task-kanban.png)

### 7.1 看板列

看板按以下状态分列：

- TODO。
- Running。
- Waiting Approval。
- Blocked。
- Succeeded。
- Failed。
- Cancelled。

状态与基础状态的映射遵循[产品定义](../product-definition.md#8-统一执行状态机)。

### 7.2 卡片信息

每张卡片展示：

- 实例名称。
- 事件任务配置名称。
- 事件对象或来源摘要。
- 创建时间或完成时间。
- 当前状态。
- 任务参与人。
- 关联 Agent 或 AgentFlow。
- 阻塞或失败原因摘要（如有）。

点击卡片打开实例详情抽屉；点击关联 Agent/AgentFlow 运行进入任务记录详情。

### 7.3 看板查询

- 支持按实例名称搜索。
- 支持按来源配置、AgentFlow、执行人和创建时间筛选。
- 支持手动刷新。
- 状态变化时卡片实时移动；用户打开的详情抽屉保持当前实例。

## 8. 实例操作

| 当前显示状态 | 操作 | 规则 |
| --- | --- | --- |
| TODO（人工启动） | 启动 | 仅配置为人工启动且具备权限时可用 |
| TODO（等待调度） | 无 | 展示等待原因和队列状态，不允许人工绕过并发调度 |
| Running | 关闭 | 二次确认后转为 Cancelled |
| Waiting Approval | 进入处理、关闭 | 审批在任务记录详情完成；关闭后转为 Cancelled |
| Blocked | 查看原因、关闭 | 依赖恢复后由系统继续；630版本不提供任意状态编辑 |
| Failed | 重试 | 实例重新进入 Running，保留失败尝试并记录新的 Agent Run 及重试次数 |
| Cancelled | 重试 | 实例重新进入 Running，保留取消记录并记录新的 Agent Run 及重试次数 |
| Succeeded | 查看结果 | 不允许修改完成状态 |

“关闭”必须填写或确认关闭原因。所有启动、关闭和重试操作记录操作者与时间。

## 9. 权限与只读表现

| 能力 | 创建者/管理员 | 团队成员 | 访客 |
| --- | --- | --- | --- |
| 查看配置清单 | 是 | 按授权只读 | 否 |
| 新建、编辑、启停配置 | 是 | 否 | 否 |
| 查看实例看板和详情 | 是 | 是 | 是 |
| 启动人工任务 | 是 | 仅任务参与人 | 否 |
| 处理 Waiting Approval | 是 | 仅任务参与人 | 否 |
| 关闭或重试实例 | 是 | 仅被授权任务 | 否 |

被授权只读查看配置的用户，表单字段只读且不展示保存、启停和删除操作。访客只进入实例看板和被授权的实例详情。

## 10. 页面状态与反馈

- **配置为空**：展示“暂无事件任务”，管理员可新建任务。
- **事件源不可用**：已有配置保持可见并标记不可用；禁止创建依赖该事件源的新启用配置。
- **无可用 Agent/AgentFlow**：表单提示先完成 Harness 配置并提供跳转。
- **看板为空**：展示当前筛选下暂无事件任务实例。
- **实时更新失败**：保留卡片并提示刷新，不回退已展示状态。
- **触发失败**：记录失败原因并进入 Failed 或系统定义的阻塞状态，不丢弃事件。
- **无权限**：看板保持只读，不展示实例控制按钮。

## 12. 630版本范围与扩展

630版本支持 eDevOps FE 事件、基础过滤、Agent/AgentFlow 执行、自动或人工启动、防循环、实例看板和运行控制。

后续可扩展团队空间内部资源事件、更多外部事件源、复杂条件组、参与人自动匹配、消息提醒和更细粒度的并发策略。
