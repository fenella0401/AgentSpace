## 1. 基本信息

| 字段 | 内容 |
| --- | --- |
| 模块名称 | 任务记录 |
| 最近更新 | 2026-06-22 |

## 2. 模块边界

本模块管理运行态记录，不管理事件任务、定时任务或 Harness 的长期配置。

包括：

- 查看来自事件任务、定时任务和临时任务的 Agent Run。
- 查看 AgentFlow Run 当前步骤及其包含的多个 Agent Run。
- 查看执行输入、过程、输出、人工交互和任务产物。
- 在被授权的 Waiting Human 节点完成审批、补充信息或审查。
- 发起临时任务并继续与 Agent 协作。

不包括：

- 修改事件或定时触发规则。
- 编辑 Agent 角色或 AgentFlow 定义。
- 直接管理 Codehub 仓库。

---

## 3. 信息架构

页面采用三栏工作区：

1. **任务记录栏**：搜索、新建临时任务、运行记录清单。
2. **运行详情区**：按 Stage 分组的 AgentFlow Step 进度、执行消息、MCP 调用、文件和用户输入。
3. **任务概览栏**：上下文用量、基础信息、待办、任务产物和本次运行使用的 Agent 配置快照。

当窗口宽度不足时，任务概览栏默认收起，通过“任务概览”按钮打开；任务记录栏允许收起，但运行详情始终保留。

## 4. 任务记录清单

### 4.1 展示粒度

清单以 Agent Run 为主要展示粒度：

- 单 Agent 任务展示一个 Agent Run。
- AgentFlow 任务展示当前运行或最后运行 Step 对应的 Agent Run，并标记所属 AgentFlow。
- 同一 AgentFlow 的其他 Step 不重复占用一级清单项，在详情内切换查看。

### 4.2 清单项信息

每条记录展示：

- 自动生成的运行名称。
- 来源类型：事件任务、定时任务或临时任务。
- 执行类型：Agent 或 AgentFlow。
- 当前状态。
- Agent 角色。
- 触发人或触发规则。
- 所属 AgentFlow（如有）。
- 所属事件任务或定时任务实例（如有）。
- 创建时间或最后更新时间。

状态视觉遵循产品定义中的统一状态映射。任务记录只展示 `Running`、`Waiting Approval`、`Done` 和 `Failed` 四类聚合状态。

### 4.3 搜索与筛选

- 搜索匹配运行名称、Agent 名称和 AgentFlow 名称。
- 支持按来源类型、执行类型和状态筛选。
- 多个筛选条件同时生效。
- 搜索或筛选变化后回到结果顶部。
- 无结果时展示“未找到符合条件的任务记录”，并提供清空筛选操作。

### 4.4 刷新与选中

- 新运行产生或状态更新时，当前清单项实时更新。
- 用户正在查看详情时，不因清单排序变化自动切换选中项。
- 选中项在清单中保持高亮。
- 记录已无权访问时，从清单移除；若正在查看，详情切换为无权限状态。

## 5. 新建临时任务

### 5.1 入口

任务记录栏顶部提供“新建任务”入口。点击后打开临时任务创建面板，不离开当前团队空间。

### 5.2 表单

| 字段 | 规则 |
| --- | --- |
| 执行方式 | 必选：Agent 或 AgentFlow |
| Agent / AgentFlow | 根据执行方式展示；选择 Agent 时允许使用团队空间默认 Agent |
| 初始输入 | 必填；支持多行文本 |
| 附件或文件引用 | 首期仅在系统已提供对应能力时展示，不作为创建必填项 |

选择 Agent 后，输入区提示该 Agent 角色的 Input Prompt；选择 AgentFlow 后，展示 Flow 启动命令并提示首个 Step 的 Input Prompt。提示仅作为输入引导，不自动覆盖用户已经输入的内容。

### 5.3 创建行为

1. 用户选择执行方式和目标。
2. 输入本次任务要求。
3. 点击发送后立即创建临时 Task Run。
4. 系统根据初始输入自动生成任务名称。
5. 页面进入新任务详情，并将记录插入清单顶部。
6. 创建失败时保留全部输入，展示失败原因并允许重试。

不指定 Agent 角色时使用团队空间默认 Agent。团队空间没有可用默认 Agent 时，阻止创建并引导前往 Harness 配置。

## 6. 运行详情

### 6.1 顶部信息

顶部展示：

- 运行名称与来源。
- 当前状态和最后更新时间。
- 按 Stage 分组的 AgentFlow Step 进度（仅 AgentFlow Run）。
- 会话、总览和场景等视图入口；首期以会话详情为默认视图。

### 6.2 AgentFlow Stage 与 Step 切换

- Stage 按 AgentFlow 顺序展示；Stage 内的 Step 按配置顺序展示 Agent 名称和状态。
- Stage 仅用于逻辑分组和聚合展示，不生成独立 Agent Run。
- 当前 Step 高亮，并展示其对应 Agent Run。
- 已完成 Step 可点击回看。
- 未开始 Step 只展示定义摘要，不展示虚构运行记录。
- Step 状态变化时更新进度，但不强制把用户从正在回看的 Step 切走。
- Step 自动门禁失败时展示产物校验结果、当前重试次数和最大重试次数；重试耗尽后展示 Step 失败原因。
- Step 开启人工确认时，仅在自动门禁通过后进入 Waiting Human 并展示确认操作。

### 6.3 执行消息

详情按时间顺序展示：

- 用户初始输入和后续指令。
- Agent 文本消息。
- Plan 和可展示的过程摘要。
- Skill、MCP 和 SubAgent 调用。
- 状态变化和 AgentFlow Step 流转。
- 错误、重试和阻塞原因。
- 文件读取、生成和修改记录。

工具调用默认折叠，展示工具名称、调用状态和耗时；展开后展示允许用户查看的输入输出。敏感信息必须脱敏。

### 6.4 输入信息

任务概览中展示本次运行固化的输入快照：

- 触发来源和关联任务配置。
- 初始 Prompt。
- Agent 角色和模型。
- Agent System Prompt 与最终 Input Prompt。
- 可用 Skill、MCP 和知识库范围。
- 环境变量引用状态，不展示值。
- AgentFlow、Stage 与 Step 配置快照（如有），包括 Flow 启动命令、产物清单、自动门禁重试次数和人工确认开关。

### 6.5 输出与产物

- 文本结果显示在会话时间线。
- 文件产物集中显示在“任务产物”区域。
- 点击文件打开预览或 diff。
- 每个产物展示文件名、类型、大小、生成时间和关联 Agent Run。
- 指定产物清单的 Step 展示各路径是否存在变更，并将结果关联到对应自动门禁记录。
- AgentSpace 仅展示 Codehub 变更草稿和审查状态，不提供仓库管理能力。

## 7. Human in the loop

### 7.1 可处理事项

当状态为 `Waiting Human` 时，详情明确展示等待原因、处理人范围和可执行动作。动作按节点配置包括：

- 审批通过并继续。
- 驳回并填写原因。
- 补充信息或新指令。
- 审查并接受或拒绝文件变更。
- 终止本次执行。

### 7.2 处理规则

- 只有任务参与人或被指定处理人可以提交操作。
- 必填说明为空时不得提交驳回或补充信息。
- 提交期间按钮进入加载态并防止重复提交。
- 后端返回成功后再更新状态和时间线。
- 权限变化、节点已被他人处理或运行已结束时，停止提交并刷新最新状态。
- 所有人工操作记录操作者、动作、时间和输入内容。

## 8. 运行控制

| 当前状态 | 可用操作 | 结果 |
| --- | --- | --- |
| Running | 终止 | 转为 Cancelled；临时会话由用户主动结束时在清单聚合为 Done |
| Waiting Approval | 节点配置允许的处理动作、终止 | 继续运行、保持等待或转为 Cancelled |
| Failed | 查看错误 | 首期不在任务记录直接重试事件或定时任务；跳转来源实例处理 |
| Done | 继续查看、继续临时会话（若后端允许） | 历史记录不被改写；继续会话产生新的交互事件 |

事件任务和定时任务的重试、关闭操作由对应实例看板提供。

## 9. 权限与只读表现

| 能力 | 创建者/管理员 | 团队成员 | 访客 |
| --- | --- | --- | --- |
| 查看公开任务摘要 | 是 | 是 | 是 |
| 查看被授权任务详情 | 是 | 是 | 是 |
| 新建临时任务 | 是 | 是 | 否 |
| 继续输入 | 是 | 是 | 否 |
| 处理 Waiting Human | 是 | 仅被授权任务 | 否 |
| 终止执行 | 是 | 仅本人或被授权任务 | 否 |

访客不展示输入框和运行控制按钮，并显示“当前角色仅支持查看”。普通团队成员无权处理某节点时，仍可查看详情，但操作区显示处理人信息。

## 10. 页面状态与反馈

- **加载中**：清单和详情分别显示骨架，互不阻塞。
- **清单为空**：展示“暂无任务记录”，并在有创建权限时提供“新建任务”。
- **详情不存在**：提示记录已删除或不可用，并返回清单。
- **无权限**：隐藏敏感内容，展示权限说明，不保留上一次详情数据。
- **实时连接中断**：保留现有内容，提示连接中断并自动重连；允许手动刷新。
- **运行失败**：在时间线和概览展示可理解的失败原因及关联来源入口。
- **外部产物不可用**：保留产物元数据，提示 Codehub 或外部系统暂不可访问。

## 12. 首期范围与扩展

首期支持统一清单、详情查看、按 Stage 分组的顺序 AgentFlow Step 切换、自动门禁重试、人工确认和文件变更审查。

后续可扩展执行回放、当前轮次文件修改回退、复杂 AgentFlow 可视化、会话分享和更细粒度的运行数据分析。
