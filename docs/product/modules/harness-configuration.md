## 2. 模块边界

Harness 描述 Agent“可以如何执行”，不展示某次具体执行的完整运行过程。

包括：

- 能力中心。
- Agent 角色。
- AgentFlow。
- 知识库。
- 环境变量。
- 配置保存、依赖校验和全局 Publish。

不包括：

- Skill 和 MCP 在 AgentCenter 中的发布、审核和版本维护。
- Codehub 仓库管理。
- Agent Run 和 AgentFlow Run 详情。
- 用户可感知的 Harness 历史版本对比或回滚。

---

## 3. 信息架构

Harness 的配置顺序为：能力中心、Agent 角色、AgentFlow、知识库、环境变量。能力中心是 Skill 与 MCP 的内部业务分组，在630版本不作为前端导航栏目展示。

前端使用顶部或侧边二级导航，六个入口依次为：

1. Skill。
2. MCP。
3. Agent 角色。
4. AgentFlow。
5. 知识库。
6. 环境变量。

Skill 和 MCP 与 Agent 角色处于同一导航层级，不再嵌套在“能力中心”导航下。

每个子模块独立编辑和保存。Harness 顶层提供 Publish，将当前已保存且校验通过的配置同步到 AgentCore 对应团队空间运行域。

首次 Publish 时，如 AgentCore 尚不存在对应团队空间运行域，由系统完成初始化；失败时保留 AgentSpace 配置并展示可重试原因。

## 4. 通用交互规则

### 4.1 查看与编辑

- 团队创建者和团队管理员可以进入编辑态。
- 团队成员和访客只能查看已保存配置，不展示保存、删除和 Publish 操作。
- 离开存在未保存修改的页面时弹出确认。
- 保存只更新 AgentSpace 当前配置，不等同于 Publish。
- Publish 前统一校验各模块引用和必填项。

### 4.2 Publish

1. 用户点击 Publish。
2. 系统检查 Agent、AgentFlow、能力、知识库和环境变量引用。
3. 校验失败时展示按模块分组的问题，并可跳转定位。
4. 校验通过后展示本次将同步的配置摘要。
5. 用户确认后同步到 AgentCore。
6. 成功后记录发布时间、操作者和结果。
7. 失败时不改变上一次已成功发布配置，并允许修复后重试。

630版本不向用户提供 Harness 历史版本列表、版本对比和回滚，但每次 Agent Run 必须固化启动时使用的配置快照。

### 4.3 依赖失效

- 被 Agent 或 AgentFlow 引用的资源删除前必须提示影响范围。
- 资源不可用时，相关 Agent 或 AgentFlow 标记为不可执行，并阻止 Publish。
- 已启动运行继续使用其快照解释，不被当前配置状态改写。

## 5. 能力中心

### 5.1 目标

配置团队空间内可被 Agent 角色引用的 Skill 和 MCP 集合。

### 5.2 前端入口与清单

630版本前端不展示“能力中心”导航栏目，分别通过同级的“Skill”和“MCP”入口展示对应清单。每项至少展示：

- 名称和类型。
- AgentCenter 版本。
- 能力摘要。
- 授权或可用状态。
- 被哪些 Agent 角色引用。
- 最近更新时间。

### 5.3 添加能力

1. 点击“添加 Skill”或“添加 MCP”。
2. 打开 AgentCenter 已发布资源选择器。
3. 支持搜索、查看版本和能力说明。
4. 选择目标版本并确认绑定。
5. 系统校验访问权限和资源可用性。
6. 成功后加入团队空间能力中心。

AgentSpace 不提供新建或编辑 Skill、MCP 的入口。

### 5.4 移除能力

- 未被引用时可直接二次确认后移除。
- 已被 Agent 角色或 AgentFlow 引用时展示引用清单并阻止直接移除。
- AgentCenter 资源下线或授权失效时，保留引用信息并标记不可用。

## 6. Agent 角色

### 6.1 默认 Agent

团队空间创建后系统自动生成默认 Agent 角色。默认 Agent 使用系统内置的 AgentRuntime、模型和执行方式，并使用空间内被授权的能力与知识范围。

630版本默认 Agent 在清单中展示但不可编辑。用户可以新增自定义 Agent 角色，也可以只使用默认 Agent。

### 6.2 清单

每个 Agent 展示：

- 名称和说明。
- 默认 Agent 标识。
- Skill 和 MCP 引用数量。
- System Prompt 和 Input Prompt 摘要。
- 被哪些 AgentFlow 或任务配置引用。
- 可用状态。

### 6.3 新建与编辑

630版本前端配置字段：

| 字段 | 规则 |
| --- | --- |
| 角色名称 | 必填；团队空间内唯一 |
| 角色说明 | 必填 |
| Skill | 从能力中心已绑定 Skill 中选择子集 |
| MCP | 从能力中心已绑定 MCP 中选择子集 |
| System Prompt | 必填；定义 Agent 的身份、约束和执行规则 |
| Input Prompt | 必填；用于新建临时任务、任务配置和 AgentFlow Step 的输入提示 |

AgentRuntime 类型和执行方式由系统内置，630版本不在前端配置或展示。

系统内置执行方式定义：

- 低信任权限：所有工具执行均需人工确认。
- 中信任权限：低风险工具可自动执行，文件写入、脚本执行、创建 MR 等高风险工具需人工确认。
- 高信任权限：允许自动执行全部已授权工具。

保存前检查全部引用仍然有效。删除被 AgentFlow 或任务配置引用的 Agent 时，必须先解除引用。

## 7. AgentFlow

> 配图占位：AgentFlow 清单。期望文件：`pic/agentflow list.png`。

> 配图占位：AgentFlow 配置页面。期望文件：`pic/agentflow.png`。

### 7.1 清单

每个 AgentFlow 展示：

- Flow 名称、启动命令和描述。
- Step 数量。
- 被哪些事件任务或定时任务引用。
- 配置可用状态。
- 最近更新时间。

支持搜索、查看、新建、编辑和删除未被引用的 AgentFlow。

### 7.2 新建与编辑 Flow

| 字段 | 规则 |
| --- | --- |
| Flow 名称 | 必填 |
| Flow 启动命令 | 必填；格式为 `/xxx-yyy-zzz`，校验表达式为 `^/[a-z]+(?:-[a-z]+)*$` |
| Flow 描述 | 选填 |

启动命令由 `/`、小写英文字母段和段间单个连字符组成，暂不允许大写字母、数字、空格、下划线、连续连字符或首尾连字符。`/xxx`、`/xxx-yyy` 和 `/xxx-yyy-zzz` 均为合法示例。630版本不校验启动命令唯一性。

新建 Flow 时系统默认生成一个 Stage，Stage 名称初始化为 Flow 名称。该名称只在创建时复制；之后修改 Flow 名称不会同步修改 Stage 名称。

Flow 编辑后可以保存为草稿态，或者选择发布；630版本 Flow 没有版本化管理，发布后不允许再次修改。

### 7.3 Stage

- Stage 是 AgentFlow 中的逻辑阶段，不直接执行 Agent。
- Stage 名称必填。
- 每个 Flow 至少包含一个 Stage，每个 Stage 在保存或 Publish 时必须包含至少一个 Step。
- 编辑过程中允许 Stage 暂时为空，但此时配置标记为待补全且不得 Publish。
- 支持新增、编辑、删除和拖拽排序 Stage；Flow 按 Stage 列表顺序执行。
- 删除或移动 Stage、Step 后，系统必须重新校验所有 Step 输出文件引用。

### 7.4 Step

每个 Step 只配置以下字段：

| 字段 | 规则 |
| --- | --- |
| Step 名称 | 必填；当前 Flow 内唯一，用于前序输出文件引用 |
| 执行 Agent | 必填；选择当前可用 Agent 角色 |
| Input Prompt | 必填；选择 Agent 后以其预置 Input Prompt 初始化，允许在当前 Step 编辑且不回写 Agent 角色 |
| 产物清单 | 选填；按文件路径逐条新增，例如 `/product/prd.md` |
| 自动门禁 | 系统根据产物清单自动控制，只读展示 |
| 自动门禁失败重试次数 | 自动门禁开启时必填；整数 `0–30`，默认 `3`，表示门禁失败后允许重新执行当前 Step 的次数 |
| 人工确认 | 开关；默认关闭 |

选择 Agent 后，Step 展示该角色预置的 Skill、MCP、System Prompt 和 Input Prompt。其中 Skill、MCP 和 System Prompt 只读；仅 Input Prompt 可在当前 Step 编辑。Step 实际执行输入按“Agent System Prompt + Step Input Prompt”拼接生成。

Input Prompt 支持通过 `@{Step名称}.{文件路径}` 引用此前 Step 的输出文件，例如 `@step1./product/prd.md`。引用目标必须是当前执行顺序之前且仍然存在的 Step；禁止引用当前 Step、后续 Step 或不存在的 Step。调整 Stage/Step 顺序、重命名或删除 Step 后，如引用失效，阻止保存和 Publish 并定位对应 Input Prompt。

### 7.5 准出机制

- 产物清单非空时，系统自动开启唯一固定门禁“校验产物清单是否存在变更”。630版本只向用户展示规则，不允许修改、删除或新增自动门禁。
- 未指定产物清单时，自动门禁为关闭态，自动门禁失败重试次数不生效。
- 自动门禁校验失败时重新执行当前 Step，直至校验通过或达到重试次数。重试次数为 `0` 时首次校验失败即结束，不追加重试。
- 自动门禁重试耗尽后 Step 失败；该重试只处理产物变更校验失败，不作为通用执行失败重试策略。
- 人工确认独立配置且默认关闭。自动门禁和人工确认同时启用时，先执行自动门禁；自动门禁通过后才进入人工确认。
- 人工确认关闭时，自动门禁通过或自动门禁关闭即完成 Step；人工确认开启时，用户确认通过后完成 Step。

### 7.6 编排交互

- 630版本按 Stage 和 Stage 内 Step 的列表顺序编排，不提供并行、循环、条件分支或子流程。
- 支持在 Stage 下新增、编辑、复制、删除和拖拽排序 Step。
- 删除或移动 Step 前展示受影响的输出文件引用。
- Flow 必须有起始 Stage、结束 Stage 及明确的首尾 Step。
- 不允许引用不可用 Agent，或保存存在缺失必填项、空 Stage、非法输出文件引用的 Flow。
- Step 不再配置其他准出规则、显式成功/失败流转及未在本节列出的执行策略。
- 保存后更新当前配置；Publish 后才可被新任务运行使用。

### 7.7 运行关系

Stage 不产生独立 Agent Run。AgentFlow Step Run 记录所属 Stage、最终执行输入、产物清单、自动门禁结果、重试次数和人工确认结果。AgentFlow Run 的 Stage/Step 进度、Agent Run 切换、输入输出和准出过程在[任务记录](./task-records.md)中查看，不在配置页混合展示。

## 8. 知识库

![知识库配置页面低保真图](../pic/knowledge%20base.png)

### 8.1 页面结构

- 左侧：知识库文件树、目录搜索和同步状态。
- 右侧：选中文件内容、元数据和编辑操作。
- 顶部或侧边：Skill、MCP、Agent 角色、AgentFlow、知识库和环境变量导航。

### 8.2 全局 Agent.md

团队空间全局 Agent.md 是每次 Agent 执行必须加载的全局上下文，可包含：

- 团队空间目标和业务背景。
- 工作规范与术语。
- Agent 执行约束。
- 文件和代码修改规则。
- 默认沟通风格。
- 任务完成标准。

交互规则：

- 支持在线生成草稿和 Markdown 编辑。
- 生成内容进入编辑态，不自动保存或 Publish。
- 展示 token 长度、结构检查和健康状态。
- 少于 3k tokens 或超过 15k tokens 时强提示；超过 30k tokens 阻止保存。
- 提示用户将细节拆到知识库其他文件。

### 8.3 文件目录

- 630版本知识目录绑定 Codehub 仓库分支目录。
- 文件树展示目录层级和同步状态。
- 搜索匹配文件名和目录名。
- 点击文件在右侧只读预览；可编辑范围取决于 Codehub 集成能力。
- 同步失败时保留上一次可用内容并展示失败状态。
- AgentSpace 不提供分支、提交历史等仓库管理功能。

## 9. 环境变量

### 9.1 清单字段

- 变量名称。
- 用途说明。
- 是否敏感。
- 配置状态。
- 引用资源。
- 最近更新人和时间。

### 9.2 新建与编辑

| 字段 | 规则 |
| --- | --- |
| 名称 | 必填；团队空间内唯一 |
| 说明 | 必填 |
| 是否敏感 | 创建时设置；敏感值写入统一密钥管理系统 |
| 值 | 非敏感变量可编辑；敏感变量不回填明文 |

敏感变量保存后只展示“已配置”状态。重新编辑敏感变量需要输入新值，取消不会清空原安全引用。

### 9.3 删除

被 Agent、MCP 或 AgentFlow Step 引用时阻止删除，并展示引用位置。删除后不得影响已启动运行的快照。

## 10. 权限与只读表现

| 能力 | 创建者/管理员 | 团队成员 | 访客 |
| --- | --- | --- | --- |
| 查看 Harness 配置 | 是 | 按授权只读 | 仅授权知识库内容 |
| 添加或移除能力 | 是 | 否 | 否 |
| 编辑知识库和 Agent.md | 是 | 否 | 否 |
| 管理环境变量 | 是 | 否 | 否 |
| 管理 Agent 和 AgentFlow | 是 | 否 | 否 |
| Publish | 是 | 否 | 否 |

无编辑权限时，表单字段、拖拽和资源操作入口全部转为只读，不仅依靠按钮隐藏。

## 11. 页面状态与反馈

- **初始空态**：说明团队空间可使用默认 Agent，并引导按需配置。
- **资源加载失败**：分模块展示失败，不阻断其他模块查看。
- **引用失效**：在资源和引用方同时标记，并提供跳转定位。
- **保存失败**：保留编辑内容，展示字段或系统错误。
- **Publish 校验失败**：按模块列出问题，不调用 AgentCore。
- **Publish 同步失败**：保留上一次发布结果和当前配置，允许重试。
- **权限变化**：后端拒绝写操作后切换为只读并提示权限已变化。

## 13. 630版本范围与扩展

630版本支持单 Harness、AgentCenter Skill/MCP 引用、Codehub 知识目录、全局 Agent.md、环境变量、自定义 Agent 角色、Stage/Step 顺序 AgentFlow 和全局 Publish。

后续可扩展多 Harness、用户可见版本管理、配置对比和回滚、复杂分支、通用执行失败重试、并行 Step、子流程和更多知识来源。
