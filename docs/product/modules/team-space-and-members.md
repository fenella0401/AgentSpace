## 2. 模块边界

本模块定义团队空间内部的创建、进入、切换和成员权限交互。

包括：

- 创建团队空间。
- 查看和切换用户有权访问的团队空间。
- 邀请成员。
- 分配团队管理员、团队成员和访客角色。
- 调整或移除非创建者成员。
- 处理只读、无权限和权限变化状态。

不包括：

- PDU 组织树的新建、编辑、移动或删除。
- 自定义角色或自定义权限点。
- Harness、事件任务和定时任务的业务配置。

---

## 3. 信息架构

本模块分布在两个层级：

1. **全局团队空间入口**：展示当前空间、切换空间和创建团队空间。
2. **团队空间成员管理页**：展示成员清单、添加成员、调整角色和移除成员。

角色和权限不是独立可编辑的自定义配置。系统根据成员角色控制任务记录、事件任务、定时任务和 Harness 中的入口、只读态及写操作。

## 4. 团队空间创建

### 4.1 入口与资格

- 具备团队空间创建资格的已登录用户可以看到“创建团队空间”入口。
- 团队空间独立创建，不要求用户先选择上级归属容器。
- 系统从用户身份和 PDU 组织关系中获取用户所属的最子部门，并作为团队空间的所属部门元数据。
- 所属部门在首版本仅展示，不允许用户创建时修改。
- 无法识别用户最子部门或创建资格校验失败时，不允许提交创建。

### 4.2 创建表单

| 字段 | 规则 |
| --- | --- |
| 所属部门 | 必填；系统自动带入用户所属最子部门；首版本只读，不允许修改 |
| 团队空间名称 | 必填；命名规则和唯一范围待产品确认 |
| 初始成员 | 可选；创建后也可在成员管理中添加 |

未在当前产品定义中确认的项目编号、描述、图标等字段不纳入必填规格，后续增加时需更新本文档。

### 4.3 创建结果

1. 系统再次校验用户创建资格，并读取用户所属最子部门。
2. 创建团队空间，并记录所属部门元数据。
3. 将创建人设为团队创建者，并同时赋予团队管理员和团队成员能力。
4. 初始化默认 Harness 和默认 Agent。
5. 记录创建和角色初始化审计。
6. 创建成功后进入新团队空间。

任何步骤失败时不得产生用户无法治理的半成品空间；页面保留用户输入并展示可重试原因。

## 5. 团队空间选择与切换

### 5.1 空间选择器

全局侧栏顶部展示当前团队空间。展开后显示用户有权访问的空间：

- 空间名称。
- 当前用户角色。
- 成员数量和任务数量摘要（有数据时）。

支持按空间名称搜索。无权访问的空间不出现在选择器中。

### 5.2 切换行为

- 切换后刷新左侧导航、任务记录和当前权限。
- 不复用上一个空间的筛选、选中任务或编辑草稿。
- 当前页面在目标空间有对应入口时进入同类入口；否则进入目标空间默认页。
- 存在未保存配置时，切换前要求确认。
- 权限已被撤销时，清理当前空间数据并返回可访问空间列表。

## 6. 成员清单

### 6.1 清单字段

成员列表展示：

- 姓名和账号。
- 角色。
- 加入时间。
- 最近更新时间。

团队创建者显示固定“创建者”标识。创建者能力包含管理员和成员能力，但不作为可手工分配的角色。

### 6.2 搜索与筛选

- 按姓名或账号搜索。
- 按角色筛选。
- 默认优先展示创建者，其余成员按加入时间排序。
- 空列表仅可能出现在加载异常；空间至少应包含创建者。

---

## 7. 添加成员

### 7.1 交互流程

1. 创建者或管理员点击“添加成员”。
2. 通过 DevUC 搜索可被添加的公司用户。
3. 选择一个或多个用户。
4. 为每个用户选择角色：团队管理员、团队成员或访客。
5. 提交前校验用户是否可被公司统一身份识别、是否已在空间和操作者当前权限。
6. 添加成功后刷新成员列表并记录审计。

### 7.2 反馈规则

- 已是成员的用户不可重复添加，并展示当前角色。
- 用户不属于可授权范围时不可选。
- 批量添加出现部分失败时，明确列出每个用户结果；成功项不得被重复创建。
- 搜索或 DevUC 不可用时保留已选用户并允许重试。

## 8. 角色调整与成员移除

### 8.1 角色调整

- 创建者和管理员可将非创建者成员调整为管理员、团队成员或访客。
- 同一用户在同一团队空间首期只有一个有效角色。
- 提交前展示新角色带来的关键权限变化。
- 后端保存成功后立即按新角色鉴权。
- 被调整用户已打开页面时，在刷新或下一次接口被拒绝后更新为最新权限。

### 8.2 创建者保护

首期不支持：

- 移除创建者。
- 降级创建者。
- 创建者退出。
- 移交创建者身份。

相关操作不展示，且后端必须拒绝直接请求。页面说明：“创建者由系统生成，当前版本不支持移除、降级或移交。”

### 8.3 移除成员

- 只能移除非创建者成员。
- 移除前展示该成员当前角色和仍在处理的任务。
- 确认后成员关系失效，后端立即拒绝后续空间访问和写操作。
- 移除不删除该成员历史操作、任务或审计记录。
- 操作失败时保留原成员关系。

成员存在未完成 Waiting Human 任务时不允许移除。

## 9. 权限矩阵

### 9.1 团队空间角色

| 权限 | 创建者 | 管理员 | 团队成员 | 访客 |
| --- | --- | --- | --- | --- |
| 进入团队空间 | 是 | 是 | 是 | 是 |
| 查看空间内容和授权历史详情 | 是 | 是 | 是 | 是 |
| 新建临时任务 | 是 | 是 | 是 | 否 |
| 使用已发布 Agent/AgentFlow | 是 | 是 | 是 | 否 |
| 处理被授权任务实例 | 是 | 是 | 是 | 否 |
| 配置事件任务和定时任务 | 是 | 是 | 否 | 否 |
| 配置和 Publish Harness | 是 | 是 | 否 | 否 |
| 管理成员与角色 | 是 | 是 | 否 | 否 |
| 添加管理员 | 是 | 是 | 否 | 否 |
| 只读查看配置 | 是 | 是 | 按授权 | 仅授权知识库内容 |

### 9.2 外部角色边界

系统节点管理员和部门节点管理员不因该身份自动进入团队空间或获得空间写权限。进入团队空间后的权限只由有效的团队空间角色决定。

### 9.3 权限执行

- 页面加载时获取当前用户角色和权限集合。
- 前端按权限隐藏、禁用或转为只读。
- 每次写请求由后端重新校验。
- DevUC 鉴权失败、角色缺失或权限数据异常时默认拒绝。
- 权限拒绝记录操作者、空间、动作、时间和原因。

## 10. 各模块的角色表现

### 10.1 团队成员

- 可以按授权只读查看 Harness、事件任务和定时任务配置。
- 可以新建临时任务并使用已发布 Agent/AgentFlow。
- 只能处理自己被授权的任务实例。
- 不展示成员管理和配置保存入口。

### 10.2 访客

- 页面顶部或内容区展示“当前角色仅支持查看”。
- 不展示新建任务、输入、审批、启动、关闭、重试、保存、启停、删除和 Publish 操作。
- 无权查看的任务详情不应通过直接 URL 暴露。

### 10.3 权限变化

写操作返回权限不足时：

1. 停止重复提交。
2. 刷新当前角色和权限集合。
3. 将页面切换为只读或跳转无权限页。
4. 保留不敏感的未提交内容，允许用户自行带走；不得继续自动提交。

## 11. 页面状态与反馈

- **空间列表为空**：用户有创建资格时提供创建入口；否则说明暂无可访问空间。
- **创建失败**：保留表单输入并展示创建资格、所属部门元数据或系统错误。
- **成员搜索为空**：提示未找到可添加用户。
- **成员添加失败**：保留已选用户和角色。
- **角色更新冲突**：刷新目标成员最新角色后要求用户重新确认。
- **DevUC 不可用**：禁止新的成员授权操作，已有成员清单保持可读。
- **无空间权限**：不加载空间业务数据，展示需要团队空间授权。
- **审计失败**：成员或角色写操作不得静默成功。

## 13. 首期范围与扩展

首期支持团队空间创建、所属部门元数据只读记录、空间切换、创建者保护、管理员/成员/访客三种可分配角色、成员增删改和统一后端鉴权。

后续可扩展创建者移交、自定义角色、团队角色模板、任务处理人转交和更细粒度的数据访问权限。
