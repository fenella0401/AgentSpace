# AgentSpace 产品简报

## 愿景

AgentSpace 是企业内部 AI 工作空间，帮助员工发现已批准智能体、执行受治理的 AI 工作流，并在明确归属、策略与可观测性前提下持续改进企业专属自动化能力。

更完整的产品总设计见 `docs/product/overall-design.md`。该设计将 AgentSpace 进一步定位为在线化地使用 Agent 完成项目工作的作业平台，覆盖多人/单人项目、project harness 在线版本管理、任务编排、Agent 会话追踪、审查门禁和企业治理。

## 目标用户

- 需要安全使用公司批准 AI 助手的员工。
- 希望复用 AI 工作流的业务团队（支持、运营、销售、工程、财务、HR、知识管理等）。
- 管理身份、权限、连接器、策略与审计日志的管理员。
- 负责 prompts、tools、evals、成本和模型策略的 AI 平台团队。
- 负责平台运行与审查的 SRE 与安全团队。

## 初始产品面

- 项目空间：支持单人或多人项目的成员、角色、任务、审查与运行状态管理。
- Project harness：在线编辑、校验、审查、发布、回滚和版本追溯。
- 任务编排：支持项目事件任务、定时任务和一次性下发任务。
- Agent 会话追踪：查看消息、计划、工具调用、技能调用、文件 diff、配置 diff、日志、成本与错误。
- 审查门禁：在权限提升、敏感变更、外部写入、评测失败等节点提醒人工审查。
- 管理控制台：覆盖 RBAC、策略、连接器、审计日志、模型配置与数据保留。
- 评测与运营看板：跟踪质量、使用量、延迟、成本、错误、安全事件与持续改进。

## MVP 原则

- 从一个高价值内部工作流与一个可复用平台模式起步。
- 每个智能体工作流都必须可审计且有明确 owner。
- 工具、连接器与数据访问默认最小权限。
- 通过评测验证结果质量后再扩大范围。
- 在功能开关后发布，并具备清晰反馈与故障处置流程。

## 当前已决策

- 首个事件源选择 GitHub。
- Harness 底层配置使用 YAML。
- Agent runtime 首版使用容器沙箱。
- Trace 原文保留一年，token 类高密信息默认脱敏显示。
- 审查通知首版仅使用站内通知。
- 任务结果是否直接写入由用户在 harness 中配置。
- 模型供应商兼容 OpenAI 和 Anthropic 协议模型，模型路由策略由用户在 harness 中配置。

## 剩余待决策

- Web 技术栈与托管平台。
- 身份提供方与 RBAC 单一事实源。
- GitHub 连接器首批事件范围、权限范围和审批流程。
- 容器沙箱的隔离级别、网络策略、资源限额和镜像治理。
- 外部写入策略的默认值、风险分级和强制审查条件。
- OpenAI/Anthropic 兼容模型的组织级 allowlist、预算上限和数据处理要求。
