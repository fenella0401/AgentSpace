# 产品定义

## 产品概述

- 产品名称：AgentSpace
- 产品定位：AgentSpace 是公司内部员工、团队和租户在线使用 Agent 的 Web SaaS 产品，面向租户治理、团队空间、知识上下文、Agent 运行配置、权限管理和协作管理提供统一入口。
- 目标用户：
  - 系统管理员：拥有全局管理权限，负责创建租户、授予租户管理员、维护全局团队模板、治理平台级能力与审计。
  - 租户管理员：负责本租户管理面内的租户配置、租户级团队模板和租户内团队空间清单治理。
  - 团队创建者：在所属租户内创建团队/项目空间，默认拥有该团队空间管理员权限，并负责空间初始治理。
  - 团队管理员：配置团队/项目空间，管理成员、角色、知识库和 Harness 配置。
  - 团队成员：在被授权的团队空间中使用 Agent，并消费空间内统一配置的知识上下文与运行能力，不具备 Harness 配置权限。
  - 访客：在被授权的团队空间中查看空间内容，不具备任何操作权限。

## 需求背景

- 公司内部员工和团队需要在统一 Web Portal 中使用 Agent，而不是依赖各自本地环境或分散配置。
- Agent 作业需要感知项目空间内的知识上下文，包括文档、代码仓文档和 Agent 相关配置。
- 团队空间需要支持从单人工作空间平滑扩展到多人协作空间，并兼容公司通用项目编号体系。
- 多租户 SaaS 场景需要全局和租户两级管理面：系统管理员负责租户开通与租户管理员授权，租户管理员负责本租户治理。
- Agent、Skill、MCP、Workflow 和环境变量等 Harness 配置既可能来自知识库中的配置文件，也需要支持团队管理员在 Portal 中在线配置。
- 多角色团队协作需要统一的 RBAC 权限管理机制，并对接公司 devuc 服务完成空间访问鉴权。
- 团队空间初始化需要逐步沉淀可复用模板，支持后续按全局和租户范围预置知识库文件夹体系、远程配置仓和 Agent 使用配置。

## 产品目标

- 提供系统管理员全局管理面，支持创建租户、维护租户状态并授予或撤销租户管理员。
- 提供租户管理员租户管理面，支持查看本租户团队空间清单、维护租户级配置和租户级团队模板。
- 将租户作为团队空间的必选上级域，确保团队空间、权限、模板和审计具备明确归属。
- 提供团队创建者初始化项目空间、配置成员角色和关联项目编号的标准入口。
- 提供基于 RBAC 的团队空间权限管理机制，明确创建者、管理员、团队成员和访客的访问与操作边界。
- 对接公司 devuc 服务进行鉴权，确保租户访问、团队空间访问和操作权限与公司统一身份能力衔接。
- 将项目空间知识库以文件系统形态呈现，支持多层级文件夹和文件夹级远程对象绑定。
- 自动识别知识库中的 Agent 相关配置文件夹，将配置解析并渲染到 Portal 的 Harness 配置中心。
- 支持团队管理员通过 Portal 从公司 AI 市场选择能力并在线配置，配置结果可回写到知识库配置文件夹。
- 支持系统管理员预置全局团队模板，租户管理员维护租户级团队模板，团队管理员初始化团队空间时可选择模板应用。
- 为后续扩展更多文件夹绑定对象、更多 Harness 配置类型和更细粒度租户治理能力预留模型与交互扩展点。

## 范围边界

**包含：**

- 全局管理面和租户管理面。
- 系统管理员创建租户、维护租户基础信息和状态。
- 系统管理员授予、撤销租户管理员。
- 租户管理员查看本租户团队空间清单，维护租户级配置和租户级团队模板。
- 团队/项目空间初始化，且团队空间必须归属一个租户。
- 基于 RBAC 的团队空间权限管理。
- 对接公司 devuc 服务进行租户和团队空间鉴权。
- 创建者、管理员、团队成员和访客四类团队空间基础角色。
- 创建者默认拥有团队空间管理员权限；创建者和管理员可额外添加团队管理员。
- 租户管理员不默认拥有团队空间内容写权限，除非被授予对应团队空间管理员角色。
- 访客可查看团队空间所有内容，但没有新增、修改、删除、配置、邀请等操作权限。
- 团队成员可使用团队空间内容与 Agent 能力，但没有 Harness 配置权限。
- 可选成员添加与角色授予；未添加成员时形成单人工作空间。
- 可选填写公司通用项目编号。
- 知识库根目录和多层级文件夹管理。
- 文件夹绑定远程代码仓，并自动关联远程代码仓文档。
- 扫描知识库顶层文件夹中的 Agent 相关配置文件夹，例如 `.codex`。
- 解析并展示 Agent、Skill、MCP、Workflow、环境变量配置。
- 创建者和团队管理员在 Portal 中在线配置 Harness，并将配置结果同步更新到知识库配置文件夹。
- 从公司 AI 市场选择 Agent、Skill、MCP 或 Workflow 等配置资源。
- 全局团队模板和租户级团队模板的需求池占位与初始设计。
- 团队管理员初始化团队空间时选择团队模板，自动初始化知识库文件夹体系并预置远程配置仓绑定的后续能力。

**不包含：**

- Agent 任务执行、调度和运行时编排的完整设计。
- 公司 AI 市场自身的商品发布、审核、计费或治理流程。
- 公司 devuc 服务自身的账号体系、组织关系、鉴权策略实现和主数据治理。
- 租户计费、套餐、容量配额和商业化治理。
- 租户间数据迁移、租户合并、租户拆分和历史未归属空间迁移实施方案。
- 角色审批流、跨租户权限继承、跨团队权限继承和字段级权限控制。
- 远程代码仓平台的账号体系、凭据申请和仓库权限审批流程。
- 项目编号体系本身的创建、校验规则和主数据治理。
- 创建者和团队管理员以外角色的在线 Harness 配置能力。
- 团队模板格式、版本、回滚、远程仓权限和模板冲突策略的最终细化方案。

## 核心场景

| 场景 | 用户 | 目标 | 说明 |
| --- | --- | --- | --- |
| 创建租户 | 系统管理员 | 为新的组织或业务单元开通 AgentSpace 租户 | 在全局管理面创建租户，维护基础信息和启停状态。 |
| 授予租户管理员 | 系统管理员 | 指定用户负责租户治理 | 系统管理员可授予或撤销租户管理员，变更需要审计。 |
| 管理租户 | 租户管理员 | 查看和维护本租户配置 | 在租户管理面查看团队空间清单，维护租户级配置和团队模板。 |
| 创建单人工作空间 | 团队创建者 | 在所属租户内快速初始化一个只包含自己的项目空间 | 不添加成员时，空间默认作为单人工作空间使用。 |
| 创建团队项目空间 | 团队创建者 | 在所属租户内初始化项目空间并邀请成员 | 可添加成员、授予角色，并可填写公司通用项目编号。 |
| 管理团队空间权限 | 团队创建者 / 团队管理员 | 基于 RBAC 管理空间成员与角色 | 对接 devuc 完成鉴权；创建者默认是团队管理员，创建者和团队管理员可添加管理员。 |
| 访客查看团队空间 | 访客 | 只读查看团队空间内容 | 可查看空间内所有内容，但不能进行任何操作。 |
| 配置知识库 | 团队管理员 | 建立 Agent 可感知的项目知识结构 | 通过文件系统形态管理文件夹，并支持文件夹绑定远程代码仓。 |
| 自动导入 Harness 配置 | 团队管理员 | 将知识库中的配置文件可视化呈现在 Portal | 扫描顶层文件夹中的 `.codex` 等配置目录，解析后展示到各配置中心。 |
| 在线维护 Harness 配置 | 团队管理员 | 在 Portal 中配置 Agent、Skill、MCP、Workflow 和环境变量 | 支持从公司 AI 市场选择配置资源，并回写知识库配置文件。 |
| 使用团队空间 Agent | 团队成员 | 基于空间统一知识库和 Harness 配置使用 Agent | 成员消费管理员配置的上下文与运行能力，但不能进行 Harness 配置。 |
| 预置全局团队模板 | 系统管理员 | 沉淀公司级团队空间初始化标准 | 后续版本支持维护全局模板，供租户和团队空间初始化复用。 |
| 应用团队模板 | 团队管理员 | 快速初始化团队空间知识库和 Agent 配置 | 后续版本支持选择全局或租户级模板，自动初始化目录、远程配置仓和前端渲染。 |

## 关键能力

- 全局管理面：支持系统管理员创建租户、维护租户状态、授予租户管理员和维护全局团队模板。
- 租户管理面：支持租户管理员查看本租户团队空间清单、维护租户级配置和租户级团队模板。
- 租户归属：团队空间必须绑定 `Tenant`，空间、成员、知识库、Harness 配置、模板应用和审计均继承租户上下文。
- 空间初始化：支持项目空间创建、项目编号填写、成员邀请与角色授予。
- RBAC 权限管理：支持系统管理员、租户管理员、团队空间创建者、团队管理员、团队成员和访客的分层权限边界。
- 公司统一鉴权：对接公司 devuc 服务完成身份与权限校验，租户管理、团队空间查看、配置、邀请和管理操作均需通过鉴权。
- 管理员治理：创建者默认具备团队空间管理员权限；创建者和团队管理员可继续添加团队管理员。
- 只读访问：访客可查看团队空间所有内容，但没有任何操作权。
- Harness 权限边界：Harness 配置仅面向创建者和团队管理员开放，团队成员不具备配置权限。
- 知识库管理：以文件系统形式呈现知识库，支持多层级文件夹、文件夹绑定远程代码仓和自动关联代码仓文档。
- 绑定对象扩展：文件夹绑定能力采用可扩展对象模型，后续可扩展到文档库、制品库、需求系统或其他公司内部知识源。
- 配置扫描与解析：扫描知识库顶层文件夹中的 Agent 相关配置目录，解析 Agent、Skill、MCP、Workflow 和环境变量。
- Harness 配置中心：提供 Agent 中心、Skill 中心、MCP 中心、Workflow 中心和环境变量中心。
- 双向同步：知识库配置文件变化可刷新 Portal 展示；Portal 在线配置可更新知识库配置文件夹内容。
- 市场选配：支持从公司 AI 市场选择可用配置资源，并沉淀到空间 Harness 配置。
- 团队模板：支持全局模板和租户级模板的后续设计，模板可包含知识库文件夹体系、远程配置仓绑定和 Agent 使用配置。

## 约束与风险

- 租户作为团队空间必选上级域后，历史未归属空间的迁移策略仍需确认。
- 系统管理员、租户管理员和团队管理员的权限边界需要在 UI、前端守卫、后端鉴权和审计中保持一致。
- 租户管理员查看团队空间清单与进入团队空间内容之间需要明确授权边界，避免越权读取或写入。
- 知识库配置文件可能随时更新，需要处理扫描延迟、解析失败、配置冲突和版本漂移。
- Portal 在线配置和配置文件直接变更之间存在双向同步冲突风险，需要明确配置源、更新时间和冲突处理策略。
- 远程代码仓绑定依赖仓库访问权限、连接稳定性和文档识别规则。
- 环境变量可能包含敏感信息，敏感值不应以明文形式写入知识库配置文件。
- 公司 AI 市场资源的版本、兼容性和下线策略会影响空间 Harness 配置稳定性。
- devuc 服务的可用性、接口稳定性和权限数据同步延迟会影响租户与团队空间访问控制体验。
- RBAC 策略需要与既有团队空间角色和 Harness 配置权限保持一致，避免历史成员权限漂移。
- 团队模板需求尚未完全清晰，模板格式、版本、回滚、远程仓权限和模板冲突策略需要在 R-0730 前继续明确。
- 项目编号格式、角色体系和远程仓库凭据接入方式仍需与公司既有体系确认。
