# 功能设计 UI 剧本

## 适用场景

当功能设计 PRD 更新后，需要判断 UI 是否受影响时，使用本剧本。

## 输入

- 已更新的功能设计文档
- 功能设计文档中的“PRD 设计”部分
- 现有“UI 设计”部分

## 依赖模板

- `docs/guides/templates/function-design.md`

## 执行步骤

1. 阅读 PRD 更新内容，识别是否涉及页面、入口、信息展示、交互或文案变化。
2. 若 UI 不受影响，在“UI 设计”部分记录“不涉及 UI 更新”及理由。
3. 若 UI 受影响，刷新页面入口、页面结构、关键状态、交互规则和文案说明。
4. 记录 UI 变更摘要，并标记是否会影响前端实现或验收标准。

## 需要询问用户的情况

- 无法判断目标用户界面应如何呈现。
- 存在多个合理 UI 方案，且选择会影响后续实现范围。
- PRD 要求与已有 UI 约束冲突。

## 完成标准

- 已明确 UI 是否需要刷新。
- 如需刷新，UI 设计已覆盖页面、状态和关键交互。
- 已记录对前端和验收标准的影响。
