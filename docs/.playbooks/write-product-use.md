# 产品使用手册撰写剧本

## 适用场景

当版本发布验证通过，且版本验收报告和部署检查清单完成会签后，使用本剧本更新产品使用手册。

## 输入

- `docs/global/product-use.md`
- 已会签的版本验收报告
- 已会签的部署检查清单
- 本版本发布公告和关联功能设计文档

## 依赖模板

- `docs/guides/templates/product-use.md`

## 执行步骤

1. 确认本次版本已完成验收报告和部署检查清单会签。
2. 阅读发布公告和关联功能设计文档，识别用户可见变化。
3. 刷新 `docs/global/product-use.md` 中的功能说明、操作步骤、注意事项和限制条件。
4. 保持用户视角表达，避免写入内部实现细节。
5. 记录手册变更摘要。

## 需要询问用户的情况

- 版本是否已完成会签无法确认。
- 用户侧操作路径无法从功能设计或发布资料判断。
- 某项限制条件是否应对用户公开无法判断。

## 完成标准

- 产品使用手册已反映本版本上线后的真实能力。
- 用户可按手册理解入口、步骤、结果和注意事项。
- 已记录本次手册更新摘要。
