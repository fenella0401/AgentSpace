# 功能设计验收剧本

## 适用场景

当功能设计 PRD 更新，或 UI、前端、后端设计发生变化时，使用本剧本刷新功能验收标准。

## 输入

- 已更新的功能设计文档
- “PRD 设计”“UI 设计”“前端设计”“后端设计”部分
- 现有“验收标准”部分

## 依赖模板

- `docs/guides/templates/function-design.md`
- `docs/guides/templates/test-case.md`

## 执行步骤

1. 汇总 PRD、UI、前端、后端中的本次变更。
2. 刷新功能设计文档中的“验收标准”部分。
3. 覆盖正常路径、关键边界、异常场景和权限场景。
4. 如需要独立测试用例，按模板创建或更新 `docs/test-case/TC-xxx-name.md`。
5. 记录验收标准变更摘要，供版本发布验证使用。

## 需要询问用户的情况

- 无法判断某项规则是否必须验收。
- 验收标准与 PRD 或实现设计存在冲突。
- 是否需要独立测试用例无法判断。

## 完成标准

- 验收标准已与当前设计一致。
- 验收标准覆盖正常路径、边界、异常和权限。
- 如创建独立测试用例，测试用例编号和关联功能清晰。
