# 功能设计验收剧本

## 适用场景

当功能设计 PRD 更新后，使用本剧本刷新功能验收标准。

## 输入

- 已更新的功能设计文档
- “PRD 设计”部分
- 现有“验收标准”部分

## 依赖模板

- `docs/guides/templates/function-design.md`
- `docs/guides/templates/test-case.md`

## 执行步骤

1. 只基于“PRD 设计”中的用户目标、Use Story、Use Case、业务规则、边界与异常汇总验收点。
2. 刷新功能设计文档中的“验收标准”部分。
3. 覆盖正常路径、关键边界、异常场景和权限场景。
4. 如需要独立测试用例，按模板创建或更新 `docs/test-case/TC-xxx-name.md`。
5. 记录验收标准变更摘要，供版本发布验证使用。

## 规范要求

- 验收标准只能依赖 PRD 设计，不得依赖 UI、前端或后端设计。
- UI、前端或后端设计发生变化时，不得直接触发验收标准修改；如变化影响验收口径，必须先更新 PRD 设计，再按本剧本刷新验收标准。
- UI、前端或后端设计与验收标准存在冲突时，以 PRD 设计为准；若 PRD 设计未覆盖该冲突点，应先补充 PRD 设计。
- 每次生成或更新 PRD 设计后，需要单独使用一个 agent 对 PRD 设计进行 review。
- PRD review 需要检查：业务目标是否合理，Use Story 与 Use Case 是否完整一致，业务规则与边界异常是否自洽，是否存在自相矛盾，表达是否重复啰嗦。
- PRD review 发现问题时，应先修正 PRD 设计，再生成或刷新验收标准。

## 需要询问用户的情况

- 无法仅根据 PRD 判断某项规则是否必须验收。
- 验收标准与 PRD 设计存在冲突。
- 是否需要独立测试用例无法判断。

## 完成标准

- 验收标准已与 PRD 设计一致。
- 验收标准覆盖正常路径、边界、异常和权限。
- PRD 设计已完成独立 agent review，且 review 问题已关闭或明确记录为待确认。
- 如创建独立测试用例，测试用例编号和关联功能清晰。
