# 功能设计前端剧本

## 适用场景

当功能设计 PRD 更新后，需要判断前端设计是否受影响时，使用本剧本。

## 输入

- 已更新的功能设计文档
- “PRD 设计”和“UI 设计”部分
- 现有“前端设计”部分

## 依赖模板

- `docs/guides/templates/function-design.md`

## 执行步骤

1. 阅读 PRD 和 UI 更新内容，识别前端页面、组件、状态、路由、权限或接口依赖是否变化。
2. 若前端不受影响，在“前端设计”部分记录“不涉及前端更新”及理由。
3. 若前端受影响，刷新前端页面结构、组件职责、状态流转、接口调用和异常处理。
4. 记录前端变更摘要，并标记是否会影响后端设计或验收标准。

## 需要询问用户的情况

- 无法确定前端交互或状态处理规则。
- PRD、UI 与现有前端能力存在冲突。
- 接口依赖无法从现有设计中判断。

## 完成标准

- 已明确前端是否需要刷新。
- 如需刷新，前端设计已覆盖页面、组件、状态、接口和异常处理。
- 已记录对后端和验收标准的影响。
