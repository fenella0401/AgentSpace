# 功能设计后端剧本

## 适用场景

当功能设计 PRD 更新后，需要判断后端设计是否受影响时，使用本剧本。

## 输入

- 已更新的功能设计文档
- “PRD 设计”和“前端设计”部分
- 现有“后端设计”部分

## 依赖模板

- `docs/guides/templates/function-design.md`

## 执行步骤

1. 阅读 PRD 和前端更新内容，识别接口、数据、权限、业务规则或任务流程是否变化。
2. 若后端不受影响，在“后端设计”部分记录“不涉及后端更新”及理由。
3. 若后端受影响，刷新接口设计、数据设计、业务规则、权限控制和异常处理。
4. 记录后端变更摘要，并标记是否会影响前端设计或验收标准。

## 需要询问用户的情况

- 业务规则、数据归属或权限边界无法判断。
- 存在多个后端实现方案，且会影响接口或数据结构。
- PRD 与现有系统架构约束冲突。

## 完成标准

- 已明确后端是否需要刷新。
- 如需刷新，后端设计已覆盖接口、数据、规则、权限和异常处理。
- 已记录对前端和验收标准的影响。
