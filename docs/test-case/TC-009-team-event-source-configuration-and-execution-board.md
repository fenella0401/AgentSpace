# TC-009-团队空间事件源配置与事件执行看板

## 基本信息

| 字段 | 内容 |
| --- | --- |
| 测试用例编号 | TC-009 |
| 关联功能 | F-009 |
| 关联版本 | R-0630 |
| 优先级 | 高 |
| 测试范围 | eDevOps FE 事件源配置、同步、去重、触发、并发、事件执行看板和原子详情跳转 |

## 前置条件

- 准备团队创建者、团队管理员、团队成员和访客账号。
- 租户已具备 eDevOps 授权引用。
- 准备 eDevOps FE 样例，覆盖新建 FE、FE 状态变更、字段变化、缺失必填字段和重复同步。
- 准备已发布 Agent、AgentFlow、知识域、Skill、Tool 和环境变量样例。
- 准备 Agent 会话和 AgentFlow 运行成功、waiting approval、失败和完成状态样例。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 团队管理员创建 eDevOps FE 事件源 | 保存授权引用、项目范围、同步范围和启停状态，不保存凭据明文 |
| 2 | 执行试同步 | 返回脱敏后的样例 FE，展示 FE ID、标题、状态、优先级和链接 |
| 3 | 配置新建 FE 触发规则，目标为 Agent 会话 | 规则保存成功，包含同步时间、过滤条件、去重键、输入映射、知识域和并发上限 |
| 4 | 配置 FE 状态变更触发规则，目标为 AgentFlowTask | 规则保存成功，并校验 AgentFlow 输入 Schema 和依赖 |
| 5 | 到达同步时间拉取新 FE | 系统创建 EventExecutionTask，初始状态为 backlog |
| 6 | 重复同步同一个 FE | 不重复创建 EventExecutionTask，只更新同步水位和摘要 |
| 7 | FE 字段变化后再次同步 | 更新看板摘要，不覆盖已启动原子执行对象快照 |
| 8 | 并发额度允许且目标为 Agent | 创建 `source=event_source` 的 AgentSession，并将事件任务映射为 agent working |
| 9 | 并发额度允许且目标为 AgentFlow | 创建来源为事件源的 AgentFlowTask 和 AgentFlowRun，并将事件任务映射为 agent working |
| 10 | 当前运行数量达到并发上限 | 新事件任务保持 backlog，不创建原子执行对象 |
| 11 | 并发上限配置为 0 | 视为暂停触发，事件任务保持 backlog |
| 12 | 暂停事件源或事件类型规则 | 不再同步或触发新任务，已运行对象继续执行 |
| 13 | 原子对象进入等待审阅、确认或 Agent 询问 | 看板状态映射为 waiting approval |
| 14 | 原子对象完成 | 看板状态映射为 done |
| 15 | 原子对象失败 | 看板展示失败原因并保留审计信息 |
| 16 | 在看板按事件源、事件类型、触发目标和状态筛选 | 返回符合条件的事件任务 |
| 17 | 点击 Agent 会话事件卡片 | 跳转 F-006 Agent 会话详情，并展示 eDevOps FE 来源摘要 |
| 18 | 点击 AgentFlow 事件卡片 | 跳转 F-008 AgentFlow 任务详情，并展示事件来源摘要 |
| 19 | eDevOps 授权失效或系统不可达 | 同步失败，保留已有 backlog 和错误信息 |
| 20 | FE 缺少必需字段、去重键为空或输入映射失败 | 事件进入同步异常记录，不触发原子执行对象 |
| 21 | 目标 Agent、AgentFlow、知识域或依赖失效 | 事件任务留在 backlog 并展示依赖失效原因 |
| 22 | payload 包含非白名单字段或敏感字段 | prompt、日志和看板摘要均不包含敏感内容 |
| 23 | 团队成员查看事件执行看板 | 可查看有权限的事件摘要和状态，但不能修改事件源配置 |
| 24 | 访客查看事件执行看板 | 只读展示，不出现配置、暂停或重试类入口 |
| 25 | 无权用户访问原子详情 | 不泄露 Agent 输出、AgentFlow 节点或 eDevOps 敏感字段 |
