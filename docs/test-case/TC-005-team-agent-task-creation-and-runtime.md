# TC-005-团队空间 Agent 任务创建与运行

## 基本信息

| 字段 | 内容 |
| --- | --- |
| 测试用例编号 | TC-005 |
| 关联功能 | F-005 |
| 关联版本 | R-0630 / R-0730 |
| 优先级 | 高 |
| 测试范围 | R-0630 任务、Workflow 与 Hook 真实运行；R-0730 输入增强 |

## 前置条件

- 准备创建者、管理员、团队成员和访客账号。
- 准备零配置空间、已发布自定义 Agent 的空间和已发布 Workflow/Hook 的空间。
- 准备包含产品 Agent、开发 Agent、运营 Agent、条件、人工确认、并行、子 Workflow 和输出节点的 Workflow。
- 准备任务开始、Tool 调用前后、对象写入前后、任务完成和失败 Hook。
- 外部 agent core 支持任务、Agent 节点、询问、Agent 内部 Tool 调用前置事件和错误模拟。
- 市场 Tool 测试环境支持成功、失败和重复幂等键场景。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 在零配置空间输入首条指令创建任务 | 系统使用平台默认 Agent 和空扩展快照，任务进入运行中 |
| 2 | 空间发布多个 Agent 但不指定默认 Agent，创建任务时不选择执行对象 | 系统仍使用平台默认 Agent |
| 3 | 指定已发布 Agent，并通过 R-0630 知识域选择器选择一个知识域创建任务 | 快照固定该 Agent、`knowledgeDomainIds` 及相关 AGENT.md、Skill、Tool、Hook 和环境变量引用，不加载无关知识域 |
| 4 | 检查对话名称生成和手动修改 | 自动名称符合 4-10 个字；创建人可以修改 |
| 5 | 名称生成失败 | 系统使用首条指令前 8 个字或“新建对话”，不阻塞任务 |
| 6 | 修改空间 Harness 后继续历史对话 | 新轮次使用新快照，历史轮次保持原快照 |
| 7 | agent core 返回选项型和开放式询问 | 任务进入待输入，创建人回答后恢复运行 |
| 8 | 访客创建任务，或非创建人继续输入、回答询问 | 后端拒绝请求 |
| 9 | 在专用选择器选择已发布 Workflow 并填写输入 | 系统创建 AgentTask、HarnessSnapshot 和 WorkflowRun |
| 10 | 依次执行产品 Agent、开发 Agent 和运营 Agent 节点 | Workflow Engine 调用 agent core，保存各 Agent 节点输入输出和状态 |
| 11 | Agent 节点执行过程中调用 Skill 和 Tool | Skill/Tool 不作为用户侧 Workflow 节点；Tool 调用前完成 Hook 决策，调用携带幂等键并保存结果 |
| 12 | 执行条件节点 | Engine 按结构化表达式选择唯一分支，不调用模型决定流程 |
| 13 | 执行并行节点 | 各分支独立运行，并按配置汇总成功、快速失败或部分成功 |
| 14 | 执行人工确认节点 | WorkflowRun 进入等待确认，任务映射为待输入 |
| 15 | 创建人批准、拒绝和重复提交同一确认幂等键 | 流程进入对应分支；重复提交不重复推进 |
| 16 | 执行子 Workflow | 创建关联 Run，调用链深度正确增加 |
| 17 | 取消运行中的 Workflow | 不再调度新节点，Run 为已取消，已完成副作用不被静默撤销 |
| 18 | 对可重试失败 Agent 节点发起重试 | 使用相同业务幂等策略，不产生重复外部副作用 |
| 19 | 修改已发布 Workflow 后查看正在运行的 Run | 当前 Run 继续使用启动时版本 |
| 20 | 任务开始 Hook 返回允许 | 主任务正常启动，HookExecution 记录允许 |
| 21 | 任务开始 Hook 返回阻止 | 不调用 agent core 或 Workflow Engine，任务展示阻止原因 |
| 22 | Tool 调用前 Hook 要求人工确认 | Tool 暂不执行，任务待输入，确认后按决定继续 |
| 23 | 对象写入前 Hook 阻止 | agent core 不执行写入，记录 Hook 决定 |
| 24 | Tool 调用后、任务完成和失败 Hook 执行 | Hook 异步运行，通知、审计、Skill 或 Workflow 结果被保存 |
| 25 | 异步 Hook 执行失败 | 主动作结果不回滚，HookExecution 标记失败 |
| 26 | 同一 Hook 在同一调用链再次触发 | 系统跳过或拒绝重入并记录审计 |
| 27 | 构造 Hook、Workflow 和子 Workflow 组合调用超过 5 层 | 系统终止新调用并保存完整调用链 |
| 28 | 高风险 Tool 的前置 Hook 超时且无明确允许 | 系统不执行 Tool，并按失败策略记录 |
| 29 | Workflow Agent 节点、agent core 或事件流失败 | 保存节点和任务错误状态，服务恢复后记录可查询 |
| 30 | Harness 快照生成失败 | 不调用外部执行服务，保留内部失败记录 |
| 31 | 重启 Workflow Engine 后恢复未完成 Run | 根据持久化状态继续可运行节点，不重复已完成节点 |
| 32 | 团队成员使用已发布 Workflow，Workflow 内部尝试越权 Tool | 后端按当前用户权限拒绝，Workflow 不能借配置越权 |
| 33 | Hook 确认重复提交、超时后迟到提交，并在服务重启后恢复 | 统一确认记录保持单一终态，不重复执行主动作，超时按 Hook 失败策略处理 |
| 34 | 提交不存在、跨空间或无权访问的 `knowledgeDomainIds` | 系统阻止任务或 Workflow 启动并定位无效知识域 |

## R-0730 输入增强

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 35 | 在新建、继续输入或开放式回答中输入 `/` 并选择功能 | 结构化功能引用进入本轮快照 |
| 36 | 输入 `@` 并选择当前空间知识文件 | 文件 ID、路径和版本进入本轮快照，所属知识域合并进 `knowledgeDomainIds` |
| 37 | 取消 `/` 或 `@` 选择后提交普通文本 | 系统按普通文本提交 |
| 38 | 仅选择功能或文件但没有文本 | 系统阻止提交 |
| 39 | 提交已停用功能、跨空间文件、无权限文件或超过限制的文件 | 系统拒绝并要求重新选择 |
