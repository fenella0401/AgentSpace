# TC-005-团队空间 Agent 任务创建与运行

## 基本信息

| 字段 | 内容 |
| --- | --- |
| 测试用例编号 | TC-005 |
| 关联功能 | F-005 |
| 关联版本 | R-0630 |
| 优先级 | 高 |

## 前置条件

- 测试账号具备团队空间创建者、管理员、团队成员和访客四类角色样例。
- 测试团队空间已归属有效租户，且可通过 devuc 完成空间访问鉴权。
- 测试团队空间已完成 Harness 配置，至少包含一个可传递给外部 agent core 的 Agent 配置和一个可选择的 Agent 功能。
- 测试团队空间 Harness 知识空间中已准备可读文件、无权限文件、已删除或版本失效文件样例。
- 外部 agent core 测试环境可用，支持任务创建、Agent 功能选择、知识文件引用、询问事件、回答回传和错误场景模拟。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 团队成员进入团队空间，打开新建 Agent 对话主界面 | 页面展示支持结构化选择的指令输入框、提交按钮和取消入口 |
| 2 | 在指令输入框输入 `/` 并输入关键字 | 页面唤起 Agent 功能选择浮层，展示当前团队空间 Harness 中可用的 Agent 功能并支持筛选 |
| 3 | 选择一个 Agent 功能后继续输入指令 | 输入框展示已选功能标识，前端保存结构化功能 ID，用户可移除已选功能 |
| 4 | 在指令输入框输入 `@` 并搜索 Harness 知识空间文件 | 页面唤起文件选择浮层，仅展示当前团队空间 Harness 知识空间中用户可读文件 |
| 5 | 选择一个或多个可读文件并提交有效指令 | 系统创建 Agent 任务，记录创建人、团队空间、租户、初始状态、功能选择和文件引用 |
| 6 | 检查后端调用外部 agent core 的请求 | 请求包含用户指令、团队空间标识、创建人标识、指令级 Harness 配置快照、Agent 功能选择和知识文件引用 |
| 7 | 查看新建任务的对话名称 | 系统调用模型总结首次输入指令生成 4-10 个字的对话名称 |
| 8 | 模拟对话名称生成失败，首条指令不少于 8 个字 | 系统使用首条指令前 8 个字作为兜底名称，任务创建不被阻塞 |
| 9 | 模拟对话名称生成失败，首条指令少于 4 个字 | 系统使用“新建对话”作为兜底名称，任务创建不被阻塞 |
| 10 | 创建人修改 4-10 个字的对话名称并保存 | 对话名称保存成功，任务详情和任务清单展示新名称 |
| 11 | 创建人提交少于 4 个字或超过 10 个字的对话名称 | 系统阻止保存，并提示名称长度要求 |
| 12 | 创建任务后修改团队空间 Harness 配置、Agent 功能目录或知识文件版本 | 已提交首轮指令仍使用创建时的 Harness 快照、功能选择和文件版本引用 |
| 13 | 创建人进入历史对话，使用新的 `/` 功能选择和 `@` 文件引用继续输入 | 系统创建新轮次和新的 Harness 快照，并将新指令、历史上下文、功能选择和文件引用传递给外部 agent core |
| 14 | 任务已建立外部 conversation ID，并进入完成静止态或异常静止态后再次进入历史对话 | 历史对话仍被保留，创建人仍可继续输入新指令 |
| 15 | 外部 agent core 返回选项型询问事件 | 任务状态变为待输入，创建人看到选项回答控件 |
| 16 | 创建人选择选项并提交回答 | AgentSpace 将回答回传外部 agent core，任务状态重新变为运行中 |
| 17 | 外部 agent core 返回开放式询问事件 | 任务状态变为待输入，创建人可以填写支持 `/` 和 `@` 的文本回答 |
| 18 | 创建人在开放式回答中填写文本、选择 Agent 功能和知识文件后提交 | AgentSpace 校验回答长度、功能和文件引用，生成回答级 Harness 快照，并将回答、功能选择和文件引用回传外部 agent core |
| 19 | 访客尝试新建任务 | 系统拒绝操作，并展示只读或权限不足提示 |
| 20 | 非任务创建人尝试在历史对话中继续输入 | 后端返回权限不足，不创建新轮次 |
| 21 | 非任务创建人尝试修改对话名称或回答询问 | 后端返回权限不足，任务状态和名称不被错误更新 |
| 22 | 输入 `/` 后关闭功能选择浮层并提交有效普通文本 | 系统不生成结构化功能选择，普通文本任务可以正常创建或继续 |
| 23 | 输入 `@` 后关闭文件选择浮层并提交有效普通文本 | 系统不生成结构化文件引用，普通文本任务可以正常创建或继续 |
| 24 | 提交空指令 | 系统阻止创建任务或继续输入，并提示输入指令 |
| 25 | 选择 Agent 功能后在提交前模拟该功能停用或不属于当前 Harness 配置 | 后端拒绝本次创建、继续输入或开放式回答，保留用户草稿并提示重新选择功能 |
| 26 | 选择知识文件后在提交前模拟文件删除、版本失效或用户失去读取权限 | 后端拒绝本次创建、继续输入或开放式回答，保留用户草稿并提示重新选择文件 |
| 27 | 尝试通过 `@` 选择其他团队空间文件或无权限文件 | 前端不展示该文件；若绕过前端提交，后端返回权限不足或校验错误 |
| 28 | 模拟创建任务或继续输入时指令级 Harness 快照生成失败 | 系统不调用外部 agent core，保留内部失败记录，任务进入异常静止态，并提示创建人修正后重试 |
| 29 | 模拟外部 agent core 创建任务失败或认证失败，且未返回 conversation ID | 系统保留失败任务记录，任务状态为异常静止态，记录错误事件，且不允许按历史会话继续输入 |
| 30 | 模拟外部 agent core 继续对话失败 | 系统保留本轮失败记录和错误事件，任务状态为异常静止态 |
| 31 | 模拟外部 agent core 事件流中断 | 系统记录错误事件，任务状态变为异常静止态 |
| 32 | 任务运行中或待输入状态下，创建人绕过前端提交继续输入 | 后端拒绝请求，并提示需等待任务进入静止态或先回答当前询问 |
| 33 | 开放式询问回答仅选择 Agent 功能或知识文件但不填写文本 | 系统阻止回答提交，并提示填写回答内容 |
| 34 | 选项型询问回答携带 Agent 功能选择或知识文件引用 | 后端拒绝回答提交，并提示该询问只能提交选项 |
| 35 | 一次 `@` 引用超过 10 个文件或超过大小 / 上下文预算 | 系统拒绝本次输入，并提示减少文件引用 |
| 36 | 回答提交成功后，使用相同回答幂等键重复提交 | 系统返回已有回答结果，不重复回传外部 agent core |
| 37 | 提交超过 agent core 限制或默认 4000 个字的开放式询问回答 | 系统阻止回答提交，并提示回答长度超限 |
| 38 | 模拟开放式回答级 Harness 快照生成失败 | 系统不回传外部 agent core，任务保持待输入，记录回答失败事件，并允许创建人修正后以同一幂等键重试 |
