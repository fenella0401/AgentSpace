# TC-005-团队空间 Agent 任务创建与运行

## 基本信息

| 字段 | 内容 |
| --- | --- |
| 测试用例编号 | TC-005 |
| 关联功能 | F-005 |
| 关联版本 | R-0630 |
| 优先级 | 高 |
| 测试范围 | R-0630 核心闭环；不包含 US-005-007、US-005-008 |

## 前置条件

- 测试账号具备团队空间创建者、管理员、团队成员和访客四类角色样例。
- 测试团队空间已归属有效租户，且可通过 devuc 完成空间访问鉴权。
- 测试团队空间已完成 Harness 配置，至少包含一个可传递给外部 agent core 的 Agent 配置。
- 外部 agent core 测试环境可用，支持任务创建、继续对话、询问事件、回答回传和错误场景模拟。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 团队成员进入团队空间，打开新建 Agent 对话主界面 | 页面展示指令输入框、提交按钮和取消入口 |
| 2 | 输入有效首条指令并提交 | 系统创建 Agent 任务，记录创建人、团队空间、租户和初始运行状态 |
| 3 | 检查后端调用外部 agent core 的请求 | 请求包含用户指令、团队空间标识、创建人标识、任务标识和指令级 Harness 配置快照 |
| 4 | 查看新建任务的对话名称 | 系统调用模型总结首次输入指令生成 4-10 个字的对话名称 |
| 5 | 模拟对话名称生成失败，首条指令不少于 8 个字 | 系统使用首条指令前 8 个字作为兜底名称，任务创建不被阻塞 |
| 6 | 模拟对话名称生成失败，首条指令少于 4 个字 | 系统使用“新建对话”作为兜底名称，任务创建不被阻塞 |
| 7 | 创建人修改 4-10 个字的对话名称并保存 | 对话名称保存成功，任务详情和任务清单展示新名称 |
| 8 | 创建人提交少于 4 个字或超过 10 个字的对话名称 | 系统阻止保存，并提示名称长度要求 |
| 9 | 创建任务后修改团队空间 Harness 配置 | 已提交首轮指令仍使用创建时的 Harness 快照 |
| 10 | 创建人进入历史对话并提交新的有效指令 | 系统创建新轮次和新的 Harness 快照，并将新指令、历史上下文和任务标识传递给外部 agent core |
| 11 | 任务已建立外部 conversation ID，并进入完成静止态或异常静止态后再次进入历史对话 | 历史对话仍被保留，创建人仍可继续输入新指令 |
| 12 | 外部 agent core 返回选项型询问事件 | 任务状态变为待输入，创建人看到选项回答控件 |
| 13 | 创建人选择选项并提交回答 | AgentSpace 将回答回传外部 agent core，任务状态重新变为运行中 |
| 14 | 外部 agent core 返回开放式询问事件 | 任务状态变为待输入，创建人可以填写文本回答 |
| 15 | 创建人在开放式回答中填写有效文本并提交 | AgentSpace 校验回答长度，生成回答级 Harness 快照，并将回答和快照回传外部 agent core |
| 16 | 访客尝试新建任务 | 系统拒绝操作，并展示只读或权限不足提示 |
| 17 | 非任务创建人尝试在历史对话中继续输入 | 后端返回权限不足，不创建新轮次 |
| 18 | 非任务创建人尝试修改对话名称或回答询问 | 后端返回权限不足，任务状态和名称不被错误更新 |
| 19 | 提交空指令 | 系统阻止创建任务或继续输入，并提示输入指令 |
| 20 | 模拟创建任务或继续输入时指令级 Harness 快照生成失败 | 系统不调用外部 agent core，保留内部失败记录，任务进入异常静止态，并提示创建人修正后重试 |
| 21 | 模拟外部 agent core 创建任务失败或认证失败，且未返回 conversation ID | 系统保留失败任务记录，任务状态为异常静止态，记录错误事件，且不允许按历史会话继续输入 |
| 22 | 模拟外部 agent core 继续对话失败 | 系统保留本轮失败记录和错误事件，任务状态为异常静止态 |
| 23 | 模拟外部 agent core 事件流中断 | 系统记录错误事件，任务状态变为异常静止态 |
| 24 | 任务运行中或待输入状态下，创建人绕过前端提交继续输入 | 后端拒绝请求，并提示需等待任务进入静止态或先回答当前询问 |
| 25 | 提交空开放式询问回答 | 系统阻止回答提交，并提示填写回答内容 |
| 26 | 回答提交成功后，使用相同回答幂等键重复提交 | 系统返回已有回答结果，不重复回传外部 agent core |
| 27 | 提交超过 agent core 限制或默认 4000 个字的开放式询问回答 | 系统阻止回答提交，并提示回答长度超限 |
| 28 | 模拟开放式回答级 Harness 快照生成失败 | 系统不回传外部 agent core，任务保持待输入，记录回答失败事件，并允许创建人修正后以同一幂等键重试 |
| 29 | 模拟 agent core 返回未知询问格式 | 系统记录错误事件，任务进入异常静止态 |
