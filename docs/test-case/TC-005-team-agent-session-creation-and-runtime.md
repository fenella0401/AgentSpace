## 前置条件

- 准备创建者、管理员、团队成员、访客账号。
- 准备零配置空间和已发布在线 Agent 的空间。
- 准备已发布市场 Skill、市场 Tool 和环境变量样例。
- 外部 agent core 支持手工会话创建、继续对话、询问、事件流和错误模拟。
- 准备已发布 AgentFlow 供开始作业入口切换测试。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 在零配置空间输入首条指令创建手工 Agent 会话 | 系统创建 `source=manual` 的 AgentSession，使用平台默认 Agent 和空扩展快照，会话进入运行中 |
| 2 | 空间发布多个 Agent 但不指定默认 Agent，创建会话时不选择 Agent | 系统仍使用平台默认 Agent |
| 3 | 指定已发布 Agent，并通过 R-0630 知识域选择器选择一个知识域创建会话 | 快照固定该 Agent、`knowledgeDomainIds` 及相关 AGENT.md、Skill、Tool 和环境变量引用，不加载无关知识域 |
| 4 | 在同一开始作业入口切换目标为 AgentFlow 并选择已发布 AgentFlow | 前端调用 F-008 创建 AgentFlowTask，不创建 `source=manual` 的 AgentSession，并跳转 AgentFlow 任务详情 |
| 5 | 开始作业入口目标为 AgentFlow 但没有可启动 AgentFlow | 展示空态或禁用提交，不影响切回 Agent 创建手工会话 |
| 6 | 继续本人创建且已静止的手工 Agent 会话 | 系统创建新轮次和新快照，并调用 agent core 继续对话 |
| 7 | 修改空间 Harness 后继续历史手工会话 | 新轮次使用新快照，历史轮次保持原快照 |
| 8 | Agent 在手工会话中发起询问 | 会话进入待输入，询问卡片展示问题和选项 |
| 9 | 创建人回答询问 | 后端保存回答并回传 agent core，会话恢复运行 |
| 10 | 非创建人或访客回答询问 | 后端拒绝请求 |
| 11 | 尝试继续或改名 AgentFlow Agent 任务会话 | 后端拒绝，并提示回到 AgentFlow 任务页处理操作 |
| 12 | agent core 创建会话失败 | 会话进入异常静止态并保存错误原因 |
| 13 | agent core 事件流中断 | 已保存事件不丢失，详情页可查询最近状态 |
| 14 | Agent 内部 Skill 或 Tool 调用失败 | 保存结构化错误事件并按 Agent 输出策略处理 |
| 15 | 团队成员使用已发布 Agent，Agent 内部尝试越权 Tool | 后端按当前用户权限拒绝，配置不能借会话越权 |
| 16 | 询问重复提交、超时后迟到提交，并在服务重启后恢复 | 统一幂等记录保持单一终态，不重复恢复执行 |
| 17 | 访客在开始作业入口提交 Agent 或 AgentFlow 目标 | 后端拒绝请求 |
| 18 | R-0730 `/` 功能选择和 `@` 文件引用测试点 | 本版本不执行 |
