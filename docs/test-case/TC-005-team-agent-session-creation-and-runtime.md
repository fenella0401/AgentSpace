# TC-005-团队空间 Agent 会话创建与运行

## 基本信息

| 字段 | 内容 |
| --- | --- |
| 测试用例编号 | TC-005 |
| 关联功能 | F-005 |
| 关联版本 | R-0630 / R-0730 |
| 优先级 | 高 |
| 测试范围 | R-0630 统一开始作业入口、手工 Agent 会话创建、继续、询问、Hook 真实运行；R-0730 输入增强 |

## 前置条件

- 准备创建者、管理员、团队成员和访客账号。
- 准备零配置空间、已发布自定义 Agent 的空间和已发布 Hook 的空间。
- 准备任务开始、Tool 调用前后、对象写入前后、会话完成和失败 Hook。
- 外部 agent core 支持手工会话创建、继续对话、询问、Agent 内部 Tool 调用前置事件和错误模拟。
- 市场 Tool 测试环境支持成功、失败和重复幂等键场景。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 在零配置空间输入首条指令创建手工 Agent 会话 | 系统创建 `source=manual` 的 AgentSession，使用平台默认 Agent 和空扩展快照，会话进入运行中 |
| 2 | 空间发布多个 Agent 但不指定默认 Agent，创建会话时不选择 Agent | 系统仍使用平台默认 Agent |
| 3 | 指定已发布 Agent，并通过 R-0630 知识域选择器选择一个知识域创建会话 | 快照固定该 Agent、`knowledgeDomainIds` 及相关 AGENT.md、Skill、Tool、Hook 和环境变量引用，不加载无关知识域 |
| 4 | 在同一开始作业入口切换目标为 Harness CICD 流水线并选择已发布流水线 | 前端调用 F-008 创建 HarnessPipelineTask，不创建 `source=manual` 的 AgentSession，并跳转流水线任务详情 |
| 5 | 开始作业入口目标为 Harness CICD 流水线但没有可启动流水线 | 展示空态或禁用提交，不影响切回 Agent 创建手工会话 |
| 6 | 检查会话名称生成和手动修改 | 自动名称符合 4-10 个字；创建人可以修改 |
| 7 | 名称生成失败 | 系统使用首条指令前 8 个字或“新建会话”，不阻塞会话 |
| 8 | 修改空间 Harness 后继续历史手工会话 | 新轮次使用新快照，历史轮次保持原快照 |
| 9 | agent core 返回选项型和开放式询问 | 会话进入待输入，创建人回答后恢复运行 |
| 10 | 访客提交开始作业，或非创建人继续输入、回答询问、修改名称 | 后端拒绝请求 |
| 11 | 尝试通过 F-005 接口创建 `source=harness_pipeline_node` 会话 | 后端拒绝；该来源只能由 F-008 创建 |
| 12 | 尝试继续或改名 Harness CICD 流水线 Agent 任务会话 | 后端拒绝，并提示回到 Harness CICD 流水线任务页处理操作 |
| 13 | 会话开始 Hook 返回允许 | 主会话正常启动，HookExecution 记录允许 |
| 14 | 会话开始 Hook 返回阻止 | 不调用 agent core，会话展示阻止原因 |
| 15 | Tool 调用前 Hook 要求人工确认 | Tool 暂不执行，会话待输入，确认后按决定继续 |
| 16 | 对象写入前 Hook 阻止 | agent core 不执行写入，记录 Hook 决定 |
| 17 | Tool 调用后、会话完成和失败 Hook 执行 | Hook 异步运行，通知、审计或 Skill 结果被保存 |
| 18 | 异步 Hook 执行失败 | 主动作结果不回滚，HookExecution 标记失败 |
| 19 | 同一 Hook 在同一调用链再次触发 | 系统跳过或拒绝重入并记录审计 |
| 20 | 构造 Hook 调用超过 5 层 | 系统终止新调用并保存完整调用链 |
| 21 | 高风险 Tool 的前置 Hook 超时且无明确允许 | 系统不执行 Tool，并按失败策略记录 |
| 22 | agent core 创建、继续、回答回传或事件流失败 | 保存会话错误状态，服务恢复后记录可查询 |
| 23 | Harness 快照生成失败 | 不调用外部执行服务，保留内部失败记录 |
| 24 | 团队成员使用已发布 Agent，Agent 内部尝试越权 Tool | 后端按当前用户权限拒绝，配置不能借会话越权 |
| 25 | Hook 确认重复提交、超时后迟到提交，并在服务重启后恢复 | 统一确认记录保持单一终态，不重复执行主动作，超时按 Hook 失败策略处理 |
| 26 | 提交不存在、跨空间或无权访问的 `knowledgeDomainIds` | 系统阻止会话创建或继续并定位无效知识域 |

## R-0730 输入增强

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 27 | 在新建、继续输入或开放式回答中输入 `/` 并选择功能 | 结构化功能引用进入本轮快照 |
| 28 | 输入 `@` 并选择当前空间知识文件 | 文件 ID、路径和版本进入本轮快照，所属知识域合并进 `knowledgeDomainIds` |
| 29 | 取消 `/` 或 `@` 选择后提交普通文本 | 系统按普通文本提交 |
| 30 | 仅选择功能或文件但没有文本 | 系统阻止提交 |
| 31 | 提交已停用功能、跨空间文件、无权限文件或超过限制的文件 | 系统拒绝并要求重新选择 |
