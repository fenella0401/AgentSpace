| 字段 | 内容 |
| --- | --- |

| 关联功能 | F-001 |

## 前置条件

- 准备空间创建申请人、团队管理员、团队成员、访客和未获团队角色的租户管理员账号。
- 准备平台默认 Agent、AI 市场 Skill、Tool、Agent 及授权成功和失败样例。
- 准备开发、客服、法务或运营等不同类型的空间测试数据。
- 准备可选远程代码仓、普通上传资料和多知识域数据。
- 准备 Secret Service、AI 草稿服务、Workflow 模拟器和 Hook 事件模拟器。
- 准备 AGENT.md、`.agentspace`、`.codex`、`.claudecode` 和 `.opencode` 文件样例。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 创建空间后不配置知识库或 Harness，点击“开始作业” | 系统打开新建对话页面，并默认使用平台默认 Agent |
| 2 | 创建客服或法务空间，只上传资料，不绑定远程代码仓 | 知识库可用，不出现代码语言、构建命令或开发 Tool 必填项 |
| 3 | 创建纯 Markdown 空间级 `/AGENT.md`，完成章节引导、加载预览、diff 和发布 | `/AGENT.md` 不包含发布元数据；文件路径、作用域、版本、来源、内容 hash 和发布状态写入 `.agentspace/manifest.yaml`，后续任务快照包含空间级指引 |
| 4 | 为两个顶层知识域分别创建 AGENT.md，任务请求的 `knowledgeDomainIds` 只选择其中一个 | 快照只叠加被选择知识域的指引，不加载无关知识域 |
| 5 | 同时引用两个无明显冲突的知识域 | 系统按空间级后知识域级顺序生成快照 |
| 6 | 两个知识域 AGENT.md 存在自由文本疑似语义冲突 | 系统只提示用户确认，不由模型自动阻止；用户确认结果进入快照审计 |
| 7 | 创建超出上下文预算的 AGENT.md | 系统阻止发布并提示拆分到知识域或 Skill |
| 8 | 从市场添加 Skill，再通过自然语言生成自定义 Skill 草稿 | 市场引用和自定义草稿均创建成功，AI 草稿不会自动发布 |
| 9 | 自定义 Skill 缺少测试用例或依赖 Tool | 该 Skill 保持待补全，其他资源和默认 Agent 任务不受影响 |
| 10 | 用户从市场添加 Tool | 系统保存市场 ID、版本、能力快照和授权引用，不展示 MCP 连接表单 |
| 11 | 尝试调用接口新建 Tool 或提交连接地址、启动命令 | 后端拒绝请求 |
| 12 | 草稿 Tool 授权失败，再让已发布 Tool 授权后续失效 | 草稿版本进入待补全；已发布版本保持已发布但资源可用性变为阻断，不新增并列配置状态，不回显凭据 |
| 13 | 从市场添加 Agent，再创建自定义 Agent | 两种 Agent 均可测试；未发布前空间仍使用原默认 Agent |
| 14 | 发布多个 Agent 但不指定空间默认 Agent | 普通任务继续使用平台默认 Agent |
| 15 | 用自然语言生成“产品 Agent -> 人工确认 -> 开发 Agent -> 测试/发布确认 -> 运营 Agent”Workflow，并配置分支、重试、确认测试用例后模拟运行 | 画布只展示 Agent、条件、人工确认、并行、子 Workflow 和输出节点；Skill/Tool 作为 Agent 能力，不作为用户侧 Workflow 节点 |
| 16 | Workflow 存在不可达节点、无终点、无界循环或缺失输入映射 | 系统阻止发布并定位错误节点 |
| 17 | 使用“高风险 Tool 调用审批”示例创建 Hook，配置 `tool.call.before`、high 或 production 条件、管理员确认、超时阻止和审计断言后测试 | 确认通过时继续，拒绝或超时时阻止，高风险 Tool 不静默放行，审计记录完整 |
| 18 | 创建包含任意脚本、未知事件或循环调用的 Hook | 系统阻止发布 |
| 19 | 构造 Hook、Workflow、子 Workflow 超过 5 层调用 | 系统终止新调用并记录完整调用链 |
| 20 | 资源声明普通和敏感环境变量依赖并就地补充，执行 Secret 写入读取探测，再用敏感值明文扫描 AGENT.md、知识库、配置文件、日志、diff、测试记录和快照 | 环境变量版本进入测试通过，原资源重新校验；敏感值只进入 Secret Service，其他位置均无明文 |
| 21 | 环境变量中心为空且无资源声明依赖 | 空间和默认 Agent 任务正常使用 |
| 22 | 删除仍被已发布资源引用的环境变量 | 系统展示受影响资源并阻止静默删除 |
| 23 | 对测试通过资源查看 diff 并发布 | 文件原子写入，资源进入已发布并形成当前生效版本 |
| 24 | 修改已发布资源后不重新发布 | 资源同时展示当前已发布版本和新草稿版本，新任务继续使用上一生效版本 |
| 25 | 外部修改 AGENT.md 或 `.agentspace`，Portal 无草稿 | 系统只生成待确认草稿，不自动测试、发布或替换当前生效版本 |
| 26 | 外部修改 AGENT.md 或 `.agentspace`，同时 Portal 存在草稿 | 系统进入冲突状态，支持采用外部、采用 Portal 或手动合并，选定版本仍需测试、diff 确认和发布 |
| 27 | 发布 Harness 时触发 `harness.publish.before` 阻止或要求确认 | 文件未写入；阻止时结束，确认时使用幂等决定恢复同一发布请求 |
| 28 | 发布成功后触发 `harness.publish.after` 通知失败 | 已发布版本不回滚，HookExecution 记录失败 |
| 29 | 发布 Hook 草稿本身，并观察本次和下一次发布事件 | 本次 before/after 只使用请求创建时的已发布 Hook 集合，目标 Hook 从下一次发布开始生效 |
| 30 | 发布 before Hook 等待确认期间修改权限、目标草稿、基准文件、依赖、测试结果或发布 Hook 集合 | 恢复前重校验发现变化，原确认失效且不写入文件，用户需重新发起发布 |
| 31 | 市场 Tool 版本下线后启动新任务，并观察已启动任务 | 新快照被阻止；已启动任务保留原引用，后续调用按失败策略处理 |
| 32 | 重命名知识域为已存在目录名 | 系统阻止路径冲突，稳定 `domainId` 和历史快照不被改写 |
| 33 | 在知识库中放置 `.codex`、`.claudecode` 或 `.opencode` 有效旧配置但不点击导入 | 系统不扫描、不导入，也不生成 Harness 资源 |
| 34 | 用户选择“从历史配置导入”并扫描 `.codex`、`.claudecode` 和 `.opencode` | 系统展示映射预览，可导入项、不可支持项和敏感值处理结果 |
| 35 | 导入历史 MCP/server/tool 配置 | 系统只生成市场 Tool 匹配建议；找不到市场资源时进入待补全，不导入连接参数 |
| 36 | 导入历史 agent/subagent 配置 | 系统生成 Agent 草稿，subagent 来源标记为 `source=subagent_config`，不新增 Subagent 中心 |
| 37 | 导入脚本型 Hook、env/secret 配置 | 脚本型 Hook 标记为不支持且不能发布；env/secret 只导入变量名和依赖需求，不导入敏感值 |
| 38 | 导入完成后不执行发布 | 导入内容只保留为 Portal 草稿，不写入运行快照 |
| 39 | 对导入草稿完成校验、测试、diff 和发布 | 发布后才写入 AGENT.md 或 `.agentspace` 并进入新快照 |
| 40 | 检查新配置文件布局 | 只生成 `/AGENT.md`、知识域 AGENT.md 和 `/.agentspace/` 下的标准文件 |
| 41 | 使用团队成员、访客和未获团队角色的租户管理员修改 Harness | 后端拒绝无权限写请求；团队成员仍可使用已发布资源 |
