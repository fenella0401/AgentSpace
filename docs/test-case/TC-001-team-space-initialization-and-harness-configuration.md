## 前置条件

- 准备空间创建申请人、团队管理员、团队成员、访客和未获团队角色的租户管理员账号。
- 准备平台默认 Agent、AI 市场 Skill、Tool 及授权成功和失败样例。
- 准备开发、客服、法务或运营等不同类型的空间测试数据。
- 准备可选远程代码仓、普通上传资料和多知识域数据。
- 准备 Secret Service、AI 草稿服务和 AGENT.md 文件样例。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 创建空间后不配置知识库或 Harness，点击“开始作业” | 系统打开统一开始作业入口，默认目标为 Agent，并默认使用平台默认 Agent |
| 2 | 创建客服或法务空间，只上传资料，不绑定远程代码仓 | 知识库可用，不出现代码语言、构建命令或开发 Tool 必填项 |
| 3 | 为知识库文件夹绑定远程代码仓 | 远程代码仓只作为知识来源，不触发 Harness 配置同步 |
| 4 | 创建纯 Markdown 空间级 `/AGENT.md`，完成章节引导、加载预览、diff 和发布 | `/AGENT.md` 不包含发布元数据；后续会话快照包含空间级指引 |
| 5 | 为两个顶层知识域分别创建 AGENT.md，手工会话请求的 `knowledgeDomainIds` 只选择其中一个 | 快照只叠加被选择知识域的指引，不加载无关知识域 |
| 6 | 创建超出上下文预算的 AGENT.md | 系统阻止发布并提示拆分到知识域或 Agent prompt |
| 7 | 从市场选择 Skill | 系统保存市场 Skill ID、版本、能力快照和依赖摘要 |
| 8 | 尝试新建 Skill、AI 生成自定义 Skill 或提交新增 Skill 接口 | 前端无入口且后端拒绝请求 |
| 9 | 用户从市场添加 Tool | 系统保存市场 Tool ID、版本、能力快照和授权引用，不展示 MCP 连接表单 |
| 10 | 尝试调用接口新建 Tool 或提交连接地址、启动命令、MCP 参数 | 后端拒绝请求 |
| 11 | 草稿 Tool 授权失败，再让已发布 Tool 授权后续失效 | 草稿版本进入待补全；已发布版本保持已发布但资源可用性变为阻断，不回显凭据 |
| 12 | 在线配置 Agent 名称、描述、Skill、Tool 和预置 prompt 并测试发布 | Agent 进入已发布状态；未指定空间默认 Agent 时继续使用平台默认 Agent |
| 13 | 尝试从市场选择 Agent 或调用市场 Agent 引用接口 | 前端无入口且后端拒绝请求 |
| 14 | Agent 引用不存在、无权限或下线的 Skill/Tool | 系统阻止测试或发布并定位依赖 |
| 15 | 从 Harness 总览点击 AgentFlow 入口 | 系统进入 F-008 AgentFlow 中心；F-001 不提供 AgentFlow 画布、测试或发布入口 |
| 16 | 尝试配置 Hook 或提交 Hook 配置接口 | 前端无入口且后端拒绝请求 |
| 17 | 资源声明普通和敏感环境变量依赖并就地补充，执行 Secret 写入读取探测 | 环境变量版本进入测试通过，原资源重新校验；敏感值只进入 Secret Service |
| 18 | 用敏感值明文扫描 AGENT.md、知识库、配置、日志、diff、测试记录和快照 | 所有位置均无敏感值明文 |
| 19 | 环境变量中心为空且无资源声明依赖 | 空间和默认 Agent 会话正常使用 |
| 20 | 删除仍被已发布资源引用的环境变量 | 系统展示受影响资源并阻止静默删除 |
| 21 | 对测试通过资源查看 diff 并发布 | 资源进入已发布并形成当前生效版本 |
| 22 | 修改已发布资源后不重新发布 | 资源同时展示当前已发布版本和新草稿版本，新会话继续使用上一生效版本 |
| 23 | 在知识库或远程代码仓中放置 `.codex`、`.claudecode`、`.opencode` 或其他历史配置目录 | 系统不扫描、不导入，也不生成 Harness 草稿 |
| 24 | 使用团队成员、访客和未获团队角色的租户管理员修改 Harness | 后端拒绝无权限写请求；团队成员仍可使用已发布资源 |
