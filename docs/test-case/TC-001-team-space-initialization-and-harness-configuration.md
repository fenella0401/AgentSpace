## 前置条件

- 测试账号具备团队管理员权限。
- 测试账号可访问 AgentSpace Portal。
- 至少准备一个可访问的远程代码仓测试地址。
- 至少准备一份包含 `.codex` 配置目录的知识库测试数据。
- 公司 AI 市场中存在可选择的 Agent、Skill、MCP 或 Workflow 测试资源。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 团队管理员进入 AgentSpace Portal 并创建项目空间，不添加成员，不填写项目编号 | 空间创建成功，创建人成为团队管理员，空间为单人工作空间 |
| 2 | 团队管理员再次创建项目空间，填写有效项目编号并添加成员、授予角色 | 空间创建成功，项目编号保存成功，成员和角色展示正确 |
| 3 | 在知识库中创建顶层文件夹和多层级子文件夹 | 文件夹树展示正确，层级关系正确 |
| 4 | 为文件夹绑定远程代码仓 | 仓库绑定成功，展示仓库信息、同步状态和文档关联状态 |
| 5 | 触发或等待知识库扫描，顶层文件夹中包含 `.codex` 配置目录 | 系统识别配置目录，并解析 Agent、Skill、MCP、Workflow 和环境变量配置 |
| 6 | 进入 Harness 配置中心查看解析结果 | 各中心按配置类型展示配置项、来源、版本、状态和更新时间 |
| 7 | 从公司 AI 市场选择一个可用资源并保存到对应配置中心 | 配置保存成功，知识库配置文件夹内容被更新，同步状态刷新 |
| 8 | 使用非团队管理员账号尝试编辑 Harness 配置 | 系统禁止提交，并展示权限不足或只读状态 |
| 9 | 修改知识库配置文件后重新扫描 | Portal 展示刷新后的配置结果，并更新来源与更新时间 |
| 10 | 制造配置文件解析错误后重新扫描 | 系统展示解析错误，不覆盖最近成功解析结果 |
