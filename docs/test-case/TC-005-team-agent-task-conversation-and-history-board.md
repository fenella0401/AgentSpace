# TC-005-团队空间 Agent 任务对话与历史看板

## 基本信息

| 字段 | 内容 |
| --- | --- |
| 测试用例编号 | TC-005 |
| 关联功能 | F-005 |
| 关联版本 | R-0630 |
| 优先级 | 高 |

## 前置条件

- 测试账号具备团队空间创建者、管理员、团队成员和访客四类角色样例。
- 测试团队空间已归属有效租户，且可通过 devuc 完成空间访问鉴权。
- 测试团队空间已完成 Harness 配置，至少包含一个可传递给外部 agent core 的 Agent 配置。
- 外部 agent core 测试环境可用，支持任务创建、事件返回、询问事件、文档变更事件和错误场景模拟。
- 测试知识库中至少准备一个可被 Agent 修改的文档，并能识别文档 base version。
- 前端可访问 Agent 任务看板、对话详情和 SSE 事件流接口。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | 团队成员进入团队空间，打开新建 Agent 对话主界面，输入有效指令并按默认公开提交 | 系统创建 Agent 任务，记录创建人、团队空间、租户、公开可见性和初始状态 |
| 2 | 检查后端调用外部 agent core 的请求 | 请求包含用户指令、团队空间标识、创建人标识和任务级 Harness 配置快照 |
| 3 | 进入团队空间任务历史看板 | 新任务出现在清单中，展示创建人、模型生成中状态、最新更新时间和公开标识 |
| 4 | 使用创建者、管理员、团队成员和访客账号分别查看公开任务详情 | 四类账号均可只读查看任务详情；非创建人不展示输入框、回答控件、接受或拒绝按钮 |
| 5 | 外部 agent core 返回可展示 thinking 和 output 事件 | 对话详情按轮次展示 thinking 和 output，事件顺序正确 |
| 6 | 外部 agent core 返回未标记可展示的 thinking 事件 | 对话详情不展示该 thinking 内容，不影响后续 output 展示 |
| 7 | 外部 agent core 返回选项型询问事件 | 任务状态变为等待用户输入，创建人看到选项回答控件，其他用户只读查看等待状态 |
| 8 | 创建人选择选项并提交回答 | AgentSpace 将回答回传外部 agent core，任务状态重新变为模型生成中 |
| 9 | 外部 agent core 返回开放式询问事件 | 任务状态变为等待用户输入，创建人可以填写文本回答并提交 |
| 10 | 外部 agent core 返回文档变更事件 | 系统创建待接受文档变更提案，对话详情展示文档卡片、查看内容和查看 diff 入口 |
| 11 | 创建人查看文档内容和 diff 后点击接受 | 系统校验 base version，通过后写入正式文档，提案状态变为已接受 |
| 12 | 再次创建文档变更提案，创建人点击拒绝 | 正式文档保持修改前版本，提案状态变为已拒绝 |
| 13 | 创建人新建私密对话任务 | 任务只在创建人的任务历史看板和详情中可见，其他团队成员、管理员和访客不可见 |
| 14 | 访客尝试新建任务、回答询问、接受或拒绝文档变更 | 系统拒绝操作，并展示只读或权限不足提示 |
| 15 | 非任务创建人尝试回答公开任务中的询问 | 后端返回权限不足，任务状态不被错误更新 |
| 16 | 非任务创建人尝试接受或拒绝公开任务中的文档变更 | 后端返回权限不足，文档提案状态不变 |
| 17 | 模拟 Harness 快照生成失败 | 系统不调用外部 agent core，任务创建失败并提示原因 |
| 18 | 模拟外部 agent core 创建任务失败或认证失败 | 系统保留失败任务记录，任务状态为静止，并在详情中展示错误事件 |
| 19 | 模拟外部 agent core 事件流中断 | 系统记录错误事件，任务状态变为静止，重新打开详情时可查看已保存事件 |
| 20 | 在文档变更提案接受前修改正式文档 base version，再点击接受 | 系统阻止写入，提案进入冲突或提示冲突状态，正式文档不被覆盖 |
