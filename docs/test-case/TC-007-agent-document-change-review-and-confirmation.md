# TC-007-Agent 文档变更审阅与确认

## 基本信息

| 字段 | 内容 |
| --- | --- |
| 测试用例编号 | TC-007 |
| 关联功能 | F-007 |
| 关联版本 | R-0630 |
| 优先级 | 高 |

## 前置条件

- 测试账号具备会话创建人、团队成员和访客样例。
- 测试 Agent 会话已存在 Agent 文档变更提案，提案状态为待接受。
- 测试知识库中至少准备一个可被 Agent 修改的文档，并能识别文档 base version。
- 测试环境可生成或读取 base version 与 proposed version 的 diff。

## 测试步骤

| 步骤 | 操作 | 预期结果 |
| --- | --- | --- |
| 1 | agent core 返回文档变更事件 | 系统创建待接受文档变更提案，记录文档路径、变更类型、base version 和 proposed version |
| 2 | 在 Agent 会话详情中查看文档变更卡片 | 卡片展示文档名称、路径、变更类型、提案状态和操作入口 |
| 3 | 打开文档内容查看 | 系统展示 proposed version 内容 |
| 4 | 打开 diff 查看 | 系统展示 base version 与 proposed version 的差异 |
| 5 | 会话创建人点击接受 | 系统校验 base version，通过后写入正式文档，提案状态变为已接受 |
| 6 | 再次创建文档变更提案，会话创建人点击拒绝 | 正式文档保持修改前版本，提案状态变为已拒绝 |
| 7 | 已接受或已拒绝提案再次打开 | 页面不展示接受或拒绝按钮 |
| 8 | 非会话创建人尝试接受或拒绝文档变更 | 后端返回权限不足，文档提案状态不变 |
| 9 | 访客尝试接受或拒绝文档变更 | 系统拒绝操作，并展示只读或权限不足提示 |
| 10 | 在文档变更提案接受前修改正式文档 base version，再点击接受 | 系统阻止写入，提案进入冲突或提示冲突状态，正式文档不被覆盖 |
| 11 | 模拟文档内容加载失败 | 系统展示失败提示并允许重试 |
| 12 | 模拟 diff 加载失败 | 系统展示 diff 加载失败提示，不影响查看提案摘要 |
