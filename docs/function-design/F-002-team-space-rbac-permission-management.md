## 变更摘要

- 本次变更：按功能设计 PRD、UI、前端、后端与验收剧本重写团队空间 RBAC 设计，将原“范围/核心流程”口径收敛为用户目标、Use Story、Use Case、业务规则、边界异常和验收标准，并明确创建者、管理员、团队成员、访客与未获团队角色的租户管理员边界。
- 影响范围：团队空间创建、空间成员与角色管理、devuc 身份与访问鉴权、团队空间访问守卫、Agent 会话与 Harness CICD 流水线任务发起权限、知识库与 Harness 配置权限、租户管理面跳转、审计日志、前端权限态和验收标准。
- 独立评审：2026-06-08 已完成 PRD 独立 agent review；初审发现 devuc 与 SpaceMember 关系、F-002/F-003 创建职责边界等问题，已修正并经复核全部关闭，无剩余阻断问题。
- 与相邻功能的边界：
  - 租户、租户管理员授权和租户团队空间清单由 F-003 负责，本功能只消费租户上下文和租户级权限结论。
  - 团队空间初始化、知识库和 Harness 资源配置能力由 F-001 负责，本功能负责判断谁可以配置和发布。
  - 手工 Agent 会话创建和继续由 F-005 负责，Harness CICD 流水线任务启动和运行由 F-008 负责，本功能只定义创建者、管理员、团队成员和访客是否具备发起权限。
- 待确认问题：
  - devuc 的用户身份、租户访问、可授权对象校验、错误码、超时和降级策略正式契约。
  - 创建者是否支持移交、退出或降级；R-0630 暂按不支持处理。
  - 成员角色变更后的实时失效策略；R-0630 暂按后端立即生效、前端在刷新或下一次接口返回权限不足时更新处理。
  - 访客只读范围是否包含全部历史任务详情；R-0630 暂按可查看空间内容和历史详情、不可创建或继续任务处理。

## PRD 设计

### 用户目标

- 团队空间创建者可以在具备租户空间创建权限时创建团队空间，并自动成为该空间创建者和管理员。
- 创建者和管理员可以管理空间成员，向用户授予管理员、团队成员或访客角色，并维护成员状态。
- 管理员可以额外添加管理员，使团队空间治理不依赖单一创建人。
- 团队成员可以进入已授权团队空间，查看空间内容并使用已发布 Agent、Skill、Tool 和默认 Agent 创建手工 Agent 会话，或启动已发布 Harness CICD 流水线任务，但不能维护成员、知识库、Harness 基础配置或 Harness CICD 流水线配置。
- 访客可以只读查看已授权团队空间内容和历史详情，但不能邀请成员、创建 Agent 会话、启动 Harness CICD 流水线任务、继续会话、修改配置或触发任何写操作。
- 租户管理员可以在租户管理面查看本租户团队空间清单，但不因租户管理员身份自动进入团队空间或获得内容写权限。
- 平台可以对接 devuc 完成身份、租户上下文和可授权对象校验，并以 SpaceMember 角色作为团队空间内容权限来源，在后端统一执行权限判断和审计。

### 业务规则

- 角色模型：
  - 团队空间内置创建者、管理员、团队成员和访客四类角色，不支持自定义角色和自定义权限点。
  - 创建者由团队空间创建动作自动生成，默认具备管理员能力，不通过成员管理手动授予。
  - R-0630 不支持移除、降级、退出或移交创建者。
  - 管理员具备团队空间治理权限，包括成员管理、角色分配、知识库配置、Harness 配置和已发布资源治理。
  - 团队成员可以查看空间内容、查看历史详情、创建手工 Agent 会话、启动已发布 Harness CICD 流水线任务，并使用已发布 Agent、Skill、Tool、Harness CICD 流水线和默认 Agent。
  - 团队成员不能管理成员、配置知识库、配置 Harness、配置 Harness CICD 流水线、发布资源或修改空间治理设置。
  - 访客可以只读查看空间内容和历史详情，不能创建 Agent 会话、启动 Harness CICD 流水线任务、继续会话、回答询问、邀请成员、修改内容或触发写操作。
  - 同一用户在同一团队空间 R-0630 仅保留一个有效角色。
- 租户上下文：
  - 团队空间必须归属一个有效租户，所有团队空间鉴权必须携带租户上下文。
  - 同一用户可以同时是租户管理员和某团队空间成员；进入团队空间后的权限以团队空间角色为准。
  - 租户管理员未被授予团队空间角色时，只能在租户管理面查看本租户团队空间清单，不默认进入空间内容、编辑成员、配置知识库或配置 Harness。
  - 团队空间请求中的租户上下文与 TeamSpace.tenantId 不一致时，后端必须拒绝操作。
- 权限矩阵：

| 权限项 | 创建者 | 管理员 | 团队成员 | 访客 | 租户管理员（未授予团队角色） |
| --- | --- | --- | --- | --- | --- |
| 进入团队空间壳和导航 | 是 | 是 | 是 | 是 | 否 |
| 查看空间内容和历史详情 | 是 | 是 | 是 | 是 | 否 |
| 查看租户管理面团队空间清单 | 否 | 否 | 否 | 否 | 是 |
| 创建手工 Agent 会话或启动 Harness CICD 流水线任务 | 是 | 是 | 是 | 否 | 否 |
| 继续本人创建的手工会话或回答询问 | 是 | 是 | 是 | 否 | 否 |
| 管理成员与角色 | 是 | 是 | 否 | 否 | 否 |
| 添加管理员 | 是 | 是 | 否 | 否 | 否 |
| 管理知识库配置 | 是 | 是 | 否 | 否 | 否 |
| 配置、测试、发布或停用 Harness / Harness CICD 流水线 | 是 | 是 | 否 | 否 | 否 |
| 执行空间治理类写操作 | 是 | 是 | 否 | 否 | 否 |

- 鉴权与审计：
  - 身份、租户访问和可授权对象校验需对接 devuc；团队空间内容权限必须来自有效 SpaceMember 角色。
  - 若 devuc 后续返回空间级授权信息，R-0630 也必须先同步或映射为 SpaceMember 后才可生效，不能绕过团队空间 RBAC。
  - 后端 RBAC 判断是所有空间内容访问和写操作的最终依据。
  - 前端权限态只作为体验优化，不能替代后端鉴权。
  - devuc 鉴权失败、超时或返回不可信结果时，系统默认拒绝访问或受限操作。
  - 成员添加、角色调整、成员移除、权限拒绝和鉴权异常均需记录审计。
  - 审计记录至少包含租户、空间、操作者、目标用户、动作、变更前角色、变更后角色、时间、结果和失败原因。

### 边界与异常

- 用户没有有效 SpaceMember 角色时，系统拒绝进入空间内容，不因 devuc 身份或租户资格校验通过而单独放行。
- devuc 鉴权失败或超时时，系统提示稍后重试，并拒绝进入空间或执行受限操作。
- 用户角色为空、重复、失效、被删除或与租户上下文不一致时，系统按无权限处理。
- 团队成员尝试保存知识库或 Harness 配置时，后端拒绝提交并提示无配置权限。
- 访客尝试创建任务、继续任务、邀请成员或提交任意写操作时，后端拒绝并提示访客仅可查看。
- 租户管理员尝试以租户管理员身份进入未授权团队空间内容时，系统提示需要团队空间授权。
- 创建者被移除、降级或移交的请求会被拒绝。
- 成员添加或角色更新失败时，系统不得产生部分成功的脏数据，并保留可重试状态。
- 角色变更后，已打开页面的权限状态在刷新、重新进入页面或下一次接口返回权限不足时更新；后端权限立即按最新角色生效。
- 审计写入失败时，成员和角色写操作不得静默成功。

## 验收标准

### 正常路径

- 具备目标租户空间创建权限的用户创建团队空间后，系统将该用户标记为创建者并赋予管理员能力。
- 创建者可以添加管理员、团队成员和访客。
- 管理员可以添加管理员、团队成员和访客。
- 创建者和管理员可以调整或移除非创建者成员。
- 团队成员可以进入已授权团队空间，查看空间内容和历史详情，并创建手工 Agent 会话或启动已发布 Harness CICD 流水线任务。
- 访客可以进入已授权团队空间并只读查看空间内容和历史详情。
- 租户管理员可以在租户管理面查看本租户团队空间清单。
- 后端可以通过 devuc 完成身份、租户上下文和可授权对象校验，并以 SpaceMember 角色完成团队空间权限判断。
- 成员添加、角色调整、成员移除和权限拒绝均写入审计记录。

### 边界场景

- 同一用户在同一团队空间仅有一个有效角色。
- 同一用户同时具备租户管理员和团队空间成员身份时，团队空间内权限按团队空间角色生效。
- 创建者在成员列表中展示为创建者，并同时具备管理员能力。
- R-0630 不允许移除、降级、退出或移交创建者。
- 角色变更后，后端立即按最新角色鉴权；用户重新进入空间时看到最新角色和权限。
- 租户管理员未被授予团队角色时，只能查看租户管理面团队空间清单，不能进入团队空间内容。

### 异常场景

- devuc 鉴权失败、超时或返回不可信结果时，系统拒绝访问或受限操作。
- 没有有效 SpaceMember 角色的用户不能访问团队空间内容，即使 devuc 身份或租户资格校验通过也不能单独放行。
- 角色为空、失效、重复或租户上下文不一致时，系统按无权限处理。
- 成员添加失败时，系统提示失败原因，不写入半完成成员关系，并允许创建者或管理员重试。
- 角色更新失败时，系统保留原角色，不产生部分成功的权限数据。
- 审计写入失败时，成员和角色写操作不会静默成功。

### 权限场景

- 团队成员不能管理成员、配置知识库、配置 Harness 或发布资源。
- 访客不能创建任务、继续任务、回答询问、邀请成员或触发任何写操作。
- 团队成员和访客调用无权写接口时，后端返回权限不足。
- 租户管理员未被授予团队空间管理员角色时，不能编辑该团队空间成员、知识库或 Harness 配置。
- 前端隐藏或禁用无权限入口时，直接调用后端接口仍会被拒绝。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：PRD 明确角色、租户上下文、只读态、成员管理、任务入口权限和越权提示，需要刷新页面入口、关键状态、交互规则和文案。

### 页面与交互

- 成员管理页：
  - 展示当前团队空间所属租户、空间名称和当前用户角色。
  - 成员表展示姓名、账号、角色、状态、加入时间、最近更新和操作入口。
  - 创建者以系统标识展示，不出现在角色下拉可选项中。
  - 创建者和管理员可以添加成员、调整角色、移除非创建者成员。
  - 角色选择仅包含管理员、团队成员和访客。
  - 目标成员为创建者时，移除、降级和移交操作不可用。
- 权限态：
  - 创建者和管理员展示成员管理、知识库配置、Harness 配置和发布入口。
  - 团队成员展示任务创建入口和已发布能力使用入口，不展示成员管理和 Harness 写操作。
  - 访客页面保持只读，不展示任务创建、继续输入、邀请、保存、删除、发布和运行类主按钮。
  - 租户管理员从租户清单点击未授权空间时，展示需要团队空间授权的阻断页或提示。

### 状态与文案

- 空状态：`暂无成员，添加成员后即可协作。`
- 创建者保护：`创建者由系统生成，当前版本不支持移除、降级或移交。`
- 团队成员只读配置：`当前角色可使用已发布能力，但不能修改空间配置。`
- 访客只读：`当前角色仅支持查看。`
- 租户管理员无团队角色：`你可以在租户管理面查看该空间，但需要团队空间授权后才能进入内容。`
- 异常态：无访问权限、权限不足、devuc 鉴权失败、成员添加失败、角色更新失败、审计写入失败。

## 前端设计

### 是否需要刷新

- 结论：是
- 理由：前端需要消费后端权限集合，控制成员管理、任务入口、空间配置入口和只读状态，并对权限变化和接口拒绝做一致处理。

### 页面、组件与状态

- 页面路由：
  - 团队空间成员管理页。
  - 团队空间权限设置区域。
  - 租户管理面团队空间清单到空间内容的跳转拦截状态。
- 核心组件：
  - `MemberList`：成员列表、创建者标识和成员状态。
  - `AddMemberDialog`：搜索用户并授予角色。
  - `MemberRoleSelect`：管理员、团队成员和访客角色选择。
  - `PermissionGuard`：控制按钮、菜单、表单和入口是否可用。
  - `TenantScopeBadge`：展示空间所属租户和当前权限来源。
  - `ReadonlyBanner`：展示团队成员、访客或租户管理员无团队角色的只读原因。
  - `PermissionDeniedState`：承接后端拒绝、devuc 失败和租户上下文不一致状态。
- 状态管理：
  - 当前租户上下文、团队空间 ID 和 TeamSpace.tenantId。
  - 当前用户角色、权限集合和权限来源。
  - 成员列表、搜索结果、角色变更草稿和提交状态。
  - devuc 鉴权状态、权限不足状态和审计失败状态。
  - 页面打开后的权限版本或更新时间，用于收到权限不足后刷新权限。

### 接口依赖与异常处理

- 依赖接口：
  - 查询当前用户在团队空间内的角色和权限集合。
  - 查询当前团队空间所属租户和空间摘要。
  - 查询团队空间成员列表。
  - 搜索可添加用户并校验其租户访问资格。
  - 添加成员并授予角色。
  - 更新成员角色。
  - 移除非创建者成员。
  - 查询租户管理员进入空间内容前的团队角色授权状态。
- 异常处理：
  - 接口返回权限不足时，前端刷新权限集合并切换为只读或阻断状态。
  - devuc 鉴权失败时，禁止继续提交受限操作，并提示稍后重试。
  - 成员添加或角色更新失败时，保留表单输入和目标用户选择。
  - 创建者保护错误返回时，禁用对应操作并展示创建者保护文案。
  - 审计写入失败导致写操作失败时，提示操作未完成。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：后端需要对接 devuc、维护团队空间成员角色关系、解析租户上下文、输出权限集合，并对空间治理、任务创建、知识库和 Harness 写接口统一鉴权。

### 接口与数据

- 核心服务：
  - `PermissionService`：封装团队空间 RBAC 权限判断。
  - `DevucAuthAdapter`：对接 devuc，完成身份、租户访问和可授权对象校验。
  - `SpaceMembershipService`：管理空间成员、角色、状态和创建者初始化。
  - `TenantScopePermissionResolver`：合并 TeamSpace.tenantId、请求租户上下文和用户租户级权限。
  - `AuditService`：记录成员、角色和权限拒绝审计。
  - `PermissionSnapshotService`：为前端返回当前角色、权限集合和只读原因。
- 核心数据对象：
  - `SpaceMember`：租户 ID、空间 ID、用户 ID、角色、状态、加入时间、创建时间、更新时间。
  - `RolePermission`：角色、权限项、是否允许和版本。
  - `PermissionAuditLog`：租户 ID、空间 ID、操作者、目标用户、动作、变更前后内容、结果、失败原因、时间。
  - `PermissionContext`：用户 ID、租户 ID、空间 ID、devuc 鉴权结果、团队空间角色和权限集合。
- 建议接口：
  - `GET /team-spaces/{spaceId}/permissions/me`
  - `GET /team-spaces/{spaceId}/members`
  - `GET /team-spaces/{spaceId}/member-candidates`
  - `POST /team-spaces/{spaceId}/members`
  - `PATCH /team-spaces/{spaceId}/members/{userId}`
  - `DELETE /team-spaces/{spaceId}/members/{userId}`

### 业务规则、权限与异常处理

- 创建团队空间时必须在同一事务内写入 TeamSpace 和创建者 SpaceMember。
- 每次访问团队空间内容前需完成 devuc 身份与租户资格校验、租户上下文校验和有效 SpaceMember 角色解析。
- 每次执行成员、知识库、Harness、Harness CICD 流水线、空间治理和会话/任务创建类操作前需完成后端 RBAC 校验。
- 角色权限 R-0630 使用系统内置映射，不提供后台自定义权限点。
- 创建者角色只能由空间创建生成，不能被成员接口创建、删除、降级或移交。
- 管理员可以新增管理员、团队成员和访客，也可以调整或移除非创建者成员。
- 团队成员允许创建手工 Agent 会话或启动已发布 Harness CICD 流水线任务，但不允许调用成员、知识库、Harness、Harness CICD 流水线配置和空间治理写接口。
- 访客不允许调用 Agent 会话创建、Harness CICD 流水线任务启动、继续、回答询问、成员、知识库、Harness、Harness CICD 流水线配置和空间治理写接口。
- 租户管理员不默认继承团队空间管理员权限；若其被添加为团队空间管理员，则按团队空间管理员权限执行。
- 鉴权失败、角色缺失、租户不匹配、权限映射异常或审计写入失败时，默认拒绝写操作。
