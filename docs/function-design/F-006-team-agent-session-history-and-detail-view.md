## 变更摘要

- 本次变更：将历史清单统一调整为“Agent 会话历史”，展示 `source=manual` 的手工 Agent 会话和 `source=harness_pipeline_node` 的 Harness CICD 流水线节点自动会话。
- 2026-06-08 拆分口径：
  - 手工 Agent 会话由 F-005 创建和运行。
  - Harness CICD 流水线节点 Agent 会话由 F-008 HarnessPipelineEngine 自动创建。
  - F-006 只负责清单、详情、事件、最终输出、修改文件和来源链接展示，不提供创建、继续、改名、确认、取消或重试能力。
- 影响范围：Agent 会话看板、会话详情页、来源标识、Harness CICD 流水线父任务链接、执行过程流式展示、最终输出展示、plan 进度弹窗、事件查询、访客只读和 TC-006。
- 已确认规则：
  - 会话列表首版支持筛选项：全部、我的、今天、近一周。
  - 会话列表支持会话名称模糊搜索和分页。
  - 执行过程中的普通文件路径、工具参数和工具结果面向普通用户均可见。
  - 相对更新时间的最小粒度和刷新频率均为分钟。
- 待确认问题：
  - agent core 需要确认执行过程事件、plan 进度事件、最终输出事件和修改文件事件字段映射。
  - Harness CICD 流水线节点会话在“我的”筛选中的归属是否按 Harness CICD 流水线任务创建人计算；R-0630 暂按父 Harness CICD 流水线任务创建人计算。

## PRD 设计

### 用户目标

- 具备团队空间访问权限的用户可以查看团队空间中永久保留的 Agent 会话历史。
- 用户可以在同一会话清单中查看手工 Agent 会话和 Harness CICD 流水线节点自动会话，并通过来源标识理解会话由谁发起。
- 用户可以在会话历史清单中按全部、我的、今天、近一周筛选会话，按会话名称模糊搜索，并分页浏览结果。
- 访客可以只读查看团队空间 Agent 会话历史和详情。
- 用户可以进入会话详情，查看用户指令或节点输入、Agent 详细执行过程、最终 Agent 输出和修改文件。
- 用户可以在 Harness CICD 流水线节点会话中看到父 Harness CICD 流水线任务、Run 和节点链接，并可跳回 F-008 Harness CICD 流水线任务详情。
- 用户可以在 Agent 执行过程中流式查看详细执行过程，并在 Agent 开始生成最终 output 后看到默认收起的执行过程，按需点击展开。
- 用户可以在主对话界面之外，选择以弹窗查看 Agent 执行 plan 与进度。
- 手工会话创建人修改会话名称后，会话历史和详情需要展示最新名称。
- 用户可以看到会话状态和最新更新时间，判断会话处于运行中、待输入、完成静止态或异常静止态。

### 业务规则

- 会话清单和详情必须继承团队空间租户上下文。
- 具备团队空间访问权限的创建者、管理员、团队成员和访客均可查看 Agent 会话清单和详情。
- Agent 会话来源为 `manual` 或 `harness_pipeline_node`。
- `manual` 会话由 F-005 手工入口创建；创建人可以在 F-005 权限规则下继续、回答询问和修改名称。
- `harness_pipeline_node` 会话由 F-008 HarnessPipelineEngine 自动创建；清单和详情展示父 Harness CICD 流水线任务、Run 和节点链接，但不展示继续、回答、改名、取消或重试控件。
- 访客仅可只读查看，不展示继续输入、回答询问、修改名称或处理文档变更的控件。
- 除手工会话创建人外，其他查看用户均按只读展示。
- 会话清单展示会话名称、来源、创建人、状态和最新更新时间。
- 会话列表首版必须支持筛选项：全部、我的、今天、近一周；默认筛选项为全部。
- 我的筛选项展示当前用户创建的手工会话，以及父 Harness CICD 流水线任务由当前用户创建的 Harness CICD 流水线节点会话。
- 今天和近一周筛选项按会话最新更新时间筛选，并以团队空间时区计算时间边界。
- 今天筛选范围为团队空间时区自然日开始时间至当前时刻，包含边界。
- 近一周筛选范围为当前时刻向前 7 个自然日内，包含当前时刻和起始边界。
- 会话列表必须支持按会话名称进行模糊搜索，搜索条件可与筛选项组合使用。
- 会话列表必须支持分页，分页参数至少包含页码和每页数量。
- 会话列表默认按最新更新时间倒序排序；最新更新时间相同时按会话 ID 倒序稳定排序。
- 会话列表默认每页 20 条；分页响应必须返回总数、总页数、当前页、每页数量、是否有下一页和是否有上一页。
- 筛选项或搜索关键词变化时，前端必须重置到第一页重新查询。
- 会话状态展示为运行中、待输入、完成静止态、异常静止态。
- 手工会话名称来自 F-005 的模型生成、兜底生成或用户手动修改结果。
- Harness CICD 流水线节点会话默认名称由 F-008 使用 Harness CICD 流水线名称和节点名称生成，用户不可直接修改。
- 详情页必须展示输入、Agent 详细执行过程、最终 Agent 输出和修改文件。
- Agent 详细执行过程至少覆盖调用的协作 Agent、使用的 skill、工具、阅读的文件和编辑的文件。
- Agent 执行过程在会话运行中流式展示；Agent 执行结束并开始生成最终 output 时，执行过程默认收起，并保留用户点击展开入口。
- Agent 执行过程默认收起必须由后端返回的 `final_output_started` 或等价阶段事件触发，不依赖前端猜测。
- Agent 执行过程需要面向用户友好展示，避免直接堆叠原始日志；执行过程事件判定可展示后，文件路径、工具参数和工具结果对具备会话查看权限的普通用户默认完整可见。
- 仅当文件路径、工具参数或工具结果命中密钥、凭据、个人隐私或安全策略时，后端可对命中片段脱敏，并返回脱敏原因标识。
- 最终 Agent 输出包括给用户展示的最终 output 和本次会话修改的文件清单。
- 修改文件在详情页展示文件路径、变更类型和摘要入口；具体内容、diff、接受或拒绝由 F-007 承接。
- 除主界面展示进度外，用户可选择弹窗查看 Agent 执行 plan 与进度。
- 询问事件作为执行过程事件的一种展示在 Agent 执行过程时间线中；回答控件和回答权限由 F-005 承接，Harness CICD 流水线节点会话的人工确认由 F-008 承接。
- 会话状态和名称由 F-005 或 F-008 维护，F-006 负责展示。
- 最新更新时间以会话、轮次、事件、询问回答、名称修改、Harness CICD 流水线节点状态或文档提案最近更新时间为准。
- 相对更新时间由前端根据后端返回的最新更新时间渲染，最小展示粒度为分钟；会话看板和会话详情中可见的相对更新时间必须每 60 秒刷新一次。
- Agent 会话历史永久保留；首版不提供删除入口。
- F-006 不提供新建 Agent 会话、调用外部 agent core、继续输入、修改名称、回答询问、Harness CICD 流水线确认、取消、重试、文档内容查看、diff、接受或拒绝能力。

### 边界与异常

- 当前空间暂无 Agent 会话时展示空状态。
- 当前空间有会话但筛选或搜索无结果时展示无结果状态，不与暂无会话空状态混用。
- 分页参数超出可用页数时展示无结果状态，并允许用户返回上一页或重置筛选搜索条件。
- 会话清单查询失败时展示加载失败并允许重试，不与无会话空状态混用。
- 事件查询失败时展示加载失败并允许重试。
- 事件流中断时不清空已展示事件，并允许重新查询已保存事件。
- agent core 未返回执行过程事件时，详情页仍展示输入、最终 output 和修改文件，并提示暂无详细执行过程。
- 执行过程事件包含不可展示内容时，后端不返回原始内容，前端展示已隐藏或已脱敏状态。
- 文件路径、工具参数或工具结果仅在命中密钥、凭据、个人隐私或安全策略时展示脱敏状态；普通参数和普通结果不得被摘要替代。
- plan 或进度事件不存在时，弹窗展示当前会话状态和暂无 plan 提示，不阻塞主详情页。
- Harness CICD 流水线父任务、Run 或节点已不可访问时，系统保留来源标识并提示父级流水线不可访问，不泄露其他团队空间信息。
- 目标会话不存在、会话不属于当前团队空间或链接失效时，系统展示无权限或不存在提示，不泄露其他团队空间会话信息。
- 用户失去团队空间访问权限后，再次访问会话清单或详情时按无权限处理。

## 验收标准

### 正常路径

- 用户进入团队空间 Agent 会话看板后，可以看到团队空间会话历史。
- 会话清单展示会话名称、来源、创建人、状态和最新更新时间。
- 会话清单支持全部、我的、今天、近一周筛选。
- 会话清单支持按会话名称模糊搜索。
- 会话清单支持分页查看。
- 会话清单默认按最新更新时间倒序排序，并在最新更新时间相同时保持稳定排序。
- 筛选项或搜索关键词变化时，会话清单重置到第一页。
- 创建者、管理员、团队成员和访客均可查看会话清单和详情。
- 会话状态按运行中、待输入、完成静止态、异常静止态展示。
- 用户点击会话后可以进入会话详情。
- 详情页展示输入、Agent 详细执行过程、最终 Agent 输出和修改文件。
- Agent 详细执行过程展示调用的协作 Agent、使用的 skill、工具、阅读和编辑的文件。
- Agent 详细执行过程中的普通文件路径、普通工具参数和普通工具结果对普通用户完整可见。
- Agent 详细执行过程中的密钥、凭据、个人隐私或安全策略命中片段被脱敏，并展示脱敏状态。
- 会话运行时，详情页可以流式追加展示新增执行过程和进度事件。
- 后端返回 `final_output_started` 或等价阶段事件后，详情页默认收起详细执行过程，并允许用户展开查看。
- 最终输出区展示给用户的最终 output 和修改文件清单。
- 用户可以通过弹窗查看 Agent 执行 plan 与进度，弹窗不影响主界面。
- 手工会话创建人修改名称后，会话看板和详情展示最新名称。
- Harness CICD 流水线节点会话展示 Harness CICD 流水线来源、父 Harness CICD 流水线任务、Run 和节点链接。
- 修改文件入口仅跳转或调起 F-007 的文档变更审阅能力。

### 边界场景

- 当前空间暂无 Agent 会话时展示空状态。
- 当前空间有会话但筛选或搜索无结果时展示无结果状态。
- 分页页码超出可用范围时展示无结果状态，并允许返回上一页或重置筛选搜索条件。
- 会话没有执行过程事件时，详情页仍可正常展示输入、最终 output 和修改文件。
- 执行过程事件包含不可展示内容时，详情页不展示原始内容；普通文件路径、普通工具参数和普通工具结果不属于不可展示内容。
- 会话没有 plan 事件时，plan 弹窗展示暂无 plan 提示和当前会话状态。
- 相对更新时间能随会话事件、回答、名称修改、Harness CICD 流水线节点状态或文档提案变化，并在会话看板和详情页每 60 秒刷新一次。
- 完成静止态和异常静止态会话仍保留在会话历史中。
- Harness CICD 流水线节点会话详情只读，并提示回到 Harness CICD 流水线任务页处理流水线操作。

### 异常场景

- 事件查询失败时，系统展示加载失败并允许重试。
- 会话清单查询失败时，系统展示加载失败并允许重试，且不展示为空状态。
- 事件流中断时，系统保留已展示事件，并允许重新拉取已保存事件。
- 父 Harness CICD 流水线任务不可访问时，仅展示来源标识和不可访问提示，不泄露父级流水线详情。
- 目标会话不存在、会话不属于当前团队空间或链接失效时，系统展示无权限或不存在提示。
- 用户失去团队空间访问权限后，系统拒绝访问会话清单或详情。

### 权限场景

- 访客可以只读查看会话清单和详情。
- 非手工会话创建人只读查看会话清单和详情，不展示继续输入、回答询问、修改名称或处理文档变更的控件。
- Harness CICD 流水线节点会话对所有用户均不展示直接继续、改名、取消或重试控件。
- 访客不能继续输入、回答询问、修改名称或处理文档变更。
- 未加入团队空间且未被授权的用户不能查看会话清单或详情。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：需要新增来源标识、Harness CICD 流水线父任务链接、Harness CICD 流水线节点会话只读提示，并将历史入口统一为 Agent 会话看板。

### 页面与交互

- 团队空间 Agent 会话看板：
  - 展示会话名称、来源、创建人、状态和最新更新时间。
  - 来源以 `手工创建`、`Harness CICD 流水线节点` 区分。
  - 提供全部、我的、今天、近一周筛选项。
  - 提供会话名称模糊搜索输入框。
  - 提供分页控件。
  - 分页控件展示总数、总页数、当前页、每页数量、上一页和下一页状态。
  - 支持点击进入详情。
- 团队空间 Agent 主界面：
  - 提供 plan / 进度入口。
  - 用户点击后以弹窗查看 Agent plan、当前步骤、步骤状态和最新进度。
  - 弹窗关闭后不影响主界面继续展示或输入。
- Agent 会话详情页：
  - 顶部展示会话名称、来源、创建人、状态和最新更新时间。
  - Harness CICD 流水线节点会话展示父 Harness CICD 流水线任务、Run、节点和返回链接。
  - 主体展示输入、Agent 详细执行过程、最终 Agent 输出和修改文件。
  - Agent 详细执行过程以时间线或分组摘要展示调用的协作 Agent、使用的 skill、工具、阅读和编辑的文件。
  - 会话运行中流式追加执行过程和进度事件。
  - 收到 `final_output_started` 或等价阶段事件后默认收起详细执行过程，用户可点击展开。
  - 最终输出区展示给用户的最终 output 和修改文件清单。
  - 提供 plan / 进度弹窗入口。
  - 非创建人、访客和 Harness CICD 流水线节点会话看到只读提示，不展示继续输入、回答、改名或文档处理控件。

### 状态与文案

- 空状态：当前空间暂无 Agent 会话。
- 无结果状态：没有符合条件的 Agent 会话。
- 加载态：会话加载中、事件加载中、事件连接中。
- 会话状态：运行中、待输入、完成静止态、异常静止态。
- 来源：手工创建、Harness CICD 流水线节点。
- 异常态：无权限、事件加载失败、事件流中断、暂无执行过程、暂无 plan、父 Harness CICD 流水线不可访问。
- 只读态：当前用户仅可查看。
- Harness CICD 流水线来源提示：`该 Agent 会话由 Harness CICD 流水线节点自动创建，请回到 Harness CICD 流水线任务处理流水线操作。`

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：需要提供 Agent 会话列表、详情、事件查询、事件流、来源字段和父 Harness CICD 流水线链接。

### 接口与数据

- 核心服务：
  - `AgentSessionQueryService`：按筛选项、会话名称关键词、来源和分页参数查询会话列表，并查询会话详情。
  - `AgentSessionEventService`：查询会话事件。
  - `SessionEventStreamService`：向前端提供事件流。
  - `AgentPlanProgressService`：查询 Agent plan、步骤状态和进度事件。
  - `HarnessPipelineParentLinkService`：解析 Harness CICD 流水线节点会话的父任务、Run 和节点链接。
  - `PermissionService`：判断团队空间会话列表和详情查看权限。
- 核心数据对象：
  - `AgentSession`：会话 ID、租户 ID、团队空间 ID、创建人 ID、来源、会话名称、名称来源、状态、最新更新时间、父 Harness CICD 流水线关联。
  - `AgentSessionSource`：`manual`、`harness_pipeline_node`。
  - `AgentSessionTurn`：轮次 ID、会话 ID、输入、状态、创建时间、更新时间。
  - `AgentSessionExecutionEvent`：事件 ID、会话 ID、轮次 ID、事件类型、展示标题、展示摘要、结构化 payload、可展示级别、脱敏原因、序号、创建时间。
  - `AgentSessionExecutionPayload`：`agentName`、`skillName`、`toolName`、`toolParameters`、`toolResult`、`filePath`、`fileChangeType`、`questionTitle`、`progressText`。
  - `AgentSessionFinalOutput`：会话 ID、轮次 ID、给用户的最终 output、修改文件列表、创建时间。
  - `AgentModifiedFile`：文件路径、变更类型、变更摘要、文档变更提案 ID。
  - `AgentPlanProgress`：会话 ID、plan 步骤、当前步骤、步骤状态、最新进度事件。
  - `HarnessPipelineParentLink`：HarnessPipelineTask、HarnessPipelineRun、HarnessPipelineNodeRun 的可见链接和摘要。
  - `AgentSessionListPage`：会话列表、总数、总页数、当前页、每页数量、是否有下一页、是否有上一页。
  - `AgentSessionStatus`：运行中、待输入、完成静止态、异常静止态。
- 建议接口：
  - `GET /team-spaces/{spaceId}/agent-sessions`
  - `GET /team-spaces/{spaceId}/agent-sessions/{sessionId}`
  - `GET /team-spaces/{spaceId}/agent-sessions/{sessionId}/events`
  - `GET /team-spaces/{spaceId}/agent-sessions/{sessionId}/stream`
  - `GET /team-spaces/{spaceId}/agent-sessions/{sessionId}/plan-progress`

### 业务规则、权限与异常处理

- 查询会话列表时，返回当前团队空间内符合筛选、搜索和分页条件的 Agent 会话。
- 会话列表查询参数至少支持 `filter`、`sessionNameKeyword`、`source`、`page` 和 `pageSize`。
- `filter` 支持 `all`、`mine`、`today`、`last7Days`，分别对应全部、我的、今天和近一周。
- `mine` 对手工会话按当前登录用户作为创建人过滤；对 Harness CICD 流水线节点会话按父 Harness CICD 流水线任务创建人过滤。
- `today` 和 `last7Days` 按会话最新更新时间和团队空间时区过滤。
- 会话列表默认按最新更新时间倒序排序；最新更新时间相同时按会话 ID 倒序稳定排序。
- 默认 `pageSize` 为 20；响应必须包含会话列表、总数、总页数、当前页、每页数量、是否有下一页和是否有上一页。
- 查询会话详情和事件时，必须校验用户具备团队空间访问权限。
- Harness CICD 流水线父任务链接必须二次校验可见性，不可见时不返回父级流水线详情。
- 执行过程事件只有在 agent core 或 AgentSpace 后端判定为可展示时才写入或返回可展示内容。
- 执行过程事件需要按序号有序返回，支持流式追加和断线后补拉。
- `final_output_started` 或等价阶段事件是前端默认收起详细执行过程并切换最终 output 展示的唯一触发条件。
- 执行过程事件判定可展示后，文件路径、工具参数和工具结果面向具备会话查看权限的普通用户默认完整返回。
- 仅命中密钥、凭据、个人隐私或安全策略的片段允许脱敏；脱敏时必须保留事件并返回脱敏原因标识。
- 最终输出和修改文件清单需要可独立于执行过程加载，避免执行过程缺失影响用户查看结果。
- plan 进度事件需要支持主界面弹窗、详情页弹窗和详情页内嵌展示共用同一数据源。
- 事件流中断不删除已保存事件。
- Agent 会话历史永久保留，不因会话进入完成静止态、异常静止态或用户离开空间而删除。
