## 变更摘要

- 本次变更：补充用户输入增强能力，支持在新建对话、继续输入和开放式询问回答中使用 `/` 唤起 Agent 功能选择，使用 `@` 指定 Harness 知识空间文件。
- 影响范围：F-005 PRD、验收标准、团队空间对话输入框、任务创建接口、继续输入接口、询问回答接口、Harness 快照、Agent 功能目录、知识空间文件引用、外部 agent core 适配和验收口径。
- 待确认问题：
  - 外部 agent core 的正式创建任务接口、继续对话接口、认证方式、错误码和超时阈值。
  - agent core 询问事件的选项字段、单选/多选标识、开放式回答长度限制。
  - 对话名称生成模型的调用方式、提示词模板、失败判定和内容安全策略。
  - SSE 在目标网关和浏览器环境中的支持情况；若不支持，事件流语义保持不变并迁移为 WebSocket。
  - Agent 功能目录的版本字段和后续版本是否允许每轮输入选择多个 Agent 功能。
  - Harness 知识空间文件引用的大小上限，以及传递给外部 agent core 时使用文件句柄还是文件内容。

## PRD 设计

### 用户目标

- 团队空间创建者、管理员和团队成员可以在团队空间内打开对话主界面，输入指令并发起一个 Agent 对话任务。
- 团队成员无需手工拼装运行参数，系统自动使用团队空间当前 Harness 配置生成指令级快照。
- 具备创建权限的用户在输入首条指令时可以用 `/` 从当前 Agent 可用功能中选择本轮意图。
- 具备创建权限的用户在输入首条指令时可以用 `@` 指定 Harness 知识空间中的文件作为本轮明确上下文。
- 系统永久保留对话历史，对话创建人可以回到已进入静止态且已建立外部会话的历史对话中继续输入指令。
- 系统调用模型总结用户首次输入的指令作为对话名称，对话创建人可以修改对话名称。
- 对话创建人可以在 Agent 任务待输入时回答询问，开放式回答同样支持 `/` 功能选择和 `@` 文件引用。
- 团队成员可以看到 Agent 任务的关键运行状态，并在外部服务失败时理解任务已停止和失败原因。

### 业务规则

- 任务必须归属一个团队空间，并继承团队空间租户上下文。
- 创建者、管理员和团队成员可以新建任务；访客不可新建任务。
- 新建任务前必须提供有效首条指令。
- 一个对话对应一个 Agent 任务；一个 Agent 任务可以包含多轮用户指令。
- 首版只有任务创建人可以继续输入指令和回答 Agent 询问。
- 首版仅当任务处于完成静止态或异常静止态且已有外部 agent core conversation ID 时，任务创建人可以继续输入；运行中不允许追加新指令，待输入时只能回答当前询问。
- 若首轮外部 agent core 创建失败且未获得 conversation ID，系统保留异常任务记录，但只允许创建人重试首轮创建或重新发起新任务，不允许按历史会话继续输入。
- 支持结构化输入增强的输入框包括新建对话首条指令、历史对话继续输入和开放式询问文本回答；选项型询问仅提交选项，不支持额外 `/` 或 `@` 输入。
- Agent 功能指当前团队空间 Harness 配置中标记为可由用户显式选择的 Agent、Skill、Workflow 或 Agent 内置能力；未选择时使用当前团队空间默认 Agent 能力。
- 用户输入 `/` 时，前端需要唤起当前团队空间 Agent 可用功能选择；首版每轮输入至多选择一个 Agent 功能，未选择时按默认 Agent 能力运行。
- Agent 功能选择需要保存为结构化字段，至少包含功能 ID、功能名称、功能版本或来源标识；不得仅依赖指令正文中的文本。
- 当结构化 Agent 功能选择与指令正文中的自然语言意图存在冲突时，结构化功能选择作为路由优先信息传递给 agent core，指令正文作为本轮任务说明保留。
- Harness 知识空间指当前团队空间知识库中已纳入 Harness 配置、允许 Agent 引用并可按用户权限访问的文件范围。
- 用户输入 `@` 时，前端需要唤起当前 Harness 知识空间文件选择；用户可以选择一个或多个文件作为本轮输入引用，首版每轮最多引用 10 个文件。
- Harness 知识空间文件引用需要保存为结构化字段，至少包含知识空间 ID、文件 ID、文件路径或展示名、文件版本；不得仅依赖指令正文中的文本。
- Harness 知识空间文件引用的单文件大小和总上下文预算按 HarnessKnowledgeFileService 与 agent core 适配配置校验，超限时拒绝本次提交并提示用户减少引用。
- 后端提交前必须重新校验 Agent 功能仍在当前 Harness 配置中可用，且文件引用属于当前团队空间 Harness 知识空间并对当前用户可读。
- 新建对话首条指令、历史继续输入和开放式询问回答都必须包含非空文本；仅选择 Agent 功能或知识文件引用但没有文本时，不允许提交。
- 开放式询问回答长度优先使用 agent core 询问事件返回的限制；未返回时首版默认最多 4000 个字，超出时前端阻止提交且后端返回校验错误。
- 任务创建后永久保留历史，除非后续版本引入管理员治理或合规清理策略。
- 首次指令提交后，系统调用模型生成对话名称；生成失败或模型结果不满足长度规格时不阻塞任务创建。
- 自动生成名称和手动修改名称都必须为 4-10 个字；保存前去除首尾空白。
- 模型生成结果去除首尾空白后，若少于 4 个字或超过 10 个字，则按兜底规则生成名称。
- 兜底名称优先取首条指令前 8 个字；若首条指令不足 4 个字，则使用固定名称“新建对话”。
- 对话创建人可以修改对话名称；手动名称优先于自动生成名称。
- 每次用户提交指令都需要保存该轮指令、可选 Agent 功能选择、可选知识文件引用和对应 Harness 快照；团队空间 Harness 配置后续变化不影响已提交指令对应快照。
- 开放式询问回答不创建新的 AgentTaskTurn；每次开放式回答都生成回答级 Harness 快照并绑定到该次 AgentQuestionAnswer，快照中包含可选 Agent 功能选择和可选知识文件引用。
- Harness 快照需要记录配置内容摘要、来源、版本、Agent 功能选择、知识文件引用和生成时间，敏感环境变量仅传递密钥引用或受控引用，不写入明文。
- AgentSpace 前端不直接调用外部 agent core。
- AgentSpace 后端必须先保存内部任务记录和指令轮次，再生成 Harness 快照并调用外部 agent core，避免快照或外部调用失败后无历史可追踪。
- 任务状态由 AgentSpace 基于 agent core 事件和用户输入状态维护，不直接暴露外部服务内部状态。
- 任务 UI 状态首版为：运行中、待输入、完成静止态、异常静止态。
- 运行中表示 agent core 正在处理当前轮次，或用户回答已回传且任务继续推进。
- 待输入表示 agent core 已发出询问，任务需要创建人回答后继续。
- 完成静止态表示当前任务无待处理运行且最近一轮正常结束。
- 异常静止态表示当前任务因调用失败、认证失败、超时、事件流中断或外部错误停止。
- Agent 询问支持选项型和开放式回答。
- 选项型询问若收到 Agent 功能选择或知识文件引用，后端必须拒绝该回答并提示该询问只能提交选项。
- Agent 询问回答必须使用 answer ID 或客户端幂等键，回答回传失败后重试不得造成重复回答或重复状态推进。
- 外部 agent core 错误需要映射为 AgentSpace 可理解的错误类型，例如认证失败、创建失败、继续失败、超时、事件流中断。
- 任务创建、继续输入、名称生成、名称修改、Agent 功能选择、知识文件引用、询问生成、询问回答和外部服务错误需要记录审计信息。

### 边界与异常

- 用户提交空指令时，前端阻止创建任务或继续输入；后端也必须拒绝空指令请求，并返回校验错误。
- 用户输入 `/` 后未选择 Agent 功能即关闭选择器时，系统不生成结构化功能选择，不阻止用户继续按普通文本提交。
- 用户输入 `@` 后未选择文件即关闭选择器时，系统不生成结构化文件引用，不阻止用户继续按普通文本提交。
- 用户仅选择 Agent 功能或知识文件引用但未填写有效文本时，系统阻止创建、继续输入或开放式回答。
- 用户引用文件超过数量上限、大小上限或总上下文预算时，系统拒绝本次输入并提示用户减少文件引用。
- 用户提交的 Agent 功能不存在、已停用或不属于当前 Harness 配置时，后端拒绝本次输入并提示用户重新选择。
- 用户提交的知识空间文件不存在、已删除、版本不可用、跨团队空间或当前用户无读取权限时，后端拒绝本次输入并提示用户重新选择。
- 对话名称生成失败或生成结果不满足 4-10 个字时，系统使用兜底名称，不阻塞任务创建。
- 手动修改名称为空、少于 4 个字或超过 10 个字时，系统阻止保存并提示校验错误。
- 指令级 Harness 快照生成失败时，系统不调用 agent core，保留内部任务或轮次失败记录，将任务置为异常静止态，并允许创建人修正输入后重试原提交。
- 开放式回答级 Harness 快照生成失败时，系统不回传 agent core，任务保持待输入，记录回答失败事件，并允许创建人修正回答后以同一幂等键重试。
- 外部 agent core 创建任务失败时，系统保留内部任务记录，任务状态为异常静止态。
- 外部 agent core 创建任务失败且未返回 conversation ID 时，系统不允许该异常任务进入继续输入流程，只允许创建人重试首轮创建或重新发起新任务。
- 外部 agent core 继续对话失败时，系统保留本轮用户指令和错误事件，任务状态为异常静止态。
- agent core 返回未知询问格式时，系统记录错误事件，任务进入异常静止态。
- 用户回答回传外部 agent core 失败时，系统记录错误事件，任务保持待输入，并允许对话创建人重试回答。
- 用户重复提交已成功的回答幂等键时，系统返回已有回答结果，不重复回传外部 agent core。
- 非创建人尝试继续输入或回答询问时，后端拒绝并返回权限不足。
- 创建人在任务运行中继续输入，或在待输入状态绕过当前询问提交新指令时，后端拒绝并提示需等待任务进入静止态或先回答询问。
- 事件流中断时，系统记录错误事件，任务进入异常静止态。

## 验收标准

### 正常路径

- 创建者、管理员和团队成员可以打开新建对话主界面。
- 用户输入首条指令并提交后，系统创建 Agent 任务。
- 用户在新建对话、继续输入或开放式询问回答中输入 `/` 时，可以唤起当前 Agent 可用功能选择，并将选中的功能随本次输入保存。
- 用户在新建对话、继续输入或开放式询问回答中输入 `@` 时，可以唤起 Harness 知识空间文件选择，并将选中的文件引用随本次输入保存。
- 系统调用模型根据首次输入生成 4-10 个字的对话名称。
- 创建人可以修改对话名称，修改后的名称仍为 4-10 个字，任务列表和详情展示新名称。
- 系统创建任务时生成包含 Agent 功能选择和知识文件引用的 Harness 快照，并通过 AgentCoreAdapter 传递给外部 agent core。
- 历史对话永久保留，任务处于完成静止态或异常静止态且已有外部会话 ID 时，创建人重新进入历史对话后可以继续输入新指令。
- 创建人继续输入时，系统创建新轮次、新 Harness 快照，并将新指令、历史上下文、Agent 功能选择和知识文件引用传递给外部 agent core。
- agent core 返回选项型询问时，任务进入待输入，创建人可以提交选项回答。
- agent core 返回开放式询问时，任务进入待输入，创建人可以提交文本回答。
- 开放式询问回答提交时，系统生成回答级 Harness 快照；若回答携带 Agent 功能选择或知识文件引用，快照需一并固定这些结构化选择。
- 创建人回答后，AgentSpace 将回答回传给外部 agent core，任务重新进入运行中。
- AgentSpace 接收 agent core 完成事件后，任务进入完成静止态。
- AgentSpace 接收 agent core 错误事件、超时或事件流中断后，任务进入异常静止态。

### 边界场景

- 自动生成名称失败时，系统使用首条指令前 8 个字作为兜底名称。
- 模型生成名称少于 4 个字或超过 10 个字时，系统按兜底规则生成名称。
- 首条指令不足 4 个字且名称生成失败时，系统使用“新建对话”作为兜底名称。
- 用户取消 `/` 功能选择或 `@` 文件选择时，系统只按普通文本处理本次输入。
- 用户未填写有效文本，仅选择 Agent 功能或知识文件引用时，系统不允许提交。
- 用户引用文件超过首版 10 个文件上限、单文件大小上限或总上下文预算时，系统提示减少文件引用。
- 团队空间 Harness 配置在某轮指令提交后变化时，已提交轮次仍使用该轮创建时快照。
- 团队空间 Harness 功能目录或知识文件在某轮输入提交后变化时，已提交轮次仍保留提交时的功能选择和文件版本引用。
- 任务进入完成静止态或异常静止态且已有外部会话 ID 后，创建人仍可回到历史对话继续输入新指令。
- 用户回答回传外部 agent core 失败时，任务保持待输入，创建人可以重试回答。

### 异常场景

- 用户提交空指令时，系统阻止创建任务或继续输入。
- 用户绕过前端提交空指令时，后端拒绝请求并返回校验错误。
- 用户提交空开放式询问回答时，系统阻止回答提交。
- 用户提交超过长度限制的开放式询问回答时，系统阻止回答提交。
- 选项型询问回答携带 Agent 功能选择或知识文件引用时，系统拒绝回答提交。
- 任务运行中或待输入状态下，创建人绕过前端提交继续输入时，后端拒绝请求。
- 用户提交空对话名称、少于 4 个字或超过 10 个字的名称时，系统阻止保存。
- 用户提交已停用、不存在或不属于当前 Harness 配置的 Agent 功能时，系统阻止本次创建、继续输入或开放式回答。
- 用户提交不存在、已删除、跨团队空间或无读取权限的知识文件引用时，系统阻止本次创建、继续输入或开放式回答。
- 指令级 Harness 快照生成失败时，系统保留内部失败记录，任务进入异常静止态，且不调用外部 agent core。
- 开放式回答级 Harness 快照生成失败时，系统保留回答失败事件，任务保持待输入，且不回传外部 agent core。
- agent core 返回未知询问格式时，系统记录错误事件，状态为异常静止态。
- 外部 agent core 创建任务失败且未返回 conversation ID 时，系统保留失败任务记录，状态为异常静止态，且不允许按历史会话继续输入。
- 外部 agent core 继续对话失败时，系统保留本轮失败记录和错误事件，状态为异常静止态。
- 外部 agent core 认证失败、超时或事件流中断时，系统记录错误事件，状态为异常静止态。

### 权限场景

- 访客不能新建 Agent 任务。
- 非任务创建人不能在该历史对话中继续输入指令。
- 非任务创建人不能修改对话名称。
- 非任务创建人不能回答该任务的 Agent 询问。
- 未加入团队空间且未被授权的用户不能创建任务。
- 用户不能通过 `@` 选择或提交自己无读取权限的 Harness 知识空间文件。
- 任务创建、继续输入、名称生成、名称修改、Agent 功能选择、知识文件引用和询问回答写入审计日志。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：需要新增团队空间 Agent 对话创建入口、对话主界面、运行状态提示、继续输入、`/` Agent 功能选择、`@` 知识文件引用、对话名称编辑和 Agent 询问回答控件。

### 页面与交互

- 团队空间导航展示 Agent 对话入口。
- 创建者、管理员和团队成员展示新建对话主操作；访客不展示新建入口。

## 后端设计

### 是否需要刷新

- 结论：是

### 接口与数据

- 核心服务：
  - AgentTaskService：创建任务、继续历史任务、维护任务状态和任务权限。
  - AgentFunctionCatalogService：读取当前团队空间 Harness 中可用的 Agent 功能目录，并校验功能可用性。
  - HarnessKnowledgeFileService：搜索当前团队空间 Harness 知识空间文件，并校验文件归属、版本和读取权限。
  - ConversationTitleService：调用模型生成首条指令摘要作为对话名称，处理兜底名称和用户手动改名。
  - AgentCoreAdapter：封装外部 agent core 的认证、任务创建、继续对话、事件接收、用户回答回传、Agent 功能选择映射、知识文件引用映射和错误映射。
  - HarnessSnapshotService：为每轮用户指令和开放式询问回答生成包含 Agent 功能选择和知识文件引用的 Harness 配置快照。
  - AgentQuestionService：管理询问、回答状态和回答回传。
  - AgentTaskEventService：保存任务创建、用户指令、状态、询问和错误事件。
  - PermissionService：判断任务创建、继续输入、名称修改和询问回答权限。
  - AuditService：记录任务创建、继续输入、名称生成、名称修改、Agent 功能选择、知识文件引用、回答和外部服务异常。
- 核心数据对象：
  - AgentTask：任务 ID、租户 ID、团队空间 ID、创建人 ID、对话名称、名称来源、首条指令摘要、状态、agent core conversation ID、错误摘要、创建时间、更新时间。
  - ConversationTitleSource：模型生成、兜底生成、用户修改。
  - AgentTaskRetentionPolicy：永久保留。
  - AgentTaskTurn：轮次 ID、任务 ID、用户指令、Agent 功能选择、知识文件引用、Harness 快照 ID、agent core run ID、状态、创建时间、更新时间。
  - AgentTaskStatus：运行中、待输入、完成静止态、异常静止态。
  - HarnessSnapshot：快照 ID、团队空间 ID、配置摘要、配置来源、配置版本、Agent 功能选择、知识文件引用、敏感引用、生成时间。
  - AgentFunctionSelection：功能 ID、功能名称、功能版本或来源、选择来源、创建时间。
  - HarnessKnowledgeFileReference：知识空间 ID、文件 ID、文件路径或展示名、文件版本、引用来源、创建时间。
  - AgentQuestion：询问 ID、任务 ID、轮次 ID、事件 ID、问题类型、题干、选项、状态、创建时间、回答时间。
  - AgentQuestionAnswer：回答 ID、询问 ID、客户端幂等键、回答内容、Agent 功能选择、知识文件引用、回答级 Harness 快照 ID、回传状态、创建时间、更新时间。
  - AgentQuestionType：选项型、开放式。
- 建议接口：
  - `POST /team-spaces/{spaceId}/agent-tasks`
  - `POST /team-spaces/{spaceId}/agent-tasks/{taskId}/turns`
  - `GET /team-spaces/{spaceId}/agent-functions`
  - `GET /team-spaces/{spaceId}/harness-knowledge-files`
  - `PATCH /team-spaces/{spaceId}/agent-tasks/{taskId}/title`
  - `POST /team-spaces/{spaceId}/agent-tasks/{taskId}/answers`

### 业务规则、权限与异常处理

- 创建任务前必须校验用户角色为创建者、管理员或团队成员。
- 继续输入、更新对话名称和回答询问前必须校验操作者是任务创建人。
- 查询 Agent 功能目录和 Harness 知识空间文件前必须校验用户具备当前团队空间访问权限。
- 创建任务、继续输入和开放式询问回答前必须校验 Agent 功能选择仍可用、知识文件引用仍属于当前团队空间且对用户可读。
- 继续输入前必须校验任务处于完成静止态或异常静止态且已存在外部 agent core conversation ID。
- 开放式询问回答前必须校验回答文本非空；选项型询问不得接收 Agent 功能选择和知识文件引用。
- 回答询问前必须校验回答幂等键；重复提交已成功回答时不得重复回传外部 agent core。
- 任务创建和继续输入都必须在调用外部 agent core 前落库。
- 每轮用户指令都必须关联独立 Harness 快照，快照需要固定本轮的 Agent 功能选择和知识文件版本引用。
- 对话名称生成失败时不得阻断任务创建，必须按兜底规则生成可落库名称。
- 外部 agent core 的认证失败、创建失败、继续对话失败、超时和事件流中断必须映射为内部错误事件。
- 任务、轮次、询问、Agent 功能选择、知识文件引用和错误事件必须继承租户和团队空间上下文。
