  - SSE 在目标网关和浏览器环境中的支持情况；若不支持，事件流语义保持不变并迁移为 WebSocket。

## PRD 设计

### 用户目标

- 团队成员可以在团队空间中打开对话主界面，输入指令并发起 Agent 任务。
- 团队成员无需手动配置运行参数，系统自动使用团队空间当前 Harness 配置生成任务级快照并传递给外部 agent core。
- 团队成员可以在任务历史看板中查看团队空间内自己有权查看的任务，包括创建人、状态和最近更新时间。
- 对话创建人可以在 Agent 等待输入时回答询问，并确认 Agent 生成或修改的文档 diff。
- 团队管理员和访客可以只读查看公开任务过程和结果，便于团队协作、审计和知识复用。

### 范围

**包含：**

- 团队空间内新增 Agent 对话入口。
- 创建者、管理员和团队成员可以新建 Agent 对话任务；访客不可新建任务。
- 一个对话对应一个 Agent 任务。
- 新建对话时支持输入用户指令，并选择公开或私密可见性；默认公开。
- AgentSpace 后端读取当前团队空间 Harness 配置并生成任务级快照，连同用户指令、团队空间标识和创建人标识传递给外部 agent core。
- AgentSpace 后端通过适配层调用外部 agent core，并接收 thinking、output、question、document_change、status、error 等事件。
- AgentSpace 内部保存任务、事件、询问和文档变更提案，不依赖外部 agent core 作为唯一历史存储。
- 公开对话对团队空间创建者、管理员、团队成员和访客可见，但除创建人外不可继续输入指令。
- 私密对话仅创建人可见。
- 团队空间任务历史看板展示可见任务清单，标识创建人、任务状态、最新更新时间和可见性。
- 任务状态首版包括模型生成中、等待用户输入、静止。
- 对话详情按轮次展示 agent core 返回的可展示 thinking、output、询问和文档变更。
- Agent 询问支持选项型和开放式回答；首版仅对话创建人可回答。
- Agent 生成或修改文档时先形成待接受文档变更提案，支持查看内容、查看 diff、接受和拒绝。
- 前端不直接调用外部 agent core，只调用 AgentSpace 后端。

**不包含：**

- 外部 agent core 服务自身的模型调度、运行沙箱、执行编排、内部状态机和任务队列实现。
- 多 Agent 协同、复杂任务编排、任务优先级、运行资源配额和调度策略。
- 多名用户共同输入同一个对话。
- 对话模板、提示词市场、自动任务推荐和任务评分。
- 文档 diff 的复杂三方合并；首版仅处理基于创建提案时 base version 的接受或拒绝。
- 访客新建任务、回答询问、处理文档 diff 或触发任何写操作。

### 核心流程

1. 用户进入团队空间，系统根据 devuc 和团队空间 RBAC 判断其是否具备访问权限。
2. 创建者、管理员或团队成员点击新建 Agent 对话入口，Portal 弹出对话主界面。
3. 用户输入首条指令，选择公开或私密；若不选择，系统按公开处理。
4. 用户提交后，前端调用 AgentSpace 后端创建 Agent 任务。
5. 后端校验用户对团队空间的访问权限和新建任务权限。
6. 后端读取当前团队空间 Harness 配置，生成任务级 `HarnessSnapshot`。
7. 后端创建 `AgentTask`，记录团队空间、租户、创建人、指令摘要、可见性、状态和 Harness 快照。
8. `AgentCoreAdapter` 调用外部 agent core 创建任务，传递用户指令、团队空间标识、创建人标识和 Harness 快照。
9. 创建成功后，任务进入模型生成中，前端通过 AgentSpace 的 SSE 事件流接收任务事件。
10. AgentSpace 后端接收 agent core 事件，转换并保存为 `AgentTaskEvent`。
11. 前端在对话详情中按轮次展示可展示 thinking、output、询问和文档变更。
12. 当 agent core 发出询问事件时，任务进入等待用户输入。
13. 对话创建人提交选项型或开放式回答，AgentSpace 将回答回传给外部 agent core，任务重新进入模型生成中。
14. 当 agent core 返回文档生成或修改事件时，AgentSpace 创建 `DocumentChangeProposal`，状态为待接受。
15. 对话创建人查看文档内容或 diff 后，可以接受或拒绝该变更。
16. 接受后，系统将提案写入正式知识库文档并更新提案状态；拒绝后，系统保持修改前版本并标记提案已拒绝。
17. 任务完成、失败、超时或事件流中断后进入静止状态，并在对话详情中展示最终结果或错误说明。

### 业务规则

- Agent 任务必须归属一个团队空间，并继承团队空间租户上下文。
- 创建者、管理员和团队成员可以新建 Agent 任务；访客只读查看公开任务。
- 公开任务对创建者、管理员、团队成员和访客可见。
- 私密任务仅创建人可见；管理员第一版不默认穿透查看私密任务内容。
- 首版一个任务只有创建人可以继续输入、回答询问、接受或拒绝文档变更。
- 任务创建后需要保存创建时 Harness 配置快照；后续团队空间 Harness 配置变化不影响已创建任务。
- Harness 快照需要记录配置内容摘要、来源、版本和生成时间，敏感环境变量仅传递密钥引用或受控引用，不写入明文。
- AgentSpace 只展示 agent core 返回且标记为可展示的 thinking 内容。
- 任务状态由 AgentSpace 基于 agent core 事件和用户输入状态维护，不直接暴露外部服务内部状态。
- 模型生成中表示任务已提交外部 agent core，且尚未等待用户输入或结束。
- 等待用户输入表示 agent core 已发出询问，任务需要创建人提交回答后继续。
- 静止表示任务已完成、失败、被中断、超时，或当前没有继续运行动作。
- 任务历史看板的最新更新时间取任务、事件、询问回答或文档提案状态的最近更新时间。
- 相对更新时间文案由前端渲染，例如 1 分钟、6 天。
- 文档变更提案未接受前不得覆盖正式文档。
- 接受文档变更前需要校验正式文档仍处于提案的 base version；若版本不一致，阻止接受并提示冲突。
- 拒绝文档变更不会写入正式文档，只更新提案状态。
- 外部 agent core 调用失败、超时、认证失败或事件解析失败时，任务进入静止并记录错误事件。
- 所有任务创建、询问回答、文档变更接受或拒绝、外部服务错误都需要记录审计信息。

### 边界与异常

- 用户提交空指令时，前端阻止创建任务并提示输入指令。
- 当前团队空间没有可用 Harness 配置时，仍可创建任务，但 Harness 快照标记为空配置或默认配置，具体是否允许由后端配置开关控制。
- Harness 快照生成失败时，任务不提交给 agent core，并提示创建失败。
- 外部 agent core 创建任务失败时，AgentTask 保留失败记录，状态为静止。
- 外部 agent core 事件流中断时，任务状态置为静止，并提示事件流中断，允许后续按技术能力重连或重新打开详情查看已保存事件。
- agent core 返回未知事件类型时，后端保存原始错误摘要，并在前端展示无法识别的事件提示。
- agent core 返回的 thinking 未标记为可展示时，前端不展示 thinking 内容。
- 非创建人尝试回答询问时，后端拒绝并返回权限不足。
- 非创建人尝试接受或拒绝文档变更时，后端拒绝并返回权限不足。
- 私密任务被非创建人访问时，系统按无权限处理，不暴露任务标题、事件或文档内容。
- 访客访问公开任务详情时，只读展示任务内容，不展示输入框、回答控件、接受和拒绝按钮。
- 文档提案接受时发现 base version 已变化，系统阻止写入并提示文档已被更新。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：本功能新增团队空间 Agent 对话入口、对话主界面、任务历史看板、对话详情、Agent 询问、文档内容和 diff 交互。

### 页面与交互

- 团队空间导航：
  - 新增 Agent 任务或对话入口。
  - 创建者、管理员和团队成员展示新建对话主操作。
  - 访客仅展示任务历史入口，不展示新建对话操作。
- 新建对话主界面：
  - 以弹窗或抽屉形式打开，创建成功后进入同一对话详情。
  - 包含指令输入框、公开/私密可见性选择、提交按钮和取消入口。
  - 默认可见性为公开。
- 任务历史看板：
  - 展示任务标题或首条指令摘要、创建人、状态、最新更新时间、公开或私密标识。
  - 支持点击进入任务详情。
  - 私密任务仅创建人可见。
  - 状态标签使用模型生成中、等待用户输入、静止。
- 对话详情：
  - 顶部展示任务标题、创建人、可见性、状态和最新更新时间。
  - 主体按轮次展示用户指令、可展示 thinking、output、询问和文档变更。
  - 当前等待用户输入时，在详情底部展示回答区。
- Agent 询问：
  - 选项型询问展示题干和单选或多选选项，具体选择类型以 agent core 事件为准。
  - 开放式询问展示题干和文本输入框。
  - 非创建人和访客只读查看题干和等待状态，不展示可提交控件。
- 文档变更：
  - 文档卡片展示文档名称、路径、变更类型、提案状态和操作入口。
  - 支持查看内容和查看 diff。
  - 待接受状态下，创建人可点击接受或拒绝。
  - 已接受或已拒绝状态展示结果，不再展示提交类按钮。

### 状态与文案

- 空状态：当前空间暂无 Agent 任务、暂无可查看任务、当前对话暂无输出。
- 加载态：任务创建中、模型生成中、事件连接中、文档 diff 加载中、回答提交中。
- 成功态：任务已创建、回答已提交、文档变更已接受、文档变更已拒绝。
- 等待态：Agent 等待用户输入、文档变更等待确认。
- 异常态：任务创建失败、agent core 调用失败、事件流中断、Harness 快照生成失败、文档版本冲突、权限不足。
- 只读态：当前用户仅可查看，不能输入、回答或处理文档变更。

- 依赖接口：
  - 创建 Agent 任务。
  - 查询团队空间 Agent 任务列表。
  - 查询 Agent 任务详情。
  - 订阅 Agent 任务事件流，首版采用 SSE。

  - 提交 Agent 询问回答。
  - 查询文档变更提案内容。
  - 查询文档变更 diff。
  - 接受文档变更提案。
  - 拒绝文档变更提案。
- 异常处理：
  - 创建任务失败时保留用户输入，并展示失败原因。
  - SSE 断线时展示连接中断状态，并拉取已保存事件补齐详情。
  - 接口返回权限不足时，前端切换为只读态或展示无权限提示。
  - 文档 diff 加载失败时保留文档卡片，并允许重试加载。
  - 文档变更接受时发生版本冲突，前端展示冲突提示并刷新提案状态。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：后端需要新增 Agent 任务模型、外部 agent core 适配层、Harness 快照、事件存储、询问回答回传、文档变更提案和任务权限控制。

### 接口与数据

- 核心服务：
  - AgentTaskService：创建任务、查询任务、维护任务状态和任务权限。
  - AgentCoreAdapter：封装外部 agent core 的认证、任务创建、事件接收、用户回答回传和错误映射。
  - HarnessSnapshotService：根据团队空间当前 Harness 配置生成任务级快照。
  - AgentTaskEventService：保存、查询和转发任务事件。
  - AgentQuestionService：管理 Agent 询问、回答状态和回答回传。
  - DocumentChangeProposalService：管理文档变更提案、diff、接受和拒绝。
  - TaskEventStreamService：向前端提供 SSE 事件流。
  - PermissionService：复用团队空间 RBAC，判断任务查看、创建、回答和文档变更处理权限。
  - AuditService：记录任务创建、回答、文档提案处理和外部服务异常。
- 核心数据对象：
  - AgentTask：任务 ID、租户 ID、团队空间 ID、创建人 ID、标题、首条指令摘要、可见性、状态、Harness 快照 ID、agent core run ID、错误摘要、创建时间、更新时间。
  - AgentTaskVisibility：公开、私密。
  - AgentTaskStatus：模型生成中、等待用户输入、静止。
  - HarnessSnapshot：快照 ID、团队空间 ID、配置摘要、配置来源、配置版本、敏感引用、生成时间。
  - AgentTaskEvent：事件 ID、任务 ID、轮次、事件类型、可展示内容、原始内容引用、创建时间。
  - AgentTaskEventType：user_instruction、thinking、output、question、document_change、status、error。
  - AgentQuestion：询问 ID、任务 ID、事件 ID、问题类型、题干、选项、回答内容、状态、创建时间、回答时间。
  - AgentQuestionType：选项型、开放式。
  - DocumentChangeProposal：提案 ID、任务 ID、事件 ID、文档 ID 或路径、变更类型、base version、proposed version、diff 引用、状态、创建时间、更新时间。
  - DocumentChangeProposalStatus：待接受、已接受、已拒绝、冲突。
- 建议接口：
  - `POST /team-spaces/{spaceId}/agent-tasks`
  - `GET /team-spaces/{spaceId}/agent-tasks`
  - `GET /team-spaces/{spaceId}/agent-tasks/{taskId}`
  - `GET /team-spaces/{spaceId}/agent-tasks/{taskId}/events`
  - `GET /team-spaces/{spaceId}/agent-tasks/{taskId}/stream`
  - `POST /team-spaces/{spaceId}/agent-tasks/{taskId}/answers`
  - `GET /team-spaces/{spaceId}/document-change-proposals/{proposalId}`
  - `GET /team-spaces/{spaceId}/document-change-proposals/{proposalId}/diff`
  - `POST /team-spaces/{spaceId}/document-change-proposals/{proposalId}/accept`
  - `POST /team-spaces/{spaceId}/document-change-proposals/{proposalId}/reject`

### 业务规则、权限与异常处理

- 任务创建前必须校验用户具备团队空间访问权限，且角色为创建者、管理员或团队成员。
- 查询任务列表时，公开任务对团队空间内创建者、管理员、团队成员和访客可见；私密任务仅创建人可见。
- 查询任务详情、事件、文档提案和 diff 时必须复用任务可见性规则。
- 回答询问、接受文档变更、拒绝文档变更必须校验操作者是任务创建人。
- AgentTaskService 创建任务后，必须先保存内部任务记录，再调用外部 agent core，避免外部调用失败后无历史可追踪。
- AgentCoreAdapter 必须将外部错误映射为 AgentSpace 可理解的错误类型，例如认证失败、创建失败、超时、事件解析失败、事件流中断。
- 接收 agent core 事件后，后端必须先做事件类型和权限安全处理，再向前端转发。
- thinking 事件只有在 agent core 明确标记可展示时才写入可展示内容。
- 事件流中断不删除已保存事件，任务进入静止，并记录错误事件。
- 文档变更提案接受前必须校验 base version；冲突时不写入正式文档。
- 文档变更拒绝只更新提案状态，不修改正式文档。
- 任务、事件、询问和文档提案需要继承租户和团队空间上下文，用于权限、审计和后续清理。

## 验收标准

### 正常路径

- 创建者、管理员和团队成员可以在团队空间内打开新建对话主界面。
- 用户输入指令并提交后，系统创建 Agent 任务，并记录团队空间、租户、创建人和可见性。
- 系统创建任务时生成 Harness 配置快照，并通过 AgentCoreAdapter 传递给外部 agent core。
- 公开任务出现在团队空间任务历史看板中，并展示创建人、状态、最新更新时间和公开标识。
- 私密任务仅创建人可以在任务历史看板和详情中看到。
- 对话详情可以展示 agent core 返回的可展示 thinking、output 和状态变化。
- agent core 返回选项型询问时，任务进入等待用户输入，创建人可以选择选项并提交回答。
- agent core 返回开放式询问时，任务进入等待用户输入，创建人可以填写文本并提交回答。
- agent core 返回文档变更时，系统创建待接受文档变更提案。
- 创建人可以查看文档内容和 diff。
- 创建人接受文档变更后，正式文档更新为 proposed version，提案状态变为已接受。
- 创建人拒绝文档变更后，正式文档保持 base version，提案状态变为已拒绝。

### 边界场景

- 用户不选择可见性时，任务默认公开。
- 团队空间 Harness 配置在任务创建后发生变化时，已创建任务仍使用创建时快照。
- 任务没有 thinking 事件时，对话详情仍可正常展示 output、询问或文档事件。
- agent core 返回 thinking 但未标记可展示时，前端不展示 thinking 内容。
- 任务完成、失败、超时或事件流中断后，状态展示为静止。
- 相对更新时间能随任务事件、回答或文档提案状态变化刷新。

### 异常场景

- 用户提交空指令时，系统阻止创建任务。
- Harness 快照生成失败时，系统不调用 agent core，并提示任务创建失败。
- 外部 agent core 创建任务失败时，系统保留失败任务记录，状态为静止。
- 外部 agent core 认证失败、超时或事件流中断时，系统记录错误事件并展示错误状态。
- agent core 返回未知事件类型时，系统不影响已知事件展示，并记录无法识别事件。
- 文档变更接受时发现 base version 已变化，系统阻止写入并提示文档冲突。

### 权限场景

- 访客可以只读查看公开任务清单和详情。
- 访客不能新建任务、继续输入、回答询问、接受或拒绝文档变更。
- 非任务创建人不能回答该任务的 Agent 询问。
- 非任务创建人不能接受或拒绝该任务的文档变更。
- 非创建人访问私密任务时，系统拒绝访问且不暴露任务内容。
- 未加入团队空间且未被授权的用户不能查看任务清单或任务详情。
- 任务创建、询问回答、文档变更接受或拒绝都写入审计日志。
