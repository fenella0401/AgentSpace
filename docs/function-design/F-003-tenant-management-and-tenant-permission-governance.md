# F-003-租户管理与租户权限治理 功能设计

## 基本信息

| 字段 | 内容 |
| --- | --- |
| 功能编号 | F-003 |
| 功能名称 | 租户管理与租户权限治理 |
| 所属功能树节点 | AgentSpace / 租户与管理面 |
| 关联版本 | R-0630 |
| 状态 | 草稿 |
| 最近更新 | 2026-06-04 |

## 变更摘要

- 本次变更：新增全局和租户两级管理面，定义系统管理员创建租户、授予租户管理员、租户管理员治理本租户配置与团队空间清单的能力。
- 影响范围：产品定义、功能索引、租户数据模型、团队空间租户归属、租户级权限、管理面导航、后端鉴权、审计和 R-0630 发布范围。
- 待确认问题：
  - 租户编码、名称唯一性、启停状态和归属组织字段规则。
  - devuc 中系统管理员、租户管理员身份来源和同步方式。
  - 历史未归属团队空间迁移策略；本功能首期只要求新建团队空间必须归属租户。
  - 租户管理员是否需要进入团队空间内容的只读穿透权限；第一版暂按不默认拥有处理。
  - 租户级配置首期具体范围；本设计先覆盖管理面、租户管理员和团队空间清单。

## PRD 设计

### 用户目标

- 系统管理员可以在全局管理面创建租户，并维护租户基础信息和启停状态。
- 系统管理员可以授予或撤销租户管理员，明确租户治理责任人。
- 租户管理员可以进入租户管理面，查看本租户团队空间清单并维护租户级配置。
- 团队空间创建和后续治理都具备明确租户归属，便于权限隔离、审计和后续模板应用。
- 平台可以在全局、租户和团队空间三个层级上执行一致的鉴权与审计。

### 范围

**包含：**

- 全局管理面入口，仅系统管理员可访问。
- 租户列表、租户详情、租户创建、租户基础信息维护和租户启停状态。
- 系统管理员授予、撤销租户管理员。
- 租户管理面入口，仅对应租户管理员可访问。
- 租户管理员查看本租户团队空间清单。
- 新建团队空间必须关联租户，形成 `TeamSpace.tenantId`。
- 租户级权限判断和管理操作审计。
- 后端对全局管理、租户管理和团队空间创建执行权限校验。

**不包含：**

- 租户计费、套餐、容量、用量统计和商业化治理。
- 租户合并、拆分、迁移和删除后的数据清理流程。
- 历史未归属团队空间迁移实施方案。
- 租户级自定义角色和自定义权限点。
- 租户管理员自动继承团队空间管理员权限。
- 租户级团队模板的完整配置细节，详见 F-004。
- devuc 服务自身的账号、组织和权限主数据治理。

### 核心流程

1. 系统管理员进入 AgentSpace 全局管理面。
2. 系统管理员创建租户，填写租户名称、租户编码、归属组织等基础信息。
3. 系统校验租户编码和名称唯一性，创建 `Tenant` 并记录审计。
4. 系统管理员在租户详情中搜索用户，并授予租户管理员。
5. 系统写入 `TenantAdminGrant`，记录授权来源、操作者和生效状态。
6. 被授权用户进入 AgentSpace 后，可以看到对应租户管理面入口。
7. 租户管理员进入租户管理面，查看租户基础信息、租户团队空间清单和租户级配置入口。
8. 团队管理员创建团队空间时，必须选择或继承一个可用租户。
9. 系统创建团队空间时写入 `TeamSpace.tenantId`，后续成员、知识库、Harness 配置和审计均继承该租户上下文。
10. 系统管理员撤销租户管理员授权后，该用户再次访问租户管理面时权限失效。

### 业务规则

- 系统管理员具备全局管理面访问权限。
- 只有系统管理员可以创建租户、启停租户和授予或撤销租户管理员。
- 租户管理员仅能管理被授权租户，不得访问其他租户的管理面。
- 租户管理员可以查看本租户团队空间清单，但不默认拥有团队空间内容写权限。
- 租户管理员若需要管理具体团队空间，必须被授予该团队空间创建者或管理员角色。
- 团队空间必须归属一个有效租户；租户停用后不允许在该租户下新建团队空间。
- 同一用户可以管理多个租户，每个授权关系需要独立记录。
- 租户管理员授权和撤销必须记录审计，至少包含租户、操作者、目标用户、动作、时间和结果。
- 全局管理和租户管理操作必须以后端鉴权结果为准，前端权限态只作为体验优化。
- `ManagementPlaneScope` 用于区分全局管理面和租户管理面，避免接口在错误上下文中执行。

### 权限矩阵

| 权限项 | 系统管理员 | 租户管理员 | 团队管理员 |
| --- | --- | --- | --- |
| 访问全局管理面 | 是 | 否 | 否 |
| 创建和启停租户 | 是 | 否 | 否 |
| 授予或撤销租户管理员 | 是 | 否 | 否 |
| 访问被授权租户管理面 | 否 | 是 | 否 |
| 查看本租户团队空间清单 | 是 | 是 | 否 |
| 在有权限租户下创建团队空间 | 是 | 是 | 是 |
| 管理具体团队空间成员与角色 | 否 | 否 | 是 |
| 管理具体团队空间知识库和 Harness | 否 | 否 | 是 |
| 维护全局团队模板 | 是 | 否 | 否 |
| 维护租户级团队模板 | 否 | 是 | 否 |

### 边界与异常

- 租户名称或编码重复时，系统阻止保存并提示冲突字段。
- 租户创建失败时，系统保留已填写内容并提示失败原因。
- 租户停用后，租户管理员不可再新增团队空间，但历史空间访问策略待后续确认。
- 系统管理员撤销租户管理员后，该用户已打开的租户管理面应在刷新或下一次接口调用时失效。
- 租户管理员访问未授权租户管理面时，系统拒绝访问。
- 租户管理员尝试编辑未被授权的团队空间内容时，系统拒绝操作。
- devuc 鉴权失败、超时或权限数据缺失时，默认拒绝管理类写操作。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：新增全局管理面、租户管理面、租户列表、租户详情、租户管理员授权和团队空间租户归属展示。

### 页面与交互

- 全局管理面：
  - 租户列表展示租户名称、编码、状态、管理员数量、团队空间数量、创建时间和操作入口。
  - 新建租户表单包含租户名称、租户编码、归属组织、描述和状态。
  - 租户详情展示基础信息、租户管理员、团队空间清单摘要和审计入口。
  - 系统管理员可搜索用户并授予或撤销租户管理员。
- 租户管理面：
  - 顶部展示当前租户名称、编码和状态。
  - 团队空间清单展示空间名称、创建者、项目编号、成员数量、更新时间和状态。
  - 租户级配置和租户级团队模板入口可展示为待完善入口，模板细节由 F-004 承接。
- 团队空间创建入口：
  - 从全局或租户管理面进入时，所属租户由入口上下文带入。
  - 从普通空间创建入口进入时，若用户有多个可用租户，需要选择租户。

### 状态与文案

- 空状态：暂无租户、暂无租户管理员、当前租户暂无团队空间。
- 加载态：租户创建中、租户管理员授权中、权限校验中、租户团队空间加载中。
- 成功态：租户已创建、租户信息已更新、租户管理员已授权、租户管理员已撤销。
- 异常态：无全局管理权限、无租户管理权限、租户重复、租户已停用、devuc 鉴权失败。
- 权限态：系统管理员展示全局操作；租户管理员只展示被授权租户的管理入口。

## 前端设计

### 是否需要刷新

- 结论：是
- 理由：需要新增管理面路由、租户上下文、授权组件、团队空间租户选择和权限守卫。

### 页面、组件与状态

- 页面路由：
  - 全局租户管理页。
  - 租户详情页。
  - 租户管理面首页。
  - 租户团队空间清单页。
- 核心组件：
  - TenantList：租户列表。
  - TenantCreateDialog：创建租户弹窗或表单。
  - TenantDetailPanel：租户基础信息和状态展示。
  - TenantAdminGrantEditor：租户管理员搜索、授权和撤销。
  - TenantSpaceList：租户团队空间清单。
  - ManagementScopeGuard：管理面权限守卫。
  - TenantScopeBadge：展示当前页面租户上下文。
- 状态管理：
  - 当前管理面范围：`global` 或 `tenant`。
  - 当前用户系统管理员权限。
  - 当前用户可管理租户列表。
  - 当前租户详情、管理员授权列表和团队空间清单。
  - 租户创建、编辑、启停和授权提交状态。

### 接口依赖与异常处理

- 依赖接口：
  - 查询当前用户管理面权限。
  - 查询租户列表和租户详情。
  - 创建和更新租户。
  - 启用或停用租户。
  - 查询、授予、撤销租户管理员。
  - 查询租户团队空间清单。
  - 查询当前用户可创建团队空间的租户列表。
- 异常处理：
  - 管理面权限不足时跳转无权限页或展示权限不足提示。
  - 租户创建和编辑失败时保留表单内容。
  - 授权失败时保留搜索结果和已选用户。
  - 接口返回租户停用时，前端禁用新增团队空间入口。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：需要新增租户模型、租户管理员授权关系、管理面权限判断、团队空间租户归属和管理操作审计。

### 接口与数据

- 核心服务：
  - TenantService：创建、查询、更新、启停租户。
  - TenantAdminService：管理租户管理员授权和撤销。
  - ManagementPermissionService：判断全局管理面和租户管理面权限。
  - TenantSpaceQueryService：查询租户下团队空间清单。
  - DevucAuthAdapter：对接公司 devuc 服务完成身份和权限鉴权。
  - AuditService：记录租户创建、租户管理员授权、撤销和管理面拒绝操作。
- 核心数据对象：
  - Tenant：租户 ID、租户名称、租户编码、归属组织、状态、描述、创建时间、更新时间。
  - TenantAdminGrant：租户 ID、用户 ID、授权状态、授权来源、授权人、生效时间、撤销时间。
  - TeamSpace.tenantId：团队空间所属租户 ID，新建团队空间必填。
  - ManagementPlaneScope：管理面范围，取值为 `global` 或 `tenant`。
  - TenantAuditLog：租户 ID、操作者、目标对象、操作类型、变更前后内容、结果、时间。

### 业务规则、权限与异常处理

- 系统管理员身份由 devuc 或平台配置提供，后端统一判断。
- 所有全局管理接口必须校验系统管理员权限。
- 所有租户管理接口必须校验当前用户是否为该租户管理员。
- 租户编码第一版按全局唯一处理，是否支持后续修改待确认。
- 租户停用后，禁止新建团队空间和新增租户管理员；历史空间访问策略暂不在本功能内展开。
- 授权租户管理员时需要校验目标用户存在且可被 devuc 识别。
- 撤销租户管理员不能删除审计记录，应更新授权状态和撤销时间。
- 创建团队空间时必须传入有效租户 ID，且租户处于可用状态。
- 鉴权失败、租户不存在、租户停用或授权状态异常时，默认拒绝写操作。

## 验收标准

### 正常路径

- 系统管理员可以进入全局管理面并查看租户列表。
- 系统管理员可以创建租户，租户保存后出现在租户列表中。
- 系统管理员可以在租户详情中授予租户管理员。
- 被授权用户可以进入对应租户管理面。
- 租户管理员可以查看本租户团队空间清单。
- 团队空间创建时必须写入有效租户 ID。
- 系统管理员可以撤销租户管理员，撤销后该用户不再能进入对应租户管理面。

### 边界场景

- 同一用户可以被授予多个租户的租户管理员。
- 租户管理员只看到自己被授权的租户管理入口。
- 从租户管理面创建团队空间时，租户由入口上下文带入。
- 从普通创建入口创建团队空间时，有多个可用租户的用户需要选择租户。

### 异常场景

- 非系统管理员访问全局管理面时，系统拒绝访问。
- 租户名称或编码重复时，系统阻止创建并提示冲突。
- 租户停用后，系统阻止在该租户下创建团队空间。
- 租户管理员访问未授权租户时，系统拒绝访问。
- devuc 鉴权失败时，系统拒绝租户管理写操作。

### 权限场景

- 租户管理员不能创建租户、启停租户或授权其他租户管理员。
- 租户管理员未被授予团队空间管理员角色时，不能编辑该团队空间成员、知识库和 Harness 配置。
- 团队管理员不能访问全局管理面或租户管理面。
- 租户创建、租户管理员授权、撤销和权限拒绝均写入审计日志。
