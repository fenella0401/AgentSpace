## 变更摘要

- 本次变更：按功能设计 PRD、UI、前端、后端与验收剧本重写租户管理设计，将系统管理员、租户管理员、具备目标租户空间创建权限的用户和团队空间角色边界拆解为用户目标、Use Story、Use Case、业务规则、边界异常和验收标准。
- 影响范围：全局管理面、租户管理面、租户创建与启停、租户管理员授权、团队空间 tenantId、租户级权限守卫、devuc 鉴权、管理操作审计、前端租户上下文和验收标准。
- 独立评审：2026-06-08 已完成 PRD 独立 agent review；初审发现租户软停用、devuc 异常、系统管理员边界、创建职责边界和最后一个租户管理员撤销策略等问题，已修正并经复核全部关闭，无剩余阻断问题。
- 与相邻功能的边界：
  - 团队空间创建后的创建者、管理员、成员和访客权限由 F-002 负责，本功能只定义租户上级域和租户管理面权限。
  - 团队空间初始化、知识库和 Harness 配置由 F-001 负责，本功能只要求新建空间必须归属有效租户。
  - 全局或租户级团队模板的具体配置和应用由 F-004 负责，本功能只提供租户级模板入口和权限上下文。
- 待确认问题：
  - 租户编码、名称、归属组织字段的正式唯一性、格式、可修改性和展示规范。
  - devuc 中系统管理员和租户管理员身份来源、同步周期、失败错误码和降级策略。
  - 租户停用是否需要升级为冻结既有团队空间；R-0630 暂按软停用处理，即管理面只读、禁止新增空间和新增授权，但不阻断既有团队空间按 F-002 角色访问。
  - 历史未归属团队空间迁移策略；R-0630 暂只要求新建团队空间必须归属租户。

## PRD 设计

### 用户目标

- 系统管理员可以进入全局管理面，创建租户并维护租户基础信息和启停状态。
- 系统管理员可以授予或撤销租户管理员，使租户具备明确治理责任人；当租户暂无管理员时，由系统管理员在全局管理面兜底治理。
- 租户管理员可以进入被授权租户的管理面，查看租户基础信息、团队空间清单和租户级配置入口。
- 具备目标租户空间创建权限的用户在创建团队空间时必须选择或继承一个有效租户，使空间、权限、模板和审计具备明确归属。
- 平台可以在全局管理面、租户管理面和团队空间三个层级执行一致的后端鉴权、租户上下文校验和审计。
- 租户管理员不因租户管理员身份自动获得具体团队空间内容写权限；如需管理空间内容，必须被授予团队空间角色。

### 业务规则

- 管理层级：
  - AgentSpace 管理面分为全局管理面、租户管理面和团队空间三个层级。
  - 全局管理面仅系统管理员可访问，用于租户治理和租户管理员授权。
  - 租户管理面仅目标租户管理员可访问，用于查看租户基础信息、团队空间清单和租户级配置入口。
  - 系统管理员可以在全局租户详情查看任一租户的团队空间清单摘要，但这不等同于进入租户管理面，也不授予团队空间内容权限。
  - 团队空间内容权限由 F-002 的团队空间角色决定，不由租户管理员身份自动继承。
  - ManagementPlaneScope 必须区分 global 和 tenant，租户管理接口必须携带明确 tenantId。
- 租户模型：
  - Tenant 至少包含租户 ID、租户名称、租户编码、归属组织、状态、描述、创建时间和更新时间。
  - R-0630 租户名称和租户编码按全局唯一校验。
  - 租户状态至少包含启用和停用。
  - R-0630 的租户停用为软停用：停用后不允许在该租户下新建团队空间、新增租户管理员授权或提交租户级配置写操作。
  - 停用租户仍允许系统管理员在全局租户详情查看和重新启用，也允许既有租户管理员只读查看租户管理面和团队空间清单。
  - 停用租户不阻断既有团队空间内容访问；既有团队空间仍按 F-002 的 SpaceMember 角色和权限矩阵判断。
  - R-0630 不包含租户删除、合并、拆分、迁移、计费、容量和套餐治理。
- 租户管理员授权：
  - 只有系统管理员可以授予或撤销租户管理员。
  - 租户管理员只能访问被授权租户，不能访问其他租户管理面。
  - 同一用户可以被授予多个租户的租户管理员，每个授权关系独立记录。
  - 租户管理员授权和撤销必须保留历史记录，不通过物理删除抹除审计。
  - R-0630 允许撤销最后一个租户管理员；撤销后该租户不再向普通用户展示租户管理面入口，由系统管理员在全局管理面兜底治理并重新授权。
  - 撤销授权后，用户已打开页面在刷新或下一次接口调用时失效；后端权限立即按撤销结果生效。
- 团队空间租户归属：
  - 新建团队空间必须写入有效 TeamSpace.tenantId。
  - F-003 负责空间创建前的租户选择、租户状态校验和租户上下文输出；TeamSpace 落库和创建者 SpaceMember 初始化由 F-001/F-002 的空间创建事务承接。
  - 从租户管理面进入空间创建时，租户由入口上下文带入。
  - 从普通空间创建入口进入时，若用户有多个可用租户，需要选择租户。
  - 租户停用、租户不存在或用户无目标租户空间创建权限时，团队空间创建被拒绝。
  - 历史未归属团队空间迁移不纳入 R-0630，本功能只要求新建团队空间具备租户归属。
- 权限矩阵：

| 权限项 | 系统管理员 | 租户管理员 | 具备目标租户空间创建权限的用户 | 团队空间管理员 |
| --- | --- | --- | --- | --- |
| 访问全局管理面 | 是 | 否 | 否 | 否 |
| 创建和维护租户 | 是 | 否 | 否 | 否 |
| 启用或停用租户 | 是 | 否 | 否 | 否 |
| 授予或撤销租户管理员 | 是 | 否 | 否 | 否 |
| 在全局租户详情查看团队空间清单摘要 | 是 | 否 | 否 | 否 |
| 访问被授权租户管理面 | 否 | 是 | 否 | 否 |
| 查看被授权租户团队空间清单 | 否 | 是 | 否 | 否 |
| 在有权限且启用的租户下创建团队空间 | 需具备空间创建权限 | 需具备空间创建权限 | 是 | 需具备空间创建权限 |
| 管理具体团队空间成员与角色 | 否 | 否 | 否 | 按 F-002 判断 |

| 进入租户级团队模板入口 | 是 | 是 | 否 | 否 |

- 鉴权与审计：
  - 系统管理员身份、租户管理员身份和租户访问能力需由 devuc 或平台配置提供，后端统一判断。
  - 全局管理和租户管理读写操作必须以后端鉴权结果为准，前端权限态只作为体验优化。
  - 租户创建、租户更新、启停、租户管理员授权、撤销、团队空间创建租户上下文和权限拒绝均需记录审计。
  - 审计记录至少包含 ManagementPlaneScope、租户、操作者、目标对象、动作、变更前后内容、时间、结果和失败原因。

### 边界与异常

- 非系统管理员访问全局管理面时，系统拒绝访问。
- 租户管理员访问未授权租户管理面时，系统拒绝访问。
- 租户名称或编码重复时，系统阻止保存并提示冲突字段。
- 租户创建或更新失败时，系统保留已填写内容并提示失败原因。
- 租户停用后，系统阻止新增团队空间、新增租户管理员授权和租户级配置写入；既有租户管理员只能只读查看租户管理面。
- 租户停用后，既有团队空间内容访问不被租户状态直接阻断，仍按 F-002 的 SpaceMember 角色判断。
- 系统管理员撤销租户管理员后，该用户已打开的租户管理面在刷新或下一次接口调用时失效。
- 系统管理员撤销最后一个租户管理员后，租户进入无租户管理员状态，不再向普通用户展示租户管理面入口，由系统管理员兜底重新授权。
- devuc 鉴权失败、超时或权限数据缺失时，系统默认拒绝管理面读写请求。
- 创建团队空间时 tenantId 为空、租户不存在、租户停用或用户无目标租户空间创建权限时，系统拒绝租户上下文校验，空间创建事务不得继续。
- 租户管理员尝试编辑未被授予团队空间角色的空间成员、知识库或 Harness 时，系统按 F-002 拒绝。
- 历史数据中出现未归属租户的团队空间时，不允许通过租户管理面将其作为正常空间治理，需进入待迁移或待确认状态。
- 审计写入失败时，租户创建、启停、授权和撤销等管理写操作不得静默成功。

## 验收标准

### 正常路径

- 系统管理员可以进入全局管理面并查看租户列表。
- 系统管理员可以在全局租户详情查看团队空间清单摘要，但不会因此进入租户管理面或获得团队空间内容权限。
- 系统管理员可以创建租户，租户保存后出现在租户列表中。
- 系统管理员可以维护租户基础信息。
- 系统管理员可以启用或停用租户。
- 系统管理员可以在租户详情中授予租户管理员。
- 被授权用户可以进入对应租户管理面。
- 租户管理员可以查看被授权租户的基础信息和团队空间清单。
- 具备目标租户空间创建权限的用户创建团队空间前，系统必须完成有效 tenantId 校验，空间创建事务必须写入该 tenantId。
- 系统管理员可以撤销租户管理员，撤销后该用户不能继续访问对应租户管理面。
- 租户创建、启停、授权、撤销和权限拒绝均写入审计记录。

### 边界场景

- 同一用户可以被授予多个租户的租户管理员，并分别看到多个租户管理入口。
- 租户管理员只看到自己被授权的租户管理入口。
- 从租户管理面创建团队空间时，租户由入口上下文带入。
- 从普通空间创建入口创建团队空间时，有多个可用租户的用户需要选择租户。
- 租户停用后，不允许新增团队空间、新增租户管理员授权或提交租户级配置写操作。
- 租户停用后，既有租户管理员可以只读查看租户管理面和团队空间清单；既有团队空间内容仍按 F-002 角色访问。
- 撤销租户管理员后，后端立即按撤销结果鉴权，已打开页面在刷新或下一次接口调用时失效。
- 撤销最后一个租户管理员后，租户管理面入口不再向普通用户展示，系统管理员仍可在全局管理面重新授权。

### 异常场景

- 非系统管理员访问全局管理面时，系统拒绝访问。
- 租户管理员访问未授权租户管理面时，系统拒绝访问。
- 租户名称或编码重复时，系统阻止创建或更新并提示冲突字段。
- devuc 鉴权失败、超时或权限数据缺失时，系统拒绝管理面读写请求。
- 创建团队空间时 tenantId 为空、租户不存在、租户停用或用户无目标租户空间创建权限时，系统拒绝租户上下文校验，空间创建事务不得继续。
- 历史未归属团队空间出现在租户治理链路时，系统标记为待迁移或待确认，不作为正常租户空间治理。
- 审计写入失败时，租户管理写操作不会静默成功。

### 权限场景

- 租户管理员不能创建租户、启停租户或授权其他租户管理员。
- 租户管理员不能访问未授权租户的管理面、团队空间清单或租户级配置入口。
- 租户管理员未被授予团队空间角色时，不能编辑该团队空间成员、知识库或 Harness 配置。
- 系统管理员在全局租户详情查看团队空间清单摘要时，不能绕过 F-002 进入团队空间内容。
- 团队空间管理员不能访问全局管理面或租户管理面，除非另有系统管理员或租户管理员授权。
- 前端隐藏无权限入口时，直接调用后端管理接口仍会被拒绝。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：PRD 新增全局管理面、租户管理面、租户详情、租户管理员授权、租户启停、团队空间租户归属展示和跨租户权限提示。

### 页面与交互

- 全局管理面：
  - 租户列表展示租户名称、编码、状态、归属组织、管理员数量、团队空间数量、创建时间和操作入口；管理员数量为 0 时展示待授权提示。
  - 新建租户表单包含租户名称、租户编码、归属组织、描述和初始状态。
  - 租户详情展示基础信息、启停状态、租户管理员、团队空间清单摘要、租户级模板入口和审计入口。
  - 系统管理员可以搜索用户并授予或撤销租户管理员。
- 租户管理面：
  - 顶部展示当前租户名称、编码、状态和当前用户管理权限来源；停用租户展示只读状态。
  - 团队空间清单展示空间名称、创建者、项目编号、成员数量、最近更新时间和空间状态。
  - 租户级配置和租户级团队模板入口展示为租户上下文入口，模板细节由 F-004 承接。
- 团队空间创建入口：
  - 从租户管理面进入时，所属租户锁定为当前租户并展示来源。
  - 从普通创建入口进入时，若用户有多个可用租户，必须选择租户；只有一个可用租户时可默认带入但仍需展示。
  - 租户停用或用户无空间创建权限时，禁用创建入口并展示原因；前端不继续提交空间创建请求。

### 状态与文案

- 空状态：`暂无租户`、`暂无租户管理员`、`当前租户暂无团队空间`。
- 成功态：`租户已创建`、`租户信息已更新`、`租户已启用`、`租户已停用`、`租户管理员已授权`、`租户管理员已撤销`。
- 异常态：无全局管理权限、无租户管理权限、租户名称重复、租户编码重复、租户已停用、devuc 鉴权失败、审计写入失败。
- 停用提示：`租户已停用，当前仅支持查看；不能新增团队空间、新增租户管理员或修改租户级配置。`
- 无租户管理员提示：`当前租户暂无租户管理员，请由系统管理员重新授权。`
- 团队空间权限提示：`租户管理员不自动拥有团队空间内容权限，需要空间角色授权。`

## 前端设计

### 是否需要刷新

- 结论：是
- 理由：前端需要新增管理面路由、租户上下文、授权组件、团队空间租户选择、管理面权限守卫和停用态处理。

### 页面、组件与状态

- 页面路由：
  - 全局租户管理页。
  - 租户详情页。
  - 租户管理面首页。
  - 租户团队空间清单页。
  - 团队空间创建页的租户选择、租户锁定和租户上下文校验状态。
- 核心组件：
  - `TenantList`：租户列表和状态展示。
  - `TenantCreateDialog`：创建租户表单。
  - `TenantDetailPanel`：基础信息、状态和统计。
  - `TenantStatusToggle`：启用、停用操作和确认。
  - `TenantAdminGrantEditor`：租户管理员搜索、授权和撤销。
  - `TenantSpaceList`：租户团队空间清单。
  - `ManagementScopeGuard`：全局和租户管理面权限守卫。
  - `TenantScopeBadge`：展示当前租户上下文和权限来源。
  - `TenantPicker`：普通空间创建入口的租户选择。
- 状态管理：
  - 当前管理面范围：global 或 tenant。
  - 当前用户系统管理员权限和可管理租户列表。
  - 当前租户详情、状态、管理员授权列表和团队空间清单。
  - 团队空间创建可用租户列表、选中租户、租户上下文校验结果和租户停用原因。
  - 租户创建、更新、启停、授权和撤销提交状态。

### 接口依赖与异常处理

- 依赖接口：
  - 查询当前用户管理面权限和可管理租户列表。
  - 查询租户列表和租户详情。
  - 创建和更新租户。
  - 启用或停用租户。
  - 查询、授予、撤销租户管理员。
  - 查询租户团队空间清单。
  - 查询当前用户可创建团队空间的租户列表。
  - 创建团队空间前校验并返回可用 tenantId。
- 异常处理：
  - 管理面权限不足时展示权限不足页或返回上一级。
  - 租户创建、更新和启停失败时保留表单内容和当前列表状态。
  - 授权失败时保留搜索结果和已选用户。
  - 接口返回租户停用时，前端禁用新增团队空间、新增租户管理员和租户级配置写入口，仅保留只读清单。
  - 接口返回权限已撤销时，清理当前租户管理面状态并提示权限已失效。
  - 审计写入失败导致写操作失败时，提示操作未完成。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：后端需要新增租户模型、租户管理员授权关系、管理面权限判断、团队空间租户上下文校验、租户软停用规则和管理操作审计。

### 接口与数据

- 核心服务：
  - `TenantService`：创建、查询、更新、启用和停用租户。
  - `TenantAdminService`：授予、撤销和查询租户管理员授权。
  - `ManagementPermissionService`：判断系统管理员、租户管理员和 ManagementPlaneScope 权限。
  - `TenantSpaceQueryService`：查询租户下团队空间清单。
  - `TenantContextResolver`：解析请求 tenantId、入口上下文并输出已校验租户上下文。
  - `DevucAuthAdapter`：对接 devuc 完成身份和权限鉴权。
  - `AuditService`：记录租户创建、更新、启停、授权、撤销、空间创建租户上下文校验和权限拒绝。
- 核心数据对象：
  - `Tenant`：租户 ID、租户名称、租户编码、归属组织、状态、描述、创建时间、更新时间。
  - `TenantAdminGrant`：租户 ID、用户 ID、授权状态、授权来源、授权人、生效时间、撤销时间。
  - `TeamSpace.tenantId`：团队空间所属租户 ID，新建团队空间必填，由 F-001/F-002 的空间创建事务写入。
  - `ManagementPlaneScope`：管理面范围，取值为 global 或 tenant。
  - `TenantAuditLog`：范围、租户 ID、操作者、目标对象、操作类型、变更前后内容、结果、失败原因、时间。
- 建议接口：
  - `GET /management/permissions/me`
  - `GET /management/tenants`
  - `POST /management/tenants`
  - `GET /management/tenants/{tenantId}`
  - `PATCH /management/tenants/{tenantId}`
  - `POST /management/tenants/{tenantId}/enable`
  - `POST /management/tenants/{tenantId}/disable`
  - `GET /management/tenants/{tenantId}/admins`
  - `POST /management/tenants/{tenantId}/admins`
  - `DELETE /management/tenants/{tenantId}/admins/{userId}`
  - `GET /management/tenants/{tenantId}/team-spaces`
  - `GET /team-space-create/available-tenants`

### 业务规则、权限与异常处理

- 系统管理员身份由 devuc 或平台配置提供，所有全局管理接口必须校验系统管理员权限。
- 所有租户管理接口必须校验当前用户是否为目标租户有效管理员。
- 租户名称和编码 R-0630 按全局唯一处理；编码是否允许后续修改待确认。
- 租户停用后，禁止新建团队空间和新增租户管理员授权；系统管理员仍可重新启用租户。
- 租户停用后，租户管理员管理面只读；既有团队空间内容访问继续交由 F-002 判断。
- 授权租户管理员时需要校验目标用户存在且可被 devuc 识别。
- 撤销租户管理员不能删除历史记录，应更新授权状态和撤销时间。
- 创建团队空间前必须传入或继承有效 tenantId，且租户处于启用状态；F-003 只输出已校验租户上下文，不单独创建 TeamSpace。
- 请求 tenantId、ManagementPlaneScope 和资源归属不一致时，默认拒绝。
- devuc 失败、租户不存在、授权状态异常或审计写入失败时，默认拒绝管理面读写请求；租户停用时拒绝管理写请求但允许授权内只读查看。
- 租户管理员进入具体团队空间内容时，必须继续交由 F-002 的团队空间角色判断。
