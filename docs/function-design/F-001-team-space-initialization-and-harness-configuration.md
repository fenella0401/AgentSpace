## 变更摘要

- 本次变更：收敛 Harness 基础配置口径，保留团队空间初始化、可选知识库、AGENT.md、市场 Skill、市场 Tool、在线 Agent 和环境变量；移除 Hook 配置和历史配置导入。
- 核心调整：
  - 所有 Harness 基础配置均为可选，新空间可以直接使用平台默认 Agent。
  - Harness 配置不从代码仓、`.codex`、`.claudecode`、`.opencode` 或其他历史目录同步；远程代码仓只作为知识库资料来源。
  - Skill 只支持从公司 AI 市场选择，不支持空间内新增或 AI 生成自定义 Skill。
  - Tool 只支持从公司 AI 市场添加；MCP 仅作为底层协议，不作为用户配置项。
  - Agent 不支持从市场选择，只支持在线配置 Agent 名称、描述、使用的 Skill、使用的 Tool 和预置 prompt。
  - F-008 负责 AgentFlow 定义、测试发布、任务启动、状态机和 Agent 任务会话关联。
- 影响范围：产品定义、功能索引、F-004、F-005、F-006、F-008、TC-001、TC-005、TC-008 和 R-0630 发布计划。

## PRD 设计

### 用户目标

- 具备目标租户空间创建权限的用户可以创建团队空间，并在不配置知识库或 Harness 的情况下立即使用平台默认 Agent。
- 团队创建者或团队管理员可以按需建立知识库，知识来源可以是上传资料、手工文件夹或远程代码仓，不限定开发项目。
- 团队创建者或团队管理员可以通过 AGENT.md 维护空间长期指引和知识域局部规则。
- 团队创建者或团队管理员可以从公司 AI 市场选择已发布 Skill，用于 Agent 或 AgentFlow Agent 任务。
- 团队创建者或团队管理员可以从公司 AI 市场添加 Tool，不需要在 AgentSpace 配置 MCP、连接地址、启动命令或认证参数。
- 团队创建者或团队管理员可以在线配置空间 Agent，配置内容包括 Agent 名称、描述、使用的 Skill、使用的 Tool 和预置 prompt。
- 团队创建者或团队管理员只在资源声明依赖时补充环境变量，所有基础配置中心为空也不影响空间使用。
- 配置维护者可以校验依赖、测试、查看配置 diff，并在确认后发布当前生效版本。
- 团队管理员需要配置 AgentFlow 时，从独立 AgentFlow 入口进入 F-008。

### 业务规则

- 零配置与可选性：
  - 新空间创建后立即可使用平台默认 Agent，知识库和 Harness 基础配置中心均允许为空。
  - 没有 AGENT.md、Skill、Tool、Agent 或环境变量时不得阻止普通会话。
  - 缺失依赖只阻止引用该依赖的资源发布或运行，不影响空间和其他独立资源。
- 配置入口与生命周期：
  - Harness 基础配置中心只包含 AGENT.md、Skill、Tool、Agent 和环境变量。
  - Skill 和 Tool 只提供“从市场选择/添加”入口。
  - Agent 只提供在线配置入口，不提供市场选择入口。
  - AI 只能辅助生成 AGENT.md 或 Agent prompt 草稿，不能直接发布、启用或覆盖当前生效版本。
  - 配置版本状态统一为草稿、待补全、校验失败、测试通过、已发布和已停用。
  - 只有已发布资源进入运行快照。
- 知识库与代码仓：
  - 知识库支持上传资料、手工维护文件夹和可选远程代码仓。
  - 顶层文件夹代表知识域，远程代码仓只是知识来源之一。
  - Harness 配置不从代码仓同步，不扫描代码仓中的配置目录，不把代码仓文件变化转换为 Harness 草稿。
- AGENT.md：
  - 空间级 `/AGENT.md` 在每次手工 Agent 会话、事件源 Agent 会话和 AgentFlowRun 中加载。
  - R-0630 在手工 Agent 会话创建、事件源触发和 AgentFlow 启动请求中提供可选 `knowledgeDomainIds`。
  - AGENT.md 始终保存为纯 Markdown，只承载长期指引，不写入 Portal 元数据。
  - `.agentspace/manifest.yaml` 仅记录文件路径、作用范围、版本、来源、内容 hash 和发布状态等文件级元数据。
- Skill：
  - Skill 只能从公司 AI 市场选择；AgentSpace 不提供新建 Skill、AI 生成 Skill 或编辑 Skill 执行指引的能力。
  - Skill 引用保存市场资源 ID、版本、能力快照、依赖摘要和授权状态。
  - 市场 Skill 不存在、下线、版本不兼容或无权限时，系统阻止选择或发布引用它的 Agent。
- Tool：
  - Tool 只能从公司 AI 市场添加；AgentSpace 不提供新建 Tool、连接地址、启动命令、认证参数或 MCP 配置表单。
  - MCP 只作为市场或适配层可能使用的底层协议，不作为用户配置类型。
  - Tool 授权失效时，已发布配置版本保持不可变，资源可用性变为阻断。
- Agent：
  - Agent 在线配置字段为名称、描述、使用的 Skill、使用的 Tool 和预置 prompt。
  - Agent 可引用知识域、市场 Skill 和市场 Tool；引用缺失或无权限时不能发布。
  - 空间可发布多个 Agent，并可指定一个空间默认 Agent；未指定时继续使用平台默认 Agent。
- AgentFlow：
  - AgentFlow 定义、画布、测试、发布、任务启动和状态机均由 F-008 负责。
  - F-001 只向 F-008 提供已发布 Agent、Skill、Tool、AGENT.md 和环境变量引用。
- 环境变量：
  - 环境变量只在资源声明依赖时提示配置，环境变量中心为空不影响空间使用。
  - 敏感值只能写入 Secret Service，AGENT.md、知识库、配置、日志、diff、测试输入输出和快照均不得保存或回显明文。
- 权限与审计：
  - 只有团队创建者和团队管理员可以维护、测试和发布 Harness 基础配置。
  - 团队成员可以使用已发布 Agent、Skill 和 Tool 影响下的会话能力，访客只读。
  - 草稿生成、市场选择、测试、发布、停用和失败结果均需记录审计。

### 边界与异常

- 用户创建空间后点击开始作业时，系统打开统一开始作业入口，默认选择 Agent，并使用平台默认 Agent 和空的空间级扩展配置。
- 非开发空间没有远程仓、代码语言和开发 Tool 时，知识库、AGENT.md 和普通会话仍可正常使用。
- AGENT.md 为空时不生成该层指引；内容超过平台预算时阻止发布并提示拆分到知识域或 Agent prompt。
- 用户尝试新建 Skill、从市场选择 Agent、配置 Hook、导入历史目录或提交 MCP 连接参数时，前端不提供入口且后端拒绝请求。
- 市场 Skill 或 Tool 不存在、下线、版本不兼容或无权限时，系统阻止选择或发布相关资源。
- Tool 授权失败或失效时，不得回显市场凭据，只显示授权状态和处理入口。
- Agent 缺少名称、描述、预置 prompt，或引用不存在的 Skill/Tool 时，系统阻止测试和发布。
- 敏感环境变量安全写入失败时，相关资源不能发布，且不得降级为明文。
- 单个资源发布失败不改变其他已发布资源，也不阻止零配置普通会话。
- 权限在页面打开后被撤销时，后端拒绝后续写请求，前端切换为只读。

### PRD Review 结论

- 合理性：PRD 将 Harness 基础配置收敛为 AGENT.md、Skill、Tool、Agent 和环境变量，符合“零配置可运行”和“配置不从代码仓同步”的边界。
- 一致性：Skill、Tool、Agent 三类资源的创建口径已经区分清楚：Skill/Tool 来自市场，Agent 在线配置。
- 已修正点：移除 Hook 配置、历史目录导入、Skill 自定义和 Agent 市场选择，避免与本次需求冲突。
- 待确认点：AI 市场 Skill/Tool 的正式查询授权契约、Agent prompt 字数上限和 Secret Service 引用格式。

## 验收标准

### 正常路径

- 具备权限的用户创建空间后，可以跳过知识库和全部 Harness 基础配置，点击“开始作业”打开统一开始作业入口，并默认使用平台默认 Agent。
- 团队管理员可以创建业务知识域、上传资料，并在需要时绑定远程代码仓；远程代码仓不触发 Harness 配置同步。
- 团队管理员可以分别创建空间级和知识域级 AGENT.md，并同步为纯 Markdown 指引。
- 团队管理员可以从市场选择 Skill，保存市场资源 ID、版本和能力快照。
- 团队管理员可以从市场添加 Tool，Portal 不要求填写连接地址、启动命令或 MCP 参数。
- 团队管理员可以在线配置 Agent 名称、描述、Skill、Tool 和预置 prompt，并在测试通过后发布。
- 系统在资源缺少 Skill、Tool 或环境变量时就地提示补充，补充后重新校验原资源。
- 环境变量中心为空时空间仍可使用；敏感值保存后只产生安全引用。
- 测试通过的资源可以查看 diff 并发布，发布后形成当前生效版本。
- AgentFlow 入口跳转 F-008，不在 F-001 Harness 基础配置中编辑或发布 AgentFlow。

### 边界场景

- 所有配置中心为空时，手工 Agent 会话仍能使用平台默认 Agent。
- 只有某个 Agent 缺少 Skill 或 Tool 时，仅该 Agent 不能发布，其他资源和会话不受影响。
- 没有远程代码仓的客服、法务或运营空间可以使用上传资料、AGENT.md、市场 Skill 和默认 Agent。
- 空间级 AGENT.md 存在、知识域级 AGENT.md 为空时，只加载空间级指引。
- 市场 Tool 授权失效后，已发布版本保持不可变，但资源可用性变为阻断。
- Agent 未指定空间默认 Agent 时，普通会话继续使用平台默认 Agent。

### 异常场景

- AGENT.md 超出上下文预算或引用不存在知识域时，系统阻止发布并定位问题。
- 用户尝试新建 Skill、配置自定义 Skill 或调用新增 Skill 接口时，系统拒绝请求。
- 用户尝试从市场选择 Agent 或调用市场 Agent 引用接口时，系统拒绝请求。
- 用户尝试配置 Hook、导入历史配置目录或提交 MCP 连接参数时，系统拒绝请求。
- 市场资源不可用、版本不兼容或授权失败时，相关草稿版本保持待补全。
- 敏感环境变量写入失败时，系统不产生明文文件或快照。

### 权限场景

- 团队创建者和团队管理员可以创建、测试、发布和停用 Harness 基础资源。
- 团队成员不能修改 Harness，但可以在会话中使用已发布 Agent、Skill 和 Tool 能力。
- 访客不能启动会话，也不能执行 Harness 写操作。
- 租户管理员未被授予团队空间管理员角色时，不能配置该空间 Harness。
- 权限撤销后，已打开页面的后续发布请求被后端拒绝。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：需要移除 Hook、历史导入、Skill 自定义和 Agent 市场选择入口，并补齐 Agent 在线配置表单。

### 页面与交互

- 空间创建完成页：
  - 主操作为“开始作业”，点击后打开统一开始作业入口并默认选择 Agent；次操作为“完善 Harness”。
  - 不展示强制完成百分比，不把空中心标记为空间异常。
- Harness 总览：
  - 展示 AGENT.md、Skill、Tool、Agent 和环境变量五个基础中心。
  - 展示进入 AgentFlow 的独立入口。
  - 展示已发布资源、待补全资源、依赖问题和最近发布记录。
- AGENT.md 中心：
  - 展示空间级指引和顶层知识域列表。
  - 编辑器只展示 Markdown 正文、章节引导、上下文预算、加载预览、diff 和发布。
- Skill 中心：
  - 只有“从 AI 市场选择 Skill”入口。
  - 卡片展示 Skill 名称、市场版本、能力摘要、依赖和授权状态。
- Tool 中心：
  - 只有“从 AI 市场添加 Tool”入口。
  - 卡片展示能力、版本、权限、授权状态和引用关系，不展示 MCP 配置表单。
- Agent 中心：
  - 只有“在线配置 Agent”入口。
  - 表单字段为 Agent 名称、描述、使用的 Skill、使用的 Tool 和预置 prompt。
  - 支持测试对话、依赖校验、diff 和发布。
- 环境变量：
  - 支持从依赖问题直接打开并定位变量。
  - 敏感值不回填，保存后只展示已配置、缺失或轮换状态。

### 状态与文案

- 配置版本状态：草稿、待补全、校验失败、测试通过、已发布、已停用。
- 空状态：`当前空间使用平台默认 Agent，可直接开始会话`
- Skill 提示：`Skill 由 AI 市场统一治理，AgentSpace 仅选择已发布版本`
- Tool 提示：`Tool 由 AI 市场统一配置，AgentSpace 仅添加已发布版本`
- Agent 提示：`Agent 只支持在线配置名称、描述、Skill、Tool 和预置 prompt`
- MCP 提示：`MCP 是底层适配协议，不作为用户配置项`

## 前端设计

### 是否需要刷新

- 结论：是
- 理由：需要调整五个基础中心导航、市场资源选择器、Agent 在线配置表单、依赖图和配置状态机。

### 页面、组件与状态

- 核心页面：
  - 空间创建完成页。
  - Harness 总览与五个基础配置中心。
  - Agent 在线配置页。
- 核心组件：
  - `HarnessOverview`、`HarnessResourceCard`、`HarnessStateBadge`、`AgentFlowEntry`。
  - `AgentMdEditor`、`KnowledgeDomainScopePicker`、`ContextBudgetMeter`。
  - `MarketSkillPicker`、`MarketToolPicker`、`MarketResourceCard`、`AuthorizationStatus`。
  - `AgentConfigForm`、`PromptEditor`、`AgentTestPanel`。
  - `DependencyResolver`、`EnvVarEditor`、`SecretInput`、`ConfigDiffViewer`。
- 状态管理：
  - 配置版本状态：`draft`、`incomplete`、`validation_failed`、`tested`、`published`、`disabled`。
  - 资源可用性：`available`、`degraded`、`blocked`。
  - 依赖状态：满足、缺失、授权失效、版本不兼容。
  - 零配置状态独立于异常状态，不显示红色告警。

### 接口依赖与异常处理

- 查询五类基础资源、依赖图、当前生效版本和发布历史。
- 查询市场 Skill 和 Tool，添加市场资源引用并发起授权。
- CRUD AGENT.md、Agent 和环境变量草稿。
- 校验、测试、查看 diff、发布和停用基础资源。
- 跳转 F-008 AgentFlow 中心。
- AI 草稿生成失败时保留用户描述并允许手动编辑 AGENT.md 或 Agent prompt。
- 市场不可用时不影响已发布资源和零配置会话。
- 单资源测试或发布失败时保留草稿，不刷新其他资源状态。
- 敏感值不进入前端持久化缓存、日志或错误详情。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：需要明确 F-001 只管理五类 Harness 基础资源、市场引用、AGENT.md 解析、依赖图、统一发布状态和快照引用。

### 接口与数据

- 核心服务：
  - `HarnessResourceService`：统一管理基础资源、状态和版本。
  - `AgentMdService`：解析空间级与知识域级指引并生成加载结果。
  - `HarnessDependencyService`：解析资源依赖和受影响关系。
  - `MarketResourceService`：查询并引用市场 Skill 和 Tool。
  - `AgentConfigService`：维护在线 Agent 草稿、测试和发布。
  - `HarnessValidationService`：执行 Schema、依赖和权限校验。
  - `HarnessTestService`：执行 Agent 测试对话、Skill/Tool 可用性探测和环境变量探测。
  - `HarnessPublishService`：生成 diff、校验基准版本并切换生效版本。
  - `SecretService`：托管敏感环境变量。
  - `AuditService`：记录配置和发布操作。
- 核心数据对象：
  - `HarnessResource`：资源 ID、空间 ID、类型、来源、当前生效版本 ID 和编辑中版本 ID。
  - `HarnessResourceVersion`：资源版本 ID、配置状态、内容摘要、依赖摘要、测试结果、创建人与时间。
  - `HarnessResourceType`：`agent_md`、`skill`、`tool`、`agent`、`env`。
  - `MarketResourceReference`：市场资源 ID、类型、版本、能力快照、权限摘要和授权引用。
  - `AgentConfig`：Agent 名称、描述、Skill 引用、Tool 引用、预置 prompt、默认标识和发布状态。
  - `HarnessDependency`：资源、依赖类型、依赖目标、是否必需和状态。
  - `EnvVarReference`：名称、敏感标识、值或安全引用和依赖关系。
  - `HarnessResourceAvailability`：资源、可用性、依赖原因和最近探测时间。

### 业务规则、权限与异常处理

- 所有基础资源必须归属同一租户和团队空间。
- 后端必须拒绝新建 Skill、新建 Tool、市场 Agent 引用、Hook 配置、历史配置导入和 MCP 连接参数写入请求。
- Agent 发布前必须校验名称唯一性、Skill/Tool 引用、prompt 长度、权限和测试结果。
- 快照只能引用已发布且当前可用的资源版本；外部授权失效不能改写已发布版本内容。
- 所有权限判断以后端当前权限为准，前端入口隐藏不能替代后端鉴权。
