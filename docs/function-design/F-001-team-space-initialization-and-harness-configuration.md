## 变更摘要

- 本次变更：新增团队空间初始化、知识库配置、Harness 配置解析与 Portal 在线配置的功能设计。
- 影响范围：产品定义、功能索引、空间初始化页面、知识库页面、Harness 配置中心、权限控制、远程仓库绑定、配置解析与同步、验收标准。
- 待确认问题：

  - 团队空间角色是否仅包含团队管理员和成员，还是需要观察者、访客、空间 Owner 等更细粒度角色。

  - `.codex` 等配置文件夹的首期标准目录结构、配置格式和兼容版本。
  - Portal 在线配置与知识库文件同时变化时的冲突解决规则。
  - 环境变量敏感值的存储介质、加密策略和是否允许回写明文。

## PRD 设计

### 用户目标

- 团队管理员可以在 AgentSpace Portal 中初始化一个项目空间，并按需添加成员、授予角色和填写项目编号。
- 团队管理员可以为项目空间建立知识库，使 Agent 作业能够感知空间内的项目资料、代码仓文档和 Agent 相关配置。
- 团队管理员可以查看由知识库配置文件解析出的 Harness 配置，并在 Portal 中维护 Agent、Skill、MCP、Workflow 和环境变量。
- 团队成员可以在授权空间中使用统一配置的 Agent 能力，避免重复配置和上下文不一致。

### 范围

**包含：**

- 团队管理员进入 AgentSpace Portal 后创建项目空间。
- 创建空间时可选添加成员并授予角色；未添加成员时，空间作为单人工作空间。
- 创建空间时可选填写公司通用项目编号。
- 空间创建完成后进入 Harness 配置流程，首个配置步骤为知识库配置。
- 知识库以文件系统形式呈现，支持新建多层级文件夹。
- 文件夹可绑定远程代码仓，绑定后自动关联远程代码仓文档。
- 文件夹绑定对象采用可扩展模型，首期支持远程代码仓，后续可扩展其他对象类型。
- 配置知识库后，系统扫描知识库顶层文件夹，识别是否包含 Agent 相关配置文件夹，例如 `.codex`。
- 若发现配置文件夹，系统解析其中的 Agent、Skill、MCP、Workflow 和环境变量配置，并渲染到 Harness 配置中心。
- Harness 配置中心包括 Agent 中心、Skill 中心、MCP 中心、Workflow 中心和环境变量中心。
- 团队管理员可在 Portal 中在线配置上述 Harness 内容，并将配置结果更新到知识库配置文件夹。
- Portal 在线配置支持从公司 AI 市场选择配置资源。
- 知识库配置文件发生变化时，Portal 需要感知并刷新对应 Harness 配置展示。

**不包含：**

- Agent 执行任务、任务队列、运行沙箱和日志观测的完整能力。
- 公司 AI 市场资源的创建、审核、发布、下线和计费规则。
- 远程代码仓平台的账号开通、审批流和组织级权限治理。
- 代码仓文档抽取算法的细节设计。
- 非团队管理员在线编辑 Harness 配置。

### 核心流程

1. 团队管理员进入 AgentSpace Portal，选择创建项目空间。
2. 团队管理员填写空间名称等基础信息，可选填写公司通用项目编号。
3. 团队管理员可选添加成员并授予角色；若跳过成员添加，系统创建仅包含管理员本人的单人工作空间。
4. 系统创建项目空间，并引导团队管理员进入 Harness 配置流程。
5. 团队管理员首先进入知识库配置，创建知识库顶层文件夹或多层级子文件夹。
6. 团队管理员可为文件夹绑定远程代码仓；系统保存绑定关系，并自动关联该仓库可识别的文档。
7. 系统扫描知识库顶层文件夹，判断是否存在 `.codex` 等 Agent 相关配置文件夹。
8. 若存在配置文件夹，系统解析 Agent、Skill、MCP、Workflow 和环境变量配置。
9. Portal 将解析结果按类型渲染到 Agent 中心、Skill 中心、MCP 中心、Workflow 中心和环境变量中心，并标记来源、更新时间和解析状态。
10. 团队管理员可在各配置中心新增、编辑、删除或从公司 AI 市场选择配置资源。
11. 团队管理员保存在线配置后，系统将变更写入知识库对应配置文件夹，并刷新配置解析结果。
12. 若知识库配置文件后续被外部更新，系统重新解析并刷新 Portal 中的 Harness 配置展示。

### 业务规则

- 空间创建人默认成为团队管理员。
- 成员添加为可选项；未添加成员时，空间仍可创建并作为单人工作空间使用。
- 项目编号为可选项；若填写，需要按公司项目编号体系进行格式和有效性校验，具体规则待确认。
- 知识库是项目空间内 Agent 作业可感知信息的统一集合，以文件系统形式组织。
- 知识库文件夹支持多层级结构；顶层文件夹用于配置扫描入口。
- 文件夹绑定对象首期支持远程代码仓，绑定模型必须保留对象类型、对象标识、连接配置、同步状态和扩展元数据。
- 远程代码仓绑定后，系统自动关联仓库中的文档类内容；首期文档识别规则待技术设计确认。
- 系统仅扫描知识库顶层文件夹下的 Agent 相关配置文件夹；配置目录命名和可识别类型需要版本化管理。
- Harness 配置以知识库配置文件作为可持久化配置源，Portal 提供可视化读取和编辑层。
- Portal 在线配置保存后，需要更新知识库配置文件夹下的内容，并记录操作者、来源和更新时间。
- 知识库配置文件变化可能来自 Portal 或外部更新，系统需要定期扫描或通过事件触发刷新。
- Agent、Skill、MCP、Workflow 和环境变量配置需要展示来源：知识库解析、Portal 创建、AI 市场选择或同步冲突。
- 从公司 AI 市场选择的配置资源应保存市场资源标识和版本信息；是否保存快照待确认。
- 在线配置仅团队管理员可操作；团队成员默认可查看或使用，具体查看范围由角色体系确认。
- 敏感环境变量不应以明文写入知识库配置文件；配置文件应保存密钥引用，敏感值由安全存储托管。

### 边界与异常

- 项目编号为空时，空间创建不受阻塞。
- 成员列表为空时，系统创建单人工作空间。
- 成员添加失败时，应允许团队管理员返回修改成员信息或先创建空间后补充成员。
- 远程代码仓绑定失败时，应展示失败原因，并允许重试、修改绑定或跳过绑定。
- 代码仓文档自动关联失败时，不应影响文件夹创建，但需要标记文档关联异常。
- 未发现配置文件夹时，Harness 配置中心展示空状态，并引导团队管理员在线配置或从 AI 市场选择。
- 配置文件解析失败时，应保留原始配置文件，不覆盖用户内容，并展示错误位置、错误类型和最近成功解析版本。
- 知识库配置文件和 Portal 在线配置发生冲突时，应提示团队管理员处理，避免静默覆盖。
- 公司 AI 市场资源不可用、版本不兼容或下线时，应阻止保存或提示替代方案。
- 非团队管理员访问在线配置操作时，应禁止提交，并提示权限不足。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：本功能新增空间初始化、知识库配置和多个 Harness 配置中心，涉及页面入口、表单、树形结构、配置状态、权限态和异常文案。

### 页面与交互

- 空间初始化入口：AgentSpace Portal 提供创建项目空间入口。
- 创建空间向导：
  - 基础信息：空间名称、可选项目编号。
  - 成员与角色：可选添加成员、选择角色、跳过成员添加。
  - 创建完成：进入 Harness 配置。
- 知识库配置页：
  - 左侧或主体区域展示文件系统树，支持新建文件夹、多层级展开、选中文件夹。
  - 文件夹详情展示绑定对象、远程代码仓信息、文档关联状态和最近扫描时间。
  - 支持为文件夹绑定远程代码仓，并展示绑定成功、连接失败、同步中、文档关联异常等状态。
- Harness 配置页：
  - 以中心或标签页组织 Agent 中心、Skill 中心、MCP 中心、Workflow 中心和环境变量中心。
  - 每个中心展示配置名称、来源、版本、状态、更新时间和操作入口。
  - 支持从公司 AI 市场选择资源、在线新增/编辑/删除配置、查看解析详情。
  - 环境变量中心需要区分普通变量、敏感变量引用和缺失值状态。
- 权限态：
  - 团队管理员可编辑。
  - 非团队管理员进入只读态或使用态，不展示保存类主操作。

### 状态与文案

- 空状态：未配置知识库、未发现配置文件夹、当前中心暂无配置。
- 加载态：空间创建中、知识库扫描中、远程仓库连接中、配置解析中、配置回写中。
- 成功态：空间已创建、仓库已绑定、文档已关联、配置已同步。
- 异常态：项目编号无效、成员添加失败、仓库连接失败、配置解析失败、回写失败、权限不足。
- 冲突态：检测到知识库配置文件已被外部更新，需要团队管理员选择保留文件版本、保留 Portal 版本或手动合并。

## 前端设计

### 是否需要刷新

- 结论：是
- 理由：本功能新增创建向导、知识库树、远程仓库绑定、Harness 配置中心、AI 市场选择、权限态和同步状态展示。

### 页面、组件与状态

- 页面路由：
  - 空间创建页：创建项目空间并完成基础信息、成员与角色配置。
  - 空间初始化配置页：承接空间创建后的 Harness 配置流程。
  - 知识库配置页：管理文件夹树、绑定对象和扫描状态。
  - Harness 配置中心页：按配置类型管理 Agent、Skill、MCP、Workflow 和环境变量。
- 核心组件：
  - SpaceCreateWizard：空间创建向导。
  - MemberRoleEditor：成员搜索、添加、移除和角色选择。
  - KnowledgeTree：知识库文件夹树。
  - FolderBindingPanel：文件夹绑定对象配置面板。
  - RepoBindingForm：远程代码仓绑定表单。
  - HarnessCenterTabs：Harness 配置中心导航。
  - HarnessConfigList：配置列表、来源、版本和状态展示。
  - MarketResourcePicker：公司 AI 市场资源选择器。
  - EnvVarEditor：环境变量编辑器，支持敏感变量引用。
  - SyncStatusBanner：解析、同步、冲突和异常提示。
- 状态管理：
  - 空间创建状态：草稿、提交中、成功、失败。
  - 成员配置状态：待添加、校验中、已添加、添加失败。
  - 知识库状态：未配置、已配置、扫描中、扫描失败、扫描完成。
  - 文件夹绑定状态：未绑定、连接中、已绑定、连接失败、文档关联异常。
  - Harness 配置状态：未解析、已解析、编辑中、回写中、已同步、冲突、解析失败。

### 接口依赖与异常处理

- 依赖接口：
  - 创建项目空间。
  - 校验项目编号。
  - 搜索成员并配置角色。
  - 查询和更新知识库文件夹树。
  - 创建、更新、删除文件夹绑定对象。
  - 触发或查询知识库扫描任务。
  - 查询 Harness 配置解析结果。
  - 新增、编辑、删除 Harness 配置。
  - 查询公司 AI 市场资源并选择资源版本。
  - 查询配置同步、解析错误和冲突详情。
- 异常处理：
  - 表单校验错误需阻止提交并定位到字段。
  - 接口失败需保留用户已输入内容。
  - 配置冲突需阻止静默覆盖，并进入冲突处理流程。
  - 非管理员访问编辑接口失败后，前端需切换或保持只读态。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：本功能新增空间、成员角色、知识库、文件夹绑定、配置扫描解析、双向同步、AI 市场引用、权限和审计等后端能力。

### 接口与数据

- 核心服务：
  - SpaceService：创建和管理项目空间、项目编号、空间状态。
  - MembershipService：管理空间成员、角色和权限。
  - KnowledgeBaseService：管理知识库文件夹树、文件夹元数据和绑定对象。
  - FolderBindingService：管理文件夹与外部对象的绑定，首期适配远程代码仓。
  - RepoIntegrationService：连接远程代码仓并关联仓库文档。
  - ConfigScanService：扫描顶层文件夹中的 Agent 相关配置目录。
  - HarnessConfigService：解析、聚合、保存和回写 Agent、Skill、MCP、Workflow、环境变量配置。
  - MarketIntegrationService：查询公司 AI 市场资源并记录资源引用。
  - SecretService：托管敏感环境变量值或密钥引用。
  - AuditService：记录空间创建、权限变更、配置修改和冲突处理。
- 核心数据对象：
  - Space：空间 ID、名称、创建人、项目编号、状态、创建时间、更新时间。
  - SpaceMember：空间 ID、用户 ID、角色、加入状态、授权来源。
  - KnowledgeFolder：文件夹 ID、空间 ID、父文件夹 ID、名称、路径、层级、状态。
  - FolderBinding：文件夹 ID、绑定类型、外部对象 ID、连接配置引用、同步状态、扩展元数据。
  - RepoBinding：仓库地址、分支或默认引用、访问凭据引用、文档关联状态。
  - HarnessConfigItem：配置类型、名称、来源、版本、状态、配置内容引用、最近解析时间。
  - ConfigSourceSnapshot：配置文件路径、内容摘要、更新时间、解析版本、来源。
  - MarketResourceReference：市场资源 ID、资源类型、资源版本、兼容性状态。
  - EnvVarReference：变量名、是否敏感、值引用、缺失状态。

### 业务规则、权限与异常处理

- 权限规则：
  - 创建空间的用户默认拥有团队管理员权限。
  - 只有团队管理员可以新增成员、分配角色、绑定仓库、在线编辑 Harness 配置和处理冲突。
  - 非团队管理员默认不允许调用写接口。
- 同步规则：
  - 配置扫描以知识库顶层文件夹为入口，识别 `.codex` 等受支持配置目录。
  - 配置目录解析需要版本化，避免旧配置格式影响新版解析。
  - Portal 保存配置时，通过 HarnessConfigService 序列化为配置文件并写回知识库配置文件夹。
  - 外部文件更新通过事件、Webhook 或定时扫描触发重新解析，具体机制待技术选型确认。
  - 每次解析和回写都需要记录内容摘要，用于识别漂移和冲突。
- 扩展规则：
  - FolderBinding 以绑定类型区分远程代码仓、文档库、制品库等未来对象。
  - 每类绑定对象通过适配器提供连接校验、同步、文档发现和状态回传能力。
- 异常规则：
  - 配置解析失败不能覆盖最近可用配置。
  - 回写失败需要保留 Portal 草稿或变更意图，便于管理员重试。
  - 敏感环境变量只能写入密钥引用，明文值必须进入安全存储。
  - 市场资源下线或版本不兼容时，配置项需要进入不可用或需处理状态。

## 验收标准

### 正常路径

- 团队管理员可以创建项目空间，并成为该空间团队管理员。
- 团队管理员不添加成员时，系统成功创建单人工作空间。
- 团队管理员添加成员并授予角色后，成员出现在空间成员列表中。
- 团队管理员可选填写项目编号；填写有效项目编号时空间保存该编号。
- 团队管理员可以创建多层级知识库文件夹。
- 团队管理员可以为文件夹绑定远程代码仓，绑定成功后展示仓库信息和文档关联状态。
- 知识库顶层文件夹存在 `.codex` 等配置目录时，系统可以解析配置并展示到对应 Harness 配置中心。
- 团队管理员可以在 Agent、Skill、MCP、Workflow 和环境变量中心查看配置来源、版本、状态和更新时间。
- 团队管理员可以从公司 AI 市场选择资源并保存到对应 Harness 配置中心。
- 团队管理员在线保存配置后，知识库配置文件夹下内容被更新，并刷新同步状态。

### 边界场景

- 项目编号为空时，空间仍可创建成功。
- 成员列表为空时，空间为单人工作空间。
- 知识库未发现配置文件夹时，各 Harness 配置中心展示空状态和可配置入口。
- 同一知识库内存在多层级文件夹时，系统仍只按规则扫描顶层文件夹中的配置目录。
- 文件夹绑定对象模型可以记录远程代码仓以外的未来绑定类型字段，不影响首期远程代码仓绑定。
- 敏感环境变量保存后，配置文件中只出现密钥引用或占位信息，不出现明文敏感值。

### 异常场景

- 项目编号格式无效时，系统阻止保存并提示校验错误。
- 远程代码仓连接失败时，系统展示失败原因，并允许管理员重试或修改绑定。
- 远程代码仓文档关联失败时，系统保留绑定关系并标记文档关联异常。
- 配置文件解析失败时，系统展示错误信息，不覆盖原配置文件和最近成功解析结果。
- Portal 在线配置保存时检测到外部文件更新，系统进入冲突处理，不静默覆盖。
- 公司 AI 市场资源不可用或版本不兼容时，系统阻止保存并提示原因。
- 配置回写失败时，系统保留编辑内容或变更记录，并允许管理员重试。

### 权限场景

- 非团队管理员无法创建、编辑、删除 Harness 配置。
- 非团队管理员无法绑定远程代码仓或处理配置冲突。
- 非团队管理员调用写接口时，后端返回权限不足，前端展示只读或无权限提示。
- 团队管理员的配置变更被审计记录，至少包含操作者、操作类型、对象、时间和结果。
