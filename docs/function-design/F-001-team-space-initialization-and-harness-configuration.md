## 变更摘要

- 本次变更：收敛 F-001 职责，保留团队空间初始化、可选知识库、AGENT.md、Skill、Tool、Agent、Hook、环境变量和 `.agentspace` 基础布局；将 Harness CICD 流水线配置、画布、发布、任务运行和状态机移交 F-008。
- 核心调整：
  - 所有 Harness 基础配置均为可选，新空间可以直接使用平台默认 Agent。
  - AGENT.md 始终保持纯 Markdown，Portal 仅保存文件级发布元数据。
  - Tool 只能从公司 AI 市场选择；Skill 和 Agent 支持市场选择或空间内新建。
  - Hook 仍由本功能配置、测试和发布；真实手工 Agent 会话 Hook 运行由 F-005 承接，Harness CICD 流水线生命周期 Hook 运行由 F-008 承接。
  - Harness CICD 流水线不再作为 F-001 的配置中心；`.agentspace/harness-pipelines/` 目录属于统一文件布局的一部分，由 F-008 管理内容。
  - 运行时只识别 AGENT.md 和 `.agentspace` 新布局；`.codex`、`.claudecode`、`.opencode` 仅可由用户显式导入为 Portal 草稿。
- 影响范围：产品定义、功能索引、F-004、F-005、F-006、F-008、空间创建完成页、Harness 基础配置中心、配置文件 Schema、Hook 事件协议、TC-001、TC-005、TC-008 和发布计划。
- 与相邻功能的边界：
  - 租户治理由 F-003 负责，团队空间角色和权限矩阵由 F-002 负责。
  - F-001 负责 Harness 基础配置、依赖、测试、发布和版本，并在 Harness 发布前后发出确定性 Hook 事件。
  - F-008 负责 Harness CICD 流水线定义配置、测试发布、任务启动、Agent 任务状态机和 Agent 任务会话关联。
  - 团队模板预置与应用由 F-004 承接。
- 待确认问题：
  - 公司通用项目编号的格式、唯一性、校验接口和创建后是否允许修改。
  - AGENT.md 单文件上限和总上下文预算的具体数值。
  - AI 市场 Tool、Skill 和 Agent 的正式资源查询、授权及版本契约。
  - Secret Service 的存储介质、密钥引用格式和轮换机制。
  - Hook 同步执行超时、失败策略和可触发 Harness CICD 流水线的权限主体。
  - 历史配置导入时不同第三方配置格式的字段映射细节。

## PRD 设计

### 用户目标

- 具备目标租户空间创建权限的用户可以创建团队空间，并在不配置知识库或 Harness 的情况下立即使用平台默认 Agent。
- 团队创建者或团队管理员可以按需建立知识库，知识来源可以是上传资料、手工文件夹或远程代码仓，不限定开发项目。
- 团队创建者或团队管理员可以通过 AGENT.md 维护空间长期指引和知识域局部规则。
- 团队创建者或团队管理员可以从市场选择 Skill，或通过自然语言生成并结构化确认自定义 Skill。
- 团队创建者或团队管理员可以从市场添加 Tool，不需要在 AgentSpace 配置底层连接和 MCP 参数。
- 团队创建者或团队管理员可以从市场选择 Agent，或通过自然语言生成并结构化确认自定义 Agent。
- 团队创建者或团队管理员可以从内置模板创建 Hook，或通过自然语言生成结构化 Hook，为运行过程增加自动校验、确认、通知和审计。
- 团队创建者或团队管理员可以从知识库中的 `.codex`、`.claudecode`、`.opencode` 等历史配置目录导入 Harness 草稿，再人工确认和发布。
- 团队创建者或团队管理员只在资源声明依赖时补充环境变量，所有基础配置中心为空也不影响空间使用。
- 配置维护者可以校验依赖、模拟测试、查看配置 diff，并在确认后发布当前生效版本。
- 团队管理员需要配置 Harness CICD 流水线时，从 Harness CICD 流水线入口进入 F-008，而不是在 F-001 Harness 基础配置中编辑 Harness CICD 流水线。

### 业务规则

- 零配置与可选性：
  - 新空间创建后立即可使用平台默认 Agent，知识库和 Harness 基础配置中心均允许为空。
  - 未配置自定义 Agent 时，手工 Agent 会话使用平台默认 Agent；没有 AGENT.md、Skill、Tool、Hook 或环境变量时不得阻止普通会话。
  - 缺失依赖只阻止引用该依赖的资源发布或运行，不影响空间和其他独立资源。
  - 空间完成状态不以“所有中心已配置”为判断条件。
- 配置入口与生命周期：
  - 配置中心提供“AI 帮我配置”“从市场选择”“手动创建”和“从历史配置导入”入口，但各资源按规则开放：Tool 仅市场选择，Hook 提供模板和结构化自定义。
  - AI 生成只能创建或修改草稿，不能直接发布、启用或覆盖当前生效版本。
  - 配置版本状态统一为草稿、待补全、校验失败、测试通过、已发布和已停用。
  - 资源身份与配置版本分离：资源保存当前生效版本 ID；编辑已发布资源时可以同时存在一个新草稿版本，当前生效版本仍保持已发布。
  - 授权缺失、依赖缺失和版本不兼容属于待补全原因，不新增“待授权”等并列配置状态。
  - 已发布版本不可变；发布后发生授权失效、市场下线或 Secret 失效时，只更新资源可用性和依赖状态，不把已发布版本回退为待补全。
  - 只有已发布资源进入 Harness 快照。
- 推荐引导顺序：
  - 空间目标描述、AGENT.md、基础 Hook 模板、Skill、自动补齐 Tool、Agent、自定义 Hook。
  - 空间目标描述是可选的空间元数据，也是 AI 生成空间级 AGENT.md 草稿的默认输入，不构成独立 Harness 类型。
  - 环境变量不作为固定步骤，在资源依赖出现时就地补充。
  - Harness CICD 流水线入口单独进入 F-008，不放在本功能推荐配置顺序中。
- 知识库与非开发适配：
  - 知识库支持上传资料、手工维护文件夹和可选远程代码仓。
  - 顶层文件夹代表知识域，远程代码仓只是知识来源之一。
  - 产品不得要求项目必须存在代码仓、构建命令、代码语言或开发 Tool。
- AGENT.md：
  - 空间级 `/AGENT.md` 在每次手工 Agent 会话和 HarnessPipelineRun 中加载。
  - R-0630 在手工 Agent 会话创建和 Harness CICD 流水线启动请求中提供可选 `knowledgeDomainIds` 结构化字段；知识域级 `/<domain>/AGENT.md` 仅在该字段选择知识域，或选中的 Agent、Skill、Harness CICD 流水线声明使用该知识域时加载。
  - R-0730 的 `@` 文件引用会确定性映射到所属知识域并合并进 `knowledgeDomainIds`，不替代 R-0630 的结构化知识域选择。
  - 空间级指引先加载，命中知识域指引后加载；知识域可细化业务规则，但不能放宽平台安全策略和空间级禁止事项。
  - 每个知识域具有稳定 `domainId` 和空间内唯一目录名；重命名前校验路径冲突，历史快照继续引用原知识域版本。
  - AGENT.md 始终保存为纯 Markdown，只承载用户和 Agent 可直接阅读的长期指引，不写入 Portal 元数据，也不拆分为规则卡片或额外编号。
  - Portal 对 AGENT.md 只提供 Markdown 编辑、章节引导、AI 草稿、上下文预算、加载预览、diff 和发布能力；用户看到和确认的是 Markdown 正文。
  - `.agentspace/manifest.yaml` 仅记录 AGENT.md 文件路径、作用范围、版本、来源、内容 hash 和发布状态等文件级元数据，不保存 AGENT.md 正文内容。
  - 自由文本仅按空间级后知识域级、再按 `domainId` 排序追加；系统可以提示疑似语义冲突，但不得仅凭模型判断自动阻止，用户确认结果进入快照审计。
  - AGENT.md 保持长期、广泛适用的规则；任务专属方法应进入 Skill，资料正文应进入知识库。
- Skill：
  - 市场 Skill 保存资源 ID、版本和配置快照；自定义 Skill 保存触发条件、输入输出、执行指引、知识域、Tool 依赖、权限和测试用例。
  - Skill 按任务相关性或显式选择加载，不进入所有任务的常驻上下文。
  - 自定义 Skill 必须至少有名称、目标、输入、输出和一条测试用例。
- Tool：
  - Tool 只能从公司 AI 市场添加；AgentSpace 不提供新建 Tool、连接地址、启动命令、认证参数或 MCP 配置表单。
  - Tool 引用至少保存市场资源 ID、版本、能力快照、权限摘要、授权引用和状态。
  - MCP 只作为市场或适配层可能使用的底层协议，不作为用户配置类型。
  - 草稿 Tool 授权失效时，配置版本进入待补全；已发布 Tool 授权失效时，版本仍为已发布，资源可用性变为阻断，依赖它的资源可用性同步变为阻断，不影响不依赖该 Tool 的会话。
- Agent：
  - Agent 支持市场选择或空间内新建；自定义 Agent 至少配置职责、边界和输出方式。
  - Agent 可引用知识域、Skill 和 Tool；引用缺失时不能发布该 Agent。
  - 空间可发布多个 Agent，并可指定一个空间默认 Agent；未指定时继续使用平台默认 Agent。
- Harness CICD 流水线：
  - Harness CICD 流水线定义、画布、测试、发布、任务启动和状态机均由 F-008 负责。
  - F-001 只提供 `.agentspace/harness-pipelines/` 目录归属和基础文件同步能力，不解析或发布 Harness CICD 流水线业务定义。
  - Harness 快照可以引用已发布 Harness CICD 流水线版本，但版本固定和运行关系由 F-008 维护。
- 测试与模拟：
  - AGENT.md 通过 Markdown 解析、路径、文件级元数据、引用、上下文预算和加载预览断言后进入测试通过。
  - 环境变量通过名称、作用域、类型、依赖引用和 Secret 写入读取探测后进入测试通过；探测过程不得返回敏感明文。
  - Tool 通过市场版本存在性、权限和授权可用性探测后进入测试通过；Skill 和 Agent 分别执行其配置的测试用例或测试对话。
  - Hook 测试用例至少包含事件载荷、作用对象、条件命中结果、预期决定、预期动作和审计断言；每个已配置事件与动作组合至少有一个测试用例。
  - 模拟环境不得调用真实外部 Tool、写入真实业务对象、发送真实通知、启动真实 Harness CICD 流水线或暴露 Secret；所有外部副作用使用 Mock 或沙箱适配器。
  - 全部必需测试用例通过且测试结果对应当前草稿摘要时，配置版本才能进入测试通过；草稿内容变化后原测试结果失效。
- Hook：
  - 支持事件：手工 Agent 会话开始、Tool 调用前、Tool 调用后、对象写入前、对象写入后、手工 Agent 会话完成、手工 Agent 会话失败、Harness CICD 流水线任务生命周期事件、Harness 发布前和 Harness 发布后。
  - 支持动作：注入上下文、校验、阻止、人工确认、执行 Skill、启动 Harness CICD 流水线、发送通知和写入审计。
  - 不允许任意脚本、Shell 命令或未登记事件。
  - `before` 类 Hook 同步执行，可阻止或要求确认；`after` 类 Hook 默认异步，不回滚已完成动作。
  - F-001 在发布事务原子写入前发出 `harness.publish.before`，在生效版本切换成功后发出 `harness.publish.after`；两类事件均由共享 Hook Engine 执行并持久化 `HookExecution`。
  - 创建发布请求时固定当前已发布且可用的发布 Hook 版本集、顺序和摘要；目标 Hook 草稿不进入本次 before/after Hook 快照，新发布 Hook 从下一次发布请求开始生效。
  - `harness.publish.before` 可以校验、阻止或创建人工确认；等待确认期间发布事务不写入文件，确认使用独立幂等键，超时按 Hook 失败策略结束。
  - 人工确认恢复发布前必须重新校验操作者权限、目标版本摘要、基准文件版本、依赖、测试结果和发布 Hook 集合摘要；任一变化都会使当前确认失效并要求重新发起发布。
  - `harness.publish.after` 只允许异步通知、审计、执行 Skill 或启动 Harness CICD 流水线，不得回滚已发布版本。
  - 同一 Hook 在同一调用链中最多执行一次；Hook 与 Harness CICD 流水线的组合调用深度最多 5 层，超限时终止新调用并记录审计。
  - Hook 编辑示例“高风险 Tool 调用审批”：事件为 `tool.call.before`；条件为 Tool 风险等级等于 `high`，或目标环境等于 `production`；动作为要求团队管理员人工确认，确认通过则继续，拒绝或超时则阻止，并写入审计；失败策略为超时默认阻止，高风险 Tool 不允许静默放行。
- 环境变量：
  - 环境变量只在资源声明依赖时提示配置，环境变量中心为空不影响空间使用。
  - 普通变量可保存结构化值；敏感值只能写入 Secret Service，AGENT.md、知识库、配置文件、日志、diff、测试输入输出和快照均不得保存或回显明文，只能保存安全引用。
  - 删除或停用环境变量前必须展示受影响资源；存在已发布依赖时不得静默删除。
- 文件布局与同步：

```text
/AGENT.md
/<knowledge-domain>/AGENT.md
/.agentspace/
  manifest.yaml
  agents/
  skills/
  hooks/
  harness-pipelines/
  tools.yaml
  env.yaml
```

  - 运行时只识别上述布局，不把历史配置目录作为运行时兼容布局。
  - `manifest.yaml` 记录 Schema 版本、资源索引、AGENT.md 文件级元数据和发布摘要。
  - `harness-pipelines/` 属于统一布局，目录内容和 HarnessPipelineDefinition Schema 由 F-008 管理。
  - Tool 与环境变量文件只保存引用，不保存市场连接参数或敏感明文。
  - Portal 保存先在暂存区完成 Schema、依赖、权限和序列化校验，再原子替换目标文件。
  - 外部文件解析失败时保留文件现状但不使其生效，也不覆盖上一生效版本。
- 历史配置导入：
  - 用户可以从知识库配置页或 Harness 总览显式选择“从历史配置导入”。
  - 系统仅扫描用户选定知识库目录下的 `.codex/`、`.claudecode/`、`.opencode/`，不会在运行时自动消费这些目录。
  - 导入结果只生成 Portal 草稿、依赖问题和市场匹配建议，不直接发布、不进入 Harness 快照。
  - 指引类文件导入为 AGENT.md 草稿；skill 或 prompt 类配置导入为 Skill 草稿；agent 或 subagent 类配置导入为 Agent 草稿，并将第三方 `subagent` 来源标记为 `source=subagent_config`。
  - Harness CICD 流水线类配置导入为 F-008 Harness CICD 流水线草稿候选；只有用户进入 F-008 完成结构化确认、测试和发布后才会生效。
  - MCP、server 或 tool 类配置只生成 Tool 市场匹配建议；找不到市场 Tool 时进入待补全，不导入连接地址、启动命令、认证参数或协议细节。
  - hook 或 settings 类配置只导入支持事件和支持动作；任意脚本、Shell 命令和未知生命周期事件标记为不支持，不能发布。
  - env 或 secret 类配置只导入变量名、作用域和依赖需求，不导入敏感值。
  - 导入草稿必须经过校验、测试、diff 和用户发布确认后才会写入 AGENT.md 或 `.agentspace`。
- 发布、版本和快照：
  - 当前生效版本是最近一次完成依赖校验、测试、文件写入和发布的成功版本。
  - 发布前必须展示配置 diff；基准版本变化时禁止静默覆盖。
  - Harness 基础类型统一为 `agent_md`、`skill`、`tool`、`agent`、`hook`、`env`；`Harness CICD 流水线` 由 F-008 管理。
  - 手工 Agent 会话快照固定解析后的 AGENT.md、Agent、Skill、Tool、Hook 版本和环境变量引用。
  - HarnessPipelineRun 快照中的 Harness CICD 流水线版本和依赖版本由 F-008 固定。
  - 已启动手工 Agent 会话或 HarnessPipelineRun 不受后续配置发布影响。
  - 外部文件变更只创建待确认草稿，不能绕过测试、diff 确认和显式发布门禁。
  - 资源可用性独立为可用、降级和阻断；它反映外部授权、市场版本和 Secret 等当前依赖，不改写不可变配置版本状态。
- 权限与审计：
  - 只有团队创建者和团队管理员可以维护、测试和发布 Harness 基础配置。
  - 团队成员可以使用已发布 Agent、Skill、Tool 和 Hook 影响下的会话能力，访客只读。
  - 草稿生成、市场选择、测试、发布、停用、冲突处理和失败结果均需记录审计。

### 边界与异常

- 用户创建空间后点击开始作业时，系统打开统一开始作业入口，默认选择 Agent，并使用平台默认 Agent 和空的空间级扩展配置。
- 非开发空间没有远程仓、代码语言和开发 Tool 时，知识库、AGENT.md 和普通会话仍可正常使用。
- AGENT.md 为空时不生成该层指引；内容超过平台预算时阻止发布并提示拆分到知识域或 Skill。
- 多知识域存在自由文本疑似语义冲突时，系统提示用户确认，不自动阻止；确认结果进入快照审计。
- 市场 Skill、Tool 或 Agent 不存在、下线、版本不兼容或无权限时，系统阻止选择或发布相关资源。
- Tool 授权失败或失效时，不得回显市场凭据，只显示授权状态和处理入口。
- 市场 Tool 版本下线或不可执行后，不允许生成包含该版本的新快照；已启动会话保留原版本引用，后续 Tool 调用失败时按 Agent 失败策略处理。
- 自定义 Skill 或 Agent 缺少必填字段、测试失败或依赖不完整时保持待补全或校验失败。
- Hook 使用不支持的事件或动作、形成同链重复调用或超过 5 层调用深度时被拒绝或终止。
- 敏感环境变量安全写入失败时，相关资源不能发布，且不得降级为明文。
- `.codex`、`.claudecode` 或 `.opencode` 存在但用户未选择导入时，系统忽略这些目录；导入后也只形成草稿，新配置仅在发布后写入 AGENT.md 或 `.agentspace`。
- Portal 与外部文件并发变化时进入冲突状态，保留双方内容和上一生效版本。
- 无 Portal 草稿的外部文件变化也只创建待确认草稿，不自动替换当前生效版本。
- 单个资源发布失败不改变其他已发布资源，也不阻止零配置普通会话。
- 权限在页面打开后被撤销时，后端拒绝后续写请求，前端切换为只读。

## 验收标准

### 正常路径

- 具备权限的用户创建空间后，可以跳过知识库和全部 Harness 基础配置，点击“开始作业”打开统一开始作业入口，并默认使用平台默认 Agent。
- 团队管理员可以创建业务知识域、上传资料，并在需要时绑定远程代码仓。
- 团队管理员可以分别创建空间级和知识域级 AGENT.md，并同步到标准文件路径。
- 手工 Agent 会话只加载空间级 AGENT.md 和本次命中的知识域级 AGENT.md。
- R-0630 可以通过 `knowledgeDomainIds` 明确选择任务知识域；R-0730 的 `@` 引用会自动补充所属知识域。
- 团队管理员可以从市场添加 Skill，也可以通过 AI 生成并确认自定义 Skill。
- 团队管理员可以从市场添加 Tool，Portal 不要求填写连接地址、启动命令或 MCP 参数。
- 团队管理员可以从市场添加 Agent，也可以新建自定义 Agent；未指定空间默认 Agent 时继续使用平台默认 Agent。
- 团队管理员可以从模板或自然语言创建结构化 Hook，并通过模拟事件测试。
- 团队管理员可以配置“高风险 Tool 调用审批”类 Hook，并在 `tool.call.before` 模拟事件中验证确认、拒绝、超时和审计结果。
- 团队管理员可以从 `.codex`、`.claudecode`、`.opencode` 导入 Harness 草稿，导入后不会直接发布。
- Hook 只有在隔离模拟中满足当前版本全部必需断言后才能进入测试通过。
- 系统在资源缺少 Tool 或环境变量时就地提示补充，补充后重新校验原资源。
- 环境变量中心为空时空间仍可使用；敏感值保存后只产生安全引用。
- 测试通过的资源可以查看 diff 并发布，发布后形成当前生效版本。
- 运行快照只解析 AGENT.md 和 `.agentspace` 布局，并使用已发布的 Harness 基础资源。
- 编辑已发布资源时，上一已发布版本与新草稿版本可以并存，新会话继续使用上一已发布版本。
- AGENT.md、环境变量、Tool、Skill、Agent 和 Hook 分别按对应确定性测试契约进入测试通过。
- Harness CICD 流水线入口跳转 F-008，不在 F-001 Harness 基础配置中编辑或发布 Harness CICD 流水线。

### 边界场景

- 所有配置中心为空时，手工 Agent 会话仍能使用平台默认 Agent。
- 只有某个 Skill 缺少 Tool 时，仅该 Skill 不能发布，其他资源和会话不受影响。
- 没有远程代码仓的客服、法务或运营空间可以使用上传资料、AGENT.md、Skill 和默认 Agent。
- 空间级 AGENT.md 存在、知识域级 AGENT.md 为空时，只加载空间级指引。
- 多个知识域被会话同时选择时，系统按空间级后知识域级顺序生成快照；疑似语义冲突只提示用户确认。
- Tool 市场版本下线后，系统阻止生成包含该版本的新快照；已启动会话保留原引用并按失败策略处理后续调用。
- 已发布 Tool 或其依赖失效时，配置版本仍为已发布，但资源可用性变为阻断。
- Hook `after` 动作失败时记录失败，不回滚已完成主动作。
- 资源停用后不进入新快照，已开始手工 Agent 会话和 HarnessPipelineRun 继续使用原快照。

### 异常场景

- AGENT.md 超出上下文预算或引用不存在知识域时，系统阻止发布并定位问题。
- 多知识域 AGENT.md 存在疑似语义冲突时，系统只提示用户确认，不因模型判断自动阻止快照。
- `.agentspace/manifest.yaml` 中 AGENT.md 文件级元数据格式错误、路径不合法或内容 hash 不匹配时，系统阻止该版本测试和发布。
- 用户尝试新建 Tool 或编辑 Tool 底层连接参数时，系统不提供入口且后端拒绝请求。
- 市场资源不可用、版本不兼容或授权失败时，相关草稿版本保持待补全；相关已发布版本保持不可变，但资源可用性变为阻断。
- Hook 包含任意脚本、不支持事件或形成循环时，系统阻止发布。
- Hook 与 Harness CICD 流水线组合调用超过 5 层时，系统终止新调用并记录调用链。
- 敏感环境变量写入失败时，系统不产生明文文件或快照。
- 在 AGENT.md、知识库、日志、diff、模拟测试记录和 Harness 快照中检索敏感值明文时均无结果。
- `.codex`、`.claudecode` 或 `.opencode` 中存在有效旧配置但用户未选择导入时，系统不扫描、不导入且不发布。
- 用户选择历史配置导入时，MCP、server 或 tool 配置只生成 Tool 市场匹配建议，不导入连接参数；subagent 配置导入为 Agent 草稿；Harness CICD 流水线类配置转入 F-008 草稿候选；脚本类 Hook 标记为不支持。
- 外部文件解析失败或 Portal 保存发生版本冲突时，系统保留上一生效版本。

### 权限场景

- 团队创建者和团队管理员可以创建、测试、发布和停用 Harness 基础资源。
- 团队成员不能修改 Harness，但可以在会话中使用已发布 Agent、Skill、Tool 和 Hook 治理后的能力。
- 访客不能启动会话，也不能执行 Harness 写操作。
- 租户管理员未被授予团队空间管理员角色时，不能配置该空间 Harness。
- 权限撤销后，已打开页面的后续发布请求被后端拒绝。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：移出 Harness CICD 流水线配置中心，保留零配置开始作业、AGENT.md、Hook 中心、Tool 市场选择、AI 配置入口、依赖补全和统一发布状态。

### 页面与交互

- 空间创建完成页：
  - 主操作为“开始作业”，点击后打开统一开始作业入口并默认选择 Agent；次操作为“完善 Harness”。
  - 不展示强制完成百分比，不把空中心标记为空间异常。
- Harness 总览：
  - 展示 AGENT.md、Skill、Tool、Agent、Hook 和环境变量六个基础中心。
  - 提供进入 Harness CICD 流水线的独立入口，说明 Harness CICD 流水线配置与任务运行由 F-008 管理。
  - 展示已发布资源、待补全资源、依赖问题和最近发布记录。
  - 提供“AI 帮我配置”“从市场选择”“手动创建”“从历史配置导入”；不适用入口按资源隐藏。
- AGENT.md 中心：
  - 展示空间级指引和顶层知识域列表。
  - 编辑器提供章节模板、自然语言生成、上下文预算和加载范围预览。
  - 编辑器只展示 Markdown 正文、章节引导和加载预览；不提供额外规则配置表单，文件级元数据由 Portal 保存到 `.agentspace/manifest.yaml`。
- Skill 与 Agent：
  - 列表区分市场引用和空间自定义资源。
  - 自定义流程展示 AI 草稿、结构化表单、依赖、测试和发布步骤。
- Tool 中心：
  - 只有“从 AI 市场添加 Tool”入口。
  - 卡片展示能力、版本、权限、授权状态和引用关系，不展示 MCP 配置表单。
- Hook 中心：
  - 首屏提供审批、通知、审计和发布检查模板。
  - 自定义 Hook 使用事件、条件、动作、失败策略和作用范围表单。
  - 示例模板包含“高风险 Tool 调用审批”，展示 `tool.call.before`、高风险条件、团队管理员确认、超时阻止和审计动作。
- 历史配置导入：
  - 从知识库配置页或 Harness 总览进入。
  - 先展示可导入目录、映射预览、不支持项和敏感值处理结果，再允许用户生成草稿。
  - Harness CICD 流水线类导入项标记为需要进入 F-008 继续确认。
- 环境变量：
  - 支持从依赖问题直接打开并定位变量。
  - 敏感值不回填，保存后只展示已配置、缺失或轮换状态。

### 状态与文案

- 配置版本状态：草稿、待补全、校验失败、测试通过、已发布、已停用。
- 空状态：`当前空间使用平台默认 Agent，可直接开始会话`。
- Harness CICD 流水线入口提示：`Harness CICD 流水线配置与任务运行由独立 Harness CICD 流水线中心管理`。
- 依赖提示：`该 Skill 还需要 1 个 Tool 和 2 个环境变量，补充后可测试`。
- Tool 提示：`Tool 由 AI 市场统一配置，AgentSpace 仅添加已发布版本`。
- Hook 风险提示：`该 Hook 会阻止会话操作，请先完成模拟测试`。
- 冲突提示：`配置文件已被外部更新，请处理差异后再发布`。
- 导入提示：`历史配置只会导入为草稿，确认发布后才会生效`。

## 前端设计

### 是否需要刷新

- 结论：是
- 理由：需要六个基础中心导航、Harness CICD 流水线独立入口、分资源创建模式、依赖图、配置状态机和 Hook 模拟测试。

### 页面、组件与状态

- 核心页面：
  - 空间创建完成页。
  - Harness 总览与六个基础配置中心。
  - Hook 规则编辑与模拟运行页。
- 核心组件：
  - `HarnessOverview`、`HarnessResourceCard`、`HarnessStateBadge`、`HarnessPipelineCenterEntry`。
  - `AgentMdEditor`、`KnowledgeDomainScopePicker`、`ContextBudgetMeter`。
  - `AiDraftAssistant`、`StructuredConfigForm`、`ConfigDiffViewer`。
  - `MarketResourcePicker`、`ToolReferenceCard`、`AuthorizationStatus`。
  - `DependencyResolver`、`EnvVarEditor`、`SecretInput`。
  - `HookTemplateGallery`、`HookRuleBuilder`、`HookSimulationPanel`。
  - `LegacyHarnessImportWizard`、`LegacyConfigMappingPreview`、`UnsupportedLegacyItemList`。
- 状态管理：
  - 配置版本状态：`draft`、`incomplete`、`validation_failed`、`tested`、`published`、`disabled`。
  - 资源状态保存当前生效版本和编辑中版本，允许“已发布 + 有未发布修改”同时展示。
  - 资源可用性：`available`、`degraded`、`blocked`，与配置版本状态分开展示。
  - 依赖状态：满足、缺失、授权失效、版本不兼容。
  - 同步状态：未保存、暂存校验中、冲突、写入中、已同步、失败。
  - 零配置状态独立于异常状态，不显示红色告警。

### 接口依赖与异常处理

- 依赖接口：
  - 查询六类基础资源、依赖图、当前生效版本和发布历史。
  - 生成 AI 配置草稿和依赖建议。
  - 查询市场 Skill、Tool 和 Agent，添加市场资源引用并发起授权。
  - 扫描用户选定知识库目录中的历史配置目录并生成映射预览。
  - CRUD 自定义 AGENT.md、Skill、Agent 和 Hook 草稿。
  - 校验、测试、模拟运行、查看 diff、发布和停用基础资源。
  - 查询和补充环境变量引用。
  - 跳转 F-008 Harness CICD 流水线中心。
- 异常处理：
  - AI 草稿生成失败时保留用户描述并允许手动创建。
  - 市场不可用时不影响已发布资源和零配置会话。
  - 单资源测试或发布失败时保留草稿，不刷新其他资源状态。
  - 版本冲突时进入差异处理，不自动重试覆盖。
  - 历史配置导入失败时保留扫描结果和错误详情，不创建半成品已发布资源。
  - 敏感值不进入前端持久化缓存、日志或错误详情。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：需要明确 F-001 只管理 Harness 基础资源、市场引用、AGENT.md 解析、依赖图、Hook Schema、统一发布状态和新文件布局；Harness CICD 流水线定义交由 F-008。

### 接口与数据

- 核心服务：
  - `HarnessResourceService`：统一管理基础资源、状态和版本。
  - `AgentMdService`：解析空间级与知识域级指引并生成加载结果。
  - `HarnessDependencyService`：解析资源依赖和受影响关系。
  - `MarketResourceService`：查询并引用市场 Skill、Tool 和 Agent。
  - `HarnessDraftAssistant`：将自然语言转换为结构化草稿，不具备发布权限。
  - `LegacyHarnessImportService`：扫描用户选定知识库目录中的历史配置目录，生成草稿、F-008 Harness CICD 流水线候选和市场 Tool 匹配建议。
  - `HarnessValidationService`：执行 Schema、依赖和权限校验。
  - `HarnessTestService`：执行 Skill/Agent 测试、Hook 模拟和 Tool 授权检查。
  - `HarnessPublishService`：生成 diff、校验基准版本、原子写入并切换生效版本。
  - `SecretService`：托管敏感环境变量。
  - `AuditService`：记录配置和发布操作。
- 核心数据对象：
  - `HarnessResource`：资源 ID、空间 ID、类型、来源、当前生效版本 ID 和编辑中版本 ID。
  - `HarnessResourceVersion`：资源版本 ID、配置状态、内容摘要、依赖摘要、测试结果、创建人与时间。
  - `HarnessResourceType`：`agent_md`、`skill`、`tool`、`agent`、`hook`、`env`。
  - `AgentMdConfig`：作用范围、知识域 ID、内容摘要、上下文预算、文件路径、来源和发布状态。
  - `AgentMdFileMetadata`：文件路径、作用范围、版本、来源、内容 hash 和发布状态，序列化到 `.agentspace/manifest.yaml`。
  - `MarketResourceReference`：市场资源 ID、类型、版本、能力快照、权限摘要和授权引用。
  - `HarnessDependency`：资源、依赖类型、依赖目标、是否必需和状态。
  - `HarnessTestCase`：资源版本、测试类型、输入、Mock、预期决定、动作断言和输出断言。
  - `HarnessTestResult`：资源版本摘要、测试用例、执行环境、断言结果和时间。
  - `LegacyHarnessImportJob`：导入任务、扫描目录、来源类型、状态、映射摘要和操作者。
  - `LegacyHarnessImportItem`：来源路径、来源类型、目标资源类型、映射状态、不支持原因和生成草稿 ID。
  - `HookDefinition`：作用范围、事件、条件、动作、同步性、失败策略和超时。
  - `EnvVarReference`：名称、敏感标识、值或安全引用和依赖关系。
  - `HarnessVersion`：配置摘要、文件摘要、测试结果、发布人和发布时间。
  - `HarnessResourceAvailability`：资源、可用性、依赖原因和最近探测时间。
  - `HarnessPublishRequest`：目标版本摘要、基准文件版本、测试结果摘要、发布 Hook 版本集摘要、状态和确认 ID。
  - `HookExecution`：Hook 版本、事件、作用对象、同步性、决定、调用链和结果，由共享 Hook Engine 持久化。
  - `RuntimeConfirmation`：来源类型、来源执行 ID、题干、选项、状态、超时、决定、操作者和幂等键。

### 业务规则、权限与异常处理

- 零配置空间生成空扩展快照并引用平台默认 Agent。
- Harness 运行时文件读写只使用 AGENT.md 和 `.agentspace` Schema；历史配置解析器只服务于用户显式导入草稿，不参与会话快照生成。
- AI 草稿接口只能创建草稿版本，发布接口必须由具备管理员权限的用户显式调用。
- Tool 创建接口只接受市场资源引用，不接受连接参数。
- 历史配置中的 MCP、server 或 tool 连接参数不得写入 AgentSpace 配置，只能用于一次性市场 Tool 匹配建议。
- 历史配置中的 subagent 只映射为 Agent 草稿，不新增 Subagent 资源类型或配置中心。
- 历史配置中的 Harness CICD 流水线类配置只形成 F-008 草稿候选，不由 F-001 发布或运行。
- 发布事务依次执行依赖校验、测试结果校验、基准版本校验、固定发布 Hook 集合、暂存序列化、原子写入和生效版本切换。
- 发布事务在原子写入前调用 `harness.publish.before`；需要人工确认时创建 `RuntimeConfirmation` 并暂停。恢复时重新校验权限、目标摘要、基准版本、依赖、测试和 Hook 集合摘要，全部不变才允许继续；生效版本切换后按固定 Hook 集合异步调用 `harness.publish.after`。
- AGENT.md 解析按空间级后知识域级和 `domainId` 排序合并；疑似语义冲突只提示用户确认，不由系统自动阻断。
- AGENT.md 读写使用纯 Markdown；Portal 不向 AGENT.md 写入发布元数据，来源、内容 hash 和发布状态等文件级元数据统一写入 `.agentspace/manifest.yaml`。
- Hook Schema 拒绝脚本动作和未知事件；发布校验调用链最大深度为 5。
- 环境变量敏感值在进入持久层前转换为安全引用。
- 任一资源发布失败不回滚其他已发布资源。
