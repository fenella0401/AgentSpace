## 变更摘要

- 本次变更：新增团队空间事件源配置与事件执行看板，支持管理员配置公司需求系统、缺陷系统等事件源，按事件类型设置自动同步时间、触发目标和同一时间可触发的 Agent 会话或 Harness CICD 流水线任务数量。
- 核心口径：
  - 事件源负责把外部系统中新需求、新问题单等结构化事件同步到 AgentSpace，不替代外部系统自身的需求或缺陷流转。
  - 每类事件进入后可以触发新建 `source=event_source` 的 Agent 会话，或触发新建来源为事件源的 Harness CICD 流水线任务。
  - 事件执行看板只聚合事件触发任务状态；点击 Agent 会话任务复用 F-006 Agent 会话详情原子功能，点击流水线任务复用 F-008 Harness CICD 流水线任务详情原子功能。
  - 同一事件类型的并发上限用于限制同一时间触发的 Agent 会话和 Harness CICD 流水线任务数量，超出部分保留在 backlog。
- 影响范围：产品定义、功能索引、F-002 权限矩阵、F-006 Agent 会话来源与详情展示、F-008 Harness CICD 流水线任务来源、事件源适配器、调度器、审计、测试用例和后续发布计划。
- 待确认问题：
  - F-009 纳入的目标版本、首批事件源类型和首批事件类型清单。
  - 事件源配置权限由系统管理员、租户管理员还是团队管理员分层承担，以及事件任务的审计主体。
  - 自动同步时间是否只支持固定间隔和每日定时，是否需要 cron 表达式和工作日窗口。
  - 事件 payload 中哪些字段允许进入 Agent prompt、流水线输入、看板摘要和日志。
  - wait review 到 done 的完成确认由事件看板直接确认，还是必须进入原子详情页完成。

## PRD 设计

### 用户目标

- 团队创建者或团队管理员可以在团队空间内配置事件源，将公司需求系统的新需求、缺陷系统的新问题单等外部事件接入 AgentSpace。
- 管理员可以按事件类型设置自动同步时间、过滤条件、去重键、输入映射、触发目标和启停状态。
- 管理员可以为每类事件设置同一时间可触发的 Agent 会话数量和 Harness CICD 流水线任务数量，避免外部事件风暴压垮执行资源。
- 系统可以按计划同步事件，将未触发的事件任务放入 backlog，并在并发额度允许时自动创建 Agent 会话或 Harness CICD 流水线任务。
- 空间用户可以通过事件执行看板查看事件触发任务在 backlog、agent working、wait review 和 done 中的处理状态。
- 空间用户可以从事件执行看板点击查看某个 Agent 会话任务或 Harness CICD 流水线任务的原子执行详情。

### 业务规则

- 权限与配置范围：
  - 团队创建者和团队管理员可以在团队空间内配置事件源、事件类型触发规则、并发上限和启停状态。
  - 团队成员和访客不可修改事件源配置；团队成员可按空间权限查看事件执行看板，访客只读。
  - 系统管理员和租户管理员是否提供事件源类型开放、租户级授权和全局并发保护，作为待确认分层治理能力。
- 事件源与事件类型：
  - 事件源类型首批示例为公司需求系统和缺陷系统；每个事件源实例归属一个团队空间。
  - 每个事件源可以配置多个事件类型规则，例如新需求、新问题单。
  - 事件类型规则必须声明同步时间、过滤条件、去重键、触发目标、输入映射、并发上限和启停状态。
  - 自动同步时间至少需要支持固定间隔和每日定时；cron 表达式、工作日窗口和节假日策略待确认。
- 去重与同步：
  - 去重键默认由 `eventSourceId + externalEventType + externalEventId` 组成；管理员可以在平台允许字段内调整。
  - 同步任务必须保存水位、外部更新时间和最近同步结果，避免重复创建任务。
  - 外部事件字段变化时更新 EventExecutionTask 摘要和 payload 快照，不覆盖已启动原子执行对象的执行快照。
  - 同步失败需要保留错误、重试次数和下一次同步时间，不清空已有 backlog。
- 触发目标：
  - 每条事件类型规则只能选择一种默认触发目标：Agent 会话或 Harness CICD 流水线任务。
  - 触发 Agent 会话时，系统创建 `source=event_source` 的 AgentSession，并记录 eventSourceId、eventExecutionTaskId、externalEventId 和触发规则版本。
  - 触发 Harness CICD 流水线任务时，系统创建来源为事件源的 HarnessPipelineTask，并记录 eventSourceId、eventExecutionTaskId、externalEventId 和触发规则版本。
  - 触发目标的 Agent、Harness CICD 流水线、知识域、Tool 和环境变量必须在触发时重新校验权限和依赖可用性。
  - 输入映射只允许把白名单字段写入任务说明、Agent 首轮输入或流水线输入，敏感字段默认不可进入 prompt、日志或看板摘要。
- 并发与调度：
  - 每个事件类型规则分别配置同一时间可触发的 Agent 会话数量或 Harness CICD 流水线任务数量。
  - 当前运行数量达到上限时，新事件任务保持 backlog，不创建原子执行对象。
  - backlog 默认按外部事件创建时间、进入 AgentSpace 时间和重试次数排序；优先级策略待确认。
  - 暂停事件源或事件类型规则后，不再创建新的 EventExecutionTask 或触发新原子执行对象；已运行对象不被自动取消。
- 事件看板状态：
  - backlog：事件已同步并等待触发，或因并发上限、暂停、依赖待恢复而暂未触发。
  - agent working：事件已触发原子执行对象，Agent 会话或 Harness CICD 流水线任务处于自动执行中；流水线目标的真实节点状态以 F-008 为准。
  - wait review：原子执行对象进入等待人工审阅、确认或 Agent 询问的卡点；实际处理入口在 F-006 或 F-008 原子详情中。
  - done：原子执行对象已完成，或人工审阅/确认后被判定完成；完成确认来源必须可审计。
  - 事件看板状态由关联 AgentSession 或 HarnessPipelineTask 状态映射得到，不允许看板与原子执行对象出现互相矛盾的终态。
- 原子详情复用：
  - 事件看板不展示完整 Agent 运行日志、Run 图、节点输出或文档 diff。
  - 点击 Agent 会话任务进入 F-006 Agent 会话详情；F-006 需要识别 `source=event_source` 并展示事件来源摘要。
  - 点击 Harness CICD 流水线任务进入 F-008 Harness CICD 流水线任务详情；F-008 需要识别任务来源为事件源并展示事件来源摘要。

### 边界与异常

- 事件源授权失效、外部系统不可达或字段 Schema 变化时，系统停止本轮同步并展示错误，不删除既有事件任务。
- 外部事件缺少必需字段、去重键为空或输入映射失败时，事件进入同步异常记录，不触发 Agent 会话或流水线任务。
- 目标 Agent、Harness CICD 流水线、知识域或依赖失效时，事件任务留在 backlog 并展示依赖失效原因。
- 并发上限为 0 时视为暂停触发；已运行对象继续执行。
- 同一外部事件重复同步时，不重复创建 EventExecutionTask；若原事件已 done，仅更新最近同步信息和审计记录。
- Agent 会话或流水线任务失败时，事件任务展示失败原因并保留重新触发入口是否开放的待确认问题。
- wait review 超时后，事件任务保持 wait review 或转为 backlog/失败的策略待确认；原子执行对象的超时规则优先。
- 用户无权访问原子详情时，看板仅展示其有权查看的事件摘要，不泄露 Agent 输出、流水线节点或外部系统敏感字段。
- 事件源同步、触发调度和原子执行对象创建必须具备幂等键；服务重启后不能重复触发同一事件。

### PRD Review 结论

- 合理性：需求被拆为事件源配置、事件类型触发策略、同步与去重、并发调度、事件执行看板和原子详情复用六个部分，能够覆盖用户提出的核心场景。
- 一致性：PRD 明确 Agent 会话和 Harness CICD 流水线任务仍是原子执行对象，事件看板只聚合状态并跳转 F-006/F-008，避免重复建设详情能力。
- 已修正点：将 agent working 定义为事件触发后的自动执行阶段；当目标为流水线时，真实节点状态以 F-008 为准，避免状态文案与流水线目标冲突。
- 待确认点：版本范围、首批事件源、同步时间表达能力、审计主体、payload 字段白名单和 wait review 完成确认入口。
