## 变更摘要

- 本次变更：适配 F-001/F-008 拆分后的新模型，团队模板可选预置知识库、AGENT.md、市场 Skill 引用、市场 Tool 引用、在线 Agent 草稿、环境变量引用和 AgentFlow 定义。
- 核心原则：模板不是空间开始作业的前置条件；未选择模板、模板为空或模板部分应用失败时，空间仍可使用平台默认 Agent。
- 影响范围：全局和租户模板管理、空间初始化、市场资源引用、AgentFlow 定义预置、模板应用状态和验收标准。
- 待确认问题：
  - 模板版本、发布、停用、复制、回滚和历史版本保留策略。
  - 模板应用后是否允许整体重新应用；首期建议只允许逐项维护。
  - 全局模板与租户模板同名时的排序和展示规则。

## PRD 设计

### 用户目标

- 系统管理员可以维护全局团队模板，沉淀公司级空间初始化实践。
- 租户管理员可以维护本租户模板，适配租户业务规则和常用能力。
- 团队创建者可以选择模板快速初始化知识库和 Harness，也可以完全跳过模板。
- 模板中的每一类内容都是可选项，不要求开发项目字段或远程代码仓。
- 模板应用失败时，用户可以继续使用默认 Agent，并单独重试失败项。

### 业务规则

- 模板范围分为全局和租户级，分别由系统管理员和租户管理员维护。
- 团队创建者只能看到启用的全局模板和当前租户模板。
- 空间初始化最多选择一个模板，也可以跳过模板。
- 模板中的以下内容全部可选：
  - 知识库顶层知识域和子文件夹。
  - 空间级与知识域级 AGENT.md。
  - 市场 Skill 和 Tool 引用。
  - 在线 Agent 草稿，字段包括名称、描述、Skill、Tool 和预置 prompt。
  - AgentFlow 定义候选，应用时转入 F-008 AgentFlow 发布流程。
  - 环境变量名称、敏感标识和安全引用占位。
  - 可选远程代码仓绑定。
- 模板不得包含敏感环境变量明文或市场 Tool 底层连接参数。
- 模板不生成或识别 `.codex`、`.claudecode`、`.opencode` 等历史配置目录。
- Tool 只能保存市场资源引用；市场授权必须在目标空间上下文重新确认。
- Skill 只能保存市场资源引用；模板不支持新建 Skill。
- Agent 不保存市场引用，只生成在线 Agent 草稿。
- 模板应用按资源独立记录状态，单项失败不回滚已成功项。
- 模板应用项的成功、待补全和失败属于 `TemplateApplicationItem` 状态，不替代 F-001 的 Harness 配置版本状态。
- 模板应用完成前后，空间都可以使用平台默认 Agent。
- 模板应用后，Harness 基础资源按照 F-001 独立维护和发布，AgentFlow 按照 F-008 独立维护和发布；模板后续更新不自动覆盖已应用空间。

### 边界与异常

- 没有可用模板或用户跳过模板时，系统正常创建零配置空间。
- 模板只包含 AGENT.md 或知识库时，可以正常发布和应用。
- 非开发模板不配置远程代码仓和开发 Tool 时，不产生缺失告警。
- 市场资源已下线、版本不兼容或授权失败时，该项进入待补全，其他项继续应用。
- 模板中的环境变量未提供值时，只创建依赖提示，不阻止空间使用。
- AgentFlow 定义模拟测试失败时不发布该资源，其他模板内容保持成功。
- 重试模板失败项必须幂等，不得重复创建知识域或资源。

## 验收标准

### 正常路径

- 系统管理员可以创建包含任意可选 Harness 内容的全局模板。
- 租户管理员可以创建只对本租户可见的模板。
- 团队创建者可以选择模板，也可以跳过模板直接创建空间。
- 模板可以预置 AGENT.md、市场 Skill、市场 Tool、在线 Agent 草稿、环境变量引用和 AgentFlow 定义候选。
- 模板应用成功项进入已发布状态，空间可使用对应资源。
- 模板应用失败项可以在补全依赖后单独重试。

### 边界场景

- 空模板、仅知识库模板或仅 AGENT.md 模板可以发布和应用。
- 模板没有远程代码仓时，不影响非开发空间使用。
- 环境变量缺失只阻止依赖资源，不阻止空间和默认 Agent。
- 模板后续更新不自动改变已应用空间。

### 异常场景

- Tool 不是市场引用时，系统阻止模板发布。
- 模板包含敏感环境变量明文、历史配置目录布局、Hook 配置或自定义 Skill 时，系统阻止发布。
- 市场资源不可用或模拟测试失败时，仅对应项失败。
- 重试失败项时不会重复创建已成功资源。

### 权限场景

- 非系统管理员不能维护全局模板。
- 租户管理员只能维护所属租户模板。
- 团队创建者和团队管理员可以查看模板应用结果并补全目标空间依赖。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：模板编辑器需要适配 Harness 基础资源、AgentFlow 定义候选、可选项、零配置跳过和分项恢复。

### 页面与交互

- 模板编辑页以模块卡片组织知识库、AGENT.md、市场 Skill、市场 Tool、在线 Agent、环境变量和 AgentFlow 定义候选。
- 每个模块默认未启用，管理员按需添加。
- Tool 模块只提供市场选择器。
- AgentFlow 使用 F-008 画布预览能力。
- 空间创建向导提供“跳过模板，直接开始”和模板选择。
- 模板应用状态按资源展示成功、待补全、失败和重试。

### 状态与文案

- 空模板提示：`该模板不预置内容，空间将使用平台默认 Agent`。
- 跳过提示：`稍后仍可在 Harness 中逐项配置`。
- 部分失败提示：`空间已可使用，3 项模板内容需要处理`。

## 前端设计

### 是否需要刷新

- 结论：是
- 理由：需要模块化模板编辑器、市场资源选择、模板预览和分项应用状态。

### 页面、组件与状态

- `TeamTemplateEditor`、`TemplateModuleCard`、`TemplateAgentMdEditor`。
- `TemplateMarketResourcePicker`、`TemplateAgentFlowPreview`、`TemplateAgentDraftEditor`。
- `TemplatePicker`、`TemplateApplicationStatus`、`TemplateRetryAction`。
- 状态按模板草稿、校验失败、已发布，以及独立的应用项成功、待补全、失败管理；应用成功后创建的 Harness 基础版本使用 F-001 状态机，AgentFlow 使用 F-008 状态机。

### 接口依赖与异常处理

- 查询、创建、更新、发布和停用全局或租户模板。
- 查询模板可用市场资源并执行模拟应用。
- 创建空间并可选应用模板。
- 查询模板应用项并单项重试。
- 模板应用失败时保留空间入口和默认 Agent 作业能力。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：需要模板模块、市场引用、分项应用和幂等重试。

### 接口与数据

- `TeamTemplateService`：维护模板和版本。
- `TemplateValidationService`：校验范围、依赖和 `.agentspace` 布局。
- `TemplateApplicationService`：按项应用模板并记录状态。
- `TemplateRetryService`：幂等重试失败项。
- `TeamTemplate`：范围、租户、名称、状态、版本和模块定义。
- `TemplateResourceDefinition`：资源类型、来源、配置或市场引用。
- `TemplateApplicationItem`：空间、资源类型、状态、失败原因和重试键。

### 业务规则、权限与异常处理

- 模板发布前执行 F-001/F-008 同类资源校验，但不要求所有模块存在。
- 应用时重新校验市场资源可见性、授权和环境变量依赖。
- 每个应用项具有独立幂等键和状态，不使用全量事务回滚。
- 模板停用不影响历史应用结果。
