## 变更摘要

- 本次变更：初始化团队模板配置与应用的完整草稿设计，支持后续将需求放入 R-0730 迭代继续澄清和调整。
- 影响范围：产品定义、功能索引、团队空间初始化、知识库文件夹体系、远程配置仓绑定、Harness 配置解析、全局管理面、租户管理面、前端自动渲染和验收标准。
- 待确认问题：
  - 团队模板的最终字段、版本策略、发布/下线流程和回滚能力。
  - 模板应用后是否允许二次编辑、重新应用或覆盖已有团队空间配置。
  - 远程配置仓权限、凭据来源、分支策略和同步失败处理。
  - 全局模板和租户级模板的优先级、可见范围、复制和继承规则。
  - 模板中 Agent 使用配置的标准目录结构、配置格式和兼容版本。

## PRD 设计

### 用户目标

- 系统管理员可以预置全局团队模板，沉淀公司级团队空间初始化标准。
- 租户管理员可以维护租户级团队模板，适配本租户团队的知识库结构和 Agent 使用配置。
- 团队管理员初始化团队空间时，可以选择一个团队模板快速完成空间初始化。
- 系统应用模板后，自动初始化团队空间知识库文件夹体系，并预置关联一个包含 Agent 使用配置的远程仓。
- 远程配置仓同步后，Portal 前端可以自动渲染模板带来的 Harness 配置。

### 范围

**包含：**

- 全局团队模板和租户级团队模板的管理入口。
- 团队模板基础信息、适用范围、状态和描述。
- 团队模板中的知识库文件夹体系定义。
- 团队模板预置远程配置仓绑定，用于关联包含 Agent 使用配置的仓库。
- 团队管理员创建团队空间时选择模板。
- 模板应用后初始化团队空间知识库文件夹体系。
- 模板应用后创建远程配置仓绑定，并触发同步和配置解析。
- 同步成功后，前端按现有 Harness 配置中心能力自动渲染 Agent、Skill、MCP、Workflow 和环境变量配置。
- 模板应用记录和基础审计。

**不包含：**

- 模板格式、版本、回滚和冲突策略的最终方案。
- 模板市场、模板评分、模板推荐和模板审批流。
- 跨租户共享租户级模板。
- 对已存在团队空间批量套用模板。
- 模板应用后覆盖已有知识库和 Harness 配置的策略。
- 远程仓平台账号体系、凭据审批和仓库权限治理。
- Agent 运行时执行、调度和编排能力。

### 核心流程

1. 系统管理员进入全局管理面，创建或维护全局团队模板。
2. 租户管理员进入租户管理面，创建或维护本租户团队模板。
3. 管理员配置模板基础信息、模板范围、知识库文件夹体系和远程配置仓绑定。
4. 团队管理员在所属租户下创建团队空间，进入空间初始化向导。
5. 系统展示可用模板列表，包括全局模板和当前租户的租户级模板。
6. 团队管理员选择一个模板，也可以在需求明确后支持跳过模板。
7. 系统创建团队空间，并根据模板初始化知识库文件夹体系。
8. 系统根据模板预置远程配置仓绑定，触发仓库同步。
9. 同步完成后，系统扫描配置仓中的 Agent 使用配置，例如 `.codex` 目录。
10. 配置解析结果进入 Harness 配置中心，前端按类型自动渲染。
11. 系统记录 `TemplateApplication`，包含模板、团队空间、操作者、应用时间、状态和结果。

### 业务规则

- 团队模板按范围分为全局模板和租户级模板，分别由系统管理员和租户管理员维护。
- 团队管理员创建团队空间时，只能看到全局可用模板和当前租户可用模板。
- 同一次团队空间初始化最多选择一个团队模板。
- 模板应用应在团队空间创建阶段执行，首期不设计对既有空间的重复应用。
- 模板中的知识库文件夹体系应以声明式结构表达，应用后生成团队空间实际文件夹。
- 模板中的远程配置仓绑定应生成团队空间级 `TemplateRepositoryBinding` 或文件夹绑定记录。
- 远程配置仓同步后的 Harness 渲染沿用 F-001 的配置扫描、解析和展示链路。
- 模板应用失败时，系统需要记录失败阶段，便于团队管理员重试或改为手动配置。
- 全局模板和租户级模板同名时的展示、排序和优先级待确认；第一版建议明确展示模板范围。
- 模板状态至少区分草稿、启用、停用；是否支持发布审批待确认。

### 边界与异常

- 当前租户没有租户级模板时，仍可展示全局模板。
- 没有任何可用模板时，团队管理员应可以进入手动初始化流程，是否强制模板待确认。
- 模板文件夹体系为空时，系统只创建默认知识库根目录或阻止启用模板，规则待确认。
- 远程配置仓绑定失败时，知识库文件夹可保留，绑定状态标记失败。
- 远程配置仓同步失败时，Harness 配置中心展示同步失败或空状态。
- 配置解析失败时，不覆盖模板应用记录，展示解析错误并允许后续重试。
- 模板停用后，不影响已经应用模板的团队空间；是否允许重新查看模板内容待确认。

## UI 设计

### 是否需要刷新

- 结论：是
- 理由：新增模板管理入口、模板配置表单、模板选择步骤、模板应用状态和自动渲染反馈。

### 页面与交互

- 全局模板管理页：
  - 系统管理员查看全局模板列表，支持创建、编辑、启用、停用模板。
  - 模板列表展示名称、状态、更新时间、适用说明和引用次数。
- 租户模板管理页：
  - 租户管理员查看本租户模板列表，支持维护租户级模板。
  - 页面需明确当前租户上下文。
- 模板编辑页：
  - 基础信息：模板名称、描述、范围、状态。
  - 知识库结构：以树形方式配置文件夹体系。
  - 远程配置仓：配置仓库地址、分支或引用、配置目录和凭据引用。
  - Agent 使用配置预览：展示可解析配置类型，具体解析规则待技术确认。
- 团队空间初始化向导：
  - 新增模板选择步骤，展示全局模板和租户级模板。
  - 模板卡片展示模板名称、范围、描述、更新时间和包含内容摘要。
  - 应用后展示知识库初始化、远程仓绑定、同步和解析状态。

### 状态与文案

- 空状态：暂无可用模板、暂无租户级模板、模板未配置知识库结构。
- 加载态：模板加载中、模板保存中、模板应用中、配置仓同步中、Harness 配置解析中。
- 成功态：模板已保存、模板已启用、模板已应用、配置已同步。
- 异常态：模板保存失败、无模板访问权限、配置仓绑定失败、同步失败、解析失败。
- 待澄清态：模板版本、回滚、覆盖策略和远程仓权限尚未确认时，在设计文档中保留待确认，不在界面暴露复杂策略。

## 前端设计

### 是否需要刷新

- 结论：是
- 理由：需要新增模板管理页面、模板编辑组件、空间初始化模板选择步骤、模板应用状态和 Harness 自动渲染联动。

### 页面、组件与状态

- 页面路由：
  - 全局团队模板管理页。
  - 租户团队模板管理页。
  - 团队模板编辑页。
  - 空间初始化向导中的模板选择步骤。
- 核心组件：
  - TeamTemplateList：模板列表。
  - TeamTemplateEditor：模板基础信息和配置编辑。
  - TemplateFolderTreeEditor：知识库文件夹体系编辑器。
  - TemplateRepositoryBindingForm：远程配置仓绑定表单。
  - TemplatePicker：团队空间初始化时选择模板。
  - TemplateApplicationStatus：模板应用、同步和解析状态展示。
  - TemplateScopeBadge：展示全局模板或租户级模板。
- 状态管理：
  - 模板列表状态：加载中、空、已加载、加载失败。
  - 模板编辑状态：草稿、保存中、保存失败、已保存。
  - 模板应用状态：待应用、应用中、文件夹已创建、仓库绑定中、同步中、解析中、成功、失败。
  - Harness 渲染状态：未解析、已解析、解析失败。

### 接口依赖与异常处理

- 依赖接口：
  - 查询全局团队模板列表。
  - 查询租户团队模板列表。
  - 创建、更新、启用、停用团队模板。
  - 查询团队模板详情。
  - 查询空间初始化可用模板。
  - 创建团队空间并应用模板。
  - 查询模板应用状态。
  - 查询配置仓同步和 Harness 解析结果。
- 异常处理：
  - 模板保存失败时保留编辑内容。
  - 模板应用失败时展示失败阶段，并允许进入手动配置或重试，具体策略待确认。
  - 同步或解析失败时，不阻塞团队空间进入，但需在知识库和 Harness 配置中心展示状态。
  - 权限不足时隐藏或禁用模板管理操作，并以后端错误为准。

## 后端设计

### 是否需要刷新

- 结论：是
- 理由：需要新增模板模型、模板范围权限、模板应用流程、远程配置仓绑定、同步任务和 Harness 配置解析联动。

### 接口与数据

- 核心服务：
  - TeamTemplateService：管理全局和租户级团队模板。
  - TemplatePermissionService：判断系统管理员、租户管理员和团队管理员对模板的可见与操作权限。
  - TemplateApplicationService：在空间初始化时应用模板。
  - TemplateRepositoryBindingService：根据模板创建远程配置仓绑定。
  - KnowledgeBaseService：根据模板初始化知识库文件夹体系。
  - ConfigScanService：扫描模板预置配置仓同步后的 Agent 使用配置。
  - HarnessConfigService：解析并渲染模板带来的 Harness 配置。
  - AuditService：记录模板创建、编辑、启停和应用。
- 核心数据对象：
  - TeamTemplate：模板 ID、范围、租户 ID、名称、描述、状态、知识库结构定义、创建人、更新时间。
  - TemplateScope：模板范围，取值为 `global` 或 `tenant`。
  - TemplateRepositoryBinding：模板 ID、仓库地址、分支或引用、配置目录、凭据引用、同步策略。
  - TemplateApplication：模板 ID、租户 ID、团队空间 ID、操作者、状态、应用阶段、失败原因、创建时间、更新时间。
  - TemplateFolderDefinition：模板文件夹节点 ID、父节点 ID、名称、排序、元数据。

### 业务规则、权限与异常处理

- 系统管理员可以创建和维护全局模板。
- 租户管理员只能创建和维护本租户模板。
- 团队管理员只能在创建团队空间时选择可见模板，不能维护全局或租户级模板。
- 全局模板对所有租户可见；租户级模板只对所属租户可见。
- 模板应用需要在团队空间创建事务或可恢复流程中记录 `TemplateApplication`，避免失败后无法追踪。
- 知识库文件夹创建、远程配置仓绑定、配置仓同步和 Harness 解析应记录阶段状态。
- 模板应用失败不能静默回滚用户可见结果；需要明确团队空间是否已创建、哪些步骤成功、哪些步骤失败。
- 模板停用后不应影响历史 `TemplateApplication` 记录。
- 模板格式、版本、回滚和覆盖策略待确认，数据模型需预留扩展字段。

## 验收标准

### 正常路径

- 系统管理员可以创建全局团队模板。
- 租户管理员可以创建本租户团队模板。
- 团队管理员初始化团队空间时，可以看到全局模板和当前租户模板。
- 团队管理员选择模板创建团队空间后，系统生成知识库文件夹体系。
- 模板预置远程配置仓绑定成功后，系统展示绑定信息和同步状态。
- 配置仓同步成功后，Harness 配置中心自动渲染 Agent、Skill、MCP、Workflow 和环境变量配置。
- 系统记录模板应用结果，包含模板、租户、团队空间、操作者和状态。

### 边界场景

- 当前租户没有租户级模板时，团队管理员仍可选择全局模板。
- 没有任何可用模板时，团队管理员可以进入手动初始化流程，是否强制模板待后续确认。
- 全局模板和租户模板同名时，页面明确展示模板范围。
- 模板停用后，不再出现在新团队空间初始化可选模板中。

### 异常场景

- 非系统管理员不能创建或编辑全局模板。
- 非租户管理员不能创建或编辑租户级模板。
- 模板应用过程中远程配置仓绑定失败时，系统展示失败阶段并记录应用失败。
- 配置仓同步失败时，Harness 配置中心展示同步失败状态。
- 配置解析失败时，系统展示解析错误，不影响团队空间基础信息访问。

### 权限场景

- 系统管理员可以维护全局模板，但不因此获得具体团队空间 Harness 写权限。
- 租户管理员可以维护本租户模板，但不因此获得具体团队空间 Harness 写权限。
- 团队管理员只能选择可见模板应用到自己有权限创建的团队空间。
- 模板创建、编辑、启停和应用都写入审计日志。
